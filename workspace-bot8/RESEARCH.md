# RESEARCH.md — 投研记忆系统操作手册

> 此文件是操作规范，描述如何记录研究、如何检索记忆、如何触发复盘。
> 由 AGENTS.md 引用。

---

## 记忆目录结构

```
memory/
  research/           ← 每次研究的完整记录（按日期命名）
    YYYY-MM-DD-{行业}.md
  views/              ← 对每个行业/赛道的持续跟踪观点（滚动更新）
    {行业名}.md
  predictions/        ← 明确预测登记 + 结果追踪
    tracker.md
  evolution/          ← 自我进化日志
    changelog.md      ← 每次进化内容
    review-log.md     ← 每次复盘的摘要和结论
```

所有文件均被 QMD 自动索引（BM25搜索），会话开始时可用 QMD 检索相关历史研究。

---

## 何时记录

以下情况，**主动**调用 `/record` 或直接写文件：

| 触发场景 | 记录类型 | 目标文件 |
|---------|---------|---------|
| 完成一次行业研究（sector-pulse） | research + views更新 | research/今日.md + views/{行业}.md |
| 得出明确预测（"X个月内会发生Y"） | prediction | predictions/tracker.md |
| 发现分析方法论盲点 | insight | 当日 research 文件 + MEMORY.md 方法论节 |
| 数据核实发现明显矛盾 | observation | 当日 research 文件 |
| 完成复盘后决定进化 | evolution | evolution/changelog.md |

## 研究记录格式（memory/research/YYYY-MM-DD-{行业}.md）

```markdown
# {行业} 研究记录 | {日期}

标签: {行业}, {相关股票代码}, {主题词}, {景气阶段}

## 结论
[直接结论：看多/看空/中性 + 核心理由]

## 数据支撑
[关键数据点，带日期和来源]

## 竞争格局判断
[行业集中度、护城河、格局稳定性]

## 景气度判断
[供需状态、价格趋势、所处阶段]

## 估值锚点
[行业估值区间、当前分位、参考标的]

## 值得关注的标的
[行业内重点公司及理由，不是推荐，是研究重点]

## 明确预测
→ 不写在此文件，单独登记到 `memory/predictions/tracker.md`（见下方格式）

## 方法论笔记（如有）
[这次研究中发现的分析盲点或有效方法]

## 信息来源
[Tushare报告期、zhipu搜索关键词、browser来源]
```

## 持续观点格式（memory/views/{行业}.md）

views 文件是滚动更新的行业档案。**每次更新必须同时记录支撑观点的事实数据快照**，供后续研究和复盘比对，而不仅仅是记录结论。

```markdown
# {行业} 持续跟踪 | 初建：{日期}

## 当前观点
**判断**：[看多/看空/中性]
**建立**：{日期}
**最后更新**：{日期}
**核心逻辑**：[1-3句]

## 历史观点演变
| 日期 | 判断 | 变化原因 |
|------|------|---------|

## 关键假设
[当前判断基于的核心假设，假设被证伪则需重新评估]

## 证伪条件
[什么数据/事件出现，我会改变判断]

## 数据快照历史

> 每次更新观点时，追加一条数据快照。保留所有历史快照，不要删改旧的。
> 复盘时可纵向对比数据变化，判断趋势是否符合预期。

### 快照 | {TODAY}

**来源报告期**：{Tushare 财务数据的报告期} | **行情日期**：{最近交易日}

**竞争格局**
- 集中度：[CR3/CR5 约 X%，分散/集中/寡头]
- 格局稳定性：[稳固/洗牌中/价格战]

**景气度**
- 需求端：[增速/趋势，数据来源+日期]
- 供给端：[产能/库存状态]
- 价格走势：[上涨/下跌/企稳，幅度]
- 所处阶段：[扩张加速/扩张放缓/触顶/下行/触底/反转]

**核心标的财务数据**（{报告期}）

| 公司 | ts_code | 营收YoY | 净利YoY | 毛利率 | ROE | 现金含量 |
|------|---------|---------|---------|-------|-----|---------|
| [公司名] | [代码] | [+X%] | [+X%] | [X%] | [X%] | [X.X] |

**估值快照**（{行情日期}）

| 公司 | PE | PB | PS | 市值(亿) | 近3M涨跌 |
|------|----|----|-----|---------|--------|

**北向资金**：近20日 [净流入/净流出] XX亿，趋势 [积极/消极/中性]

**政策环境**：[最新政策简述，原文日期：{日期}]

---

## 研究记录链接
- [{日期} 研究记录](../research/YYYY-MM-DD-{行业}.md)
```

## 预测登记格式（memory/predictions/tracker.md）

每个明确预测，追加一条记录：

```markdown
## P{序号} | {行业/标的} | 建立：{日期}

**预测类型**：行情预测 / 基本面预测
**预测内容**：[具体、可验证的陈述]
**预期时间**：[X个月内，给出截止日期 YYYY-MM-DD]
**建立时价格**：[登记时股价或指数点位，方便日后对比]
**验证方式**：[具体说明用什么数据核实，见下方示例]
**状态**：待验证 / ✅已验证（{验证日期}：[结果]）/ ❌已证伪（{日期}：[实际情况]）
**教训**（如证伪）：[预测哪里错了，下次怎么避免]
```

**验证方式示例**：

| 预测类型 | 验证工具 |
|---------|---------|
| 个股涨跌幅（"XX涨超20%"） | `tushare_daily(ts_code, 截止日)` 对比建立时收盘价 |
| 行业指数方向（"半导体指数突破X点"） | `tushare_index_daily(指数代码, 截止日)` |
| 个股相对强弱（"XX跑赢沪深300"） | `tushare_daily` + `tushare_index_daily(000300.SH)` 同期对比 |
| 营收/利润（"XX营收超Y亿"） | `tushare_income(ts_code, 报告期)` |
| 毛利率/ROE趋势 | `tushare_fina_indicator(ts_code, 报告期)` |
| 行业事件（"政策出台"、"出货量回升"） | zhipu MCP 搜索（⚠️ 时间验证必做） |

## QMD 检索方法

在需要回顾历史研究时，直接用自然语言搜索：

```
# 找某行业的历史研究
qmd search "半导体 研究 景气度"

# 找已有的观点
qmd search "AI算力 看多 判断"

# 找方法论笔记
qmd search "分析盲点 方法论"

# 找待验证的预测
qmd search "预测 待验证"
```

QMD 以 BM25 全文搜索工作，关键词越具体越准确。

## 记忆检索时机

以下情况，**先搜索记忆再行动**：

- 用户问某行业 → 先 `qmd search "{行业} 结论 判断"` 看有无历史研究
- 用户问某股票 → 先 `qmd search "{股票名} 估值 财务"` 看有无历史分析
- 开始新研究 → 先检索历史研究，避免重复劳动，直接做增量更新
- 收到新消息/新数据 → 检索相关历史观点，判断是否需要更新
