# 研报批判审阅 /report-critique

**触发词**：`/report-critique`、"这研报靠谱吗"、"帮我挑挑毛病"、"逻辑有没有问题"、"这个分析师水平怎么样"

**定位**：买方视角审视卖方研报。卖方天然有"推票"倾向，本 skill 的任务是剥掉包装，检验底层逻辑。

---

## 执行流程

### Step 1 — 卖方话术翻译

先扫描全文，找出典型的卖方委婉表达，翻译成大白话：

| 卖方原话 | 人话翻译 |
|---------|---------|
| "短期承压" | 业绩要变差 |
| "估值具有安全边际" | 跌了很多 |
| "建议关注" | 不太敢推但不写不行 |
| "盈利能力有望改善" | 现在赚不到钱 |
| "长期空间广阔" | 短期没啥催化剂 |
| "静待拐点" | 不知道什么时候能好 |
| "龙头优势显著" | 除了龙头其他都不行 |
| "行业景气度回升" | 之前太差了稍微好一点 |
| "维持买入评级" | 之前推了不好意思降级 |
| "首次覆盖，给予增持" | 标准话术，不代表真看好 |
| "下调盈利预测，维持买入" | 打脸了但不想认错 |

**注意**：不是所有正面表述都是话术，结合上下文判断。有数据支撑的正面结论应当尊重。

---

### Step 2 — 逻辑链检验

提取研报的核心投资逻辑链（参考 report-digest Step 4），然后逐环节检验：

```markdown
### 逻辑环节 1：{描述}
- **研报论述**：xxx
- **检验**：
  - 数据支撑：✅ 有 / ❌ 无 / ⚠️ 不充分
  - 因果关系：✅ 成立 / ❌ 跳跃 / ⚠️ 有条件成立
  - 替代解释：是否存在研报未考虑的其他可能
- **结论**：🟢 可信 / 🟡 需验证 / 🔴 存疑
```

重点检查：
- **数据→结论**：数据是否真的能推出这个结论？
- **相关≠因果**：是否把相关性当因果？
- **选择性引用**：是否只引用有利数据，忽略不利数据？
- **外推陷阱**：是否把短期趋势线性外推？

---

### Step 3 — 盈利预测审计

针对盈利预测表（如有），检查：

```markdown
| 检查项 | 结果 | 说明 |
|--------|------|------|
| 收入增速假设 | 激进/合理/保守 | vs 历史增速、行业均值 |
| 毛利率假设 | 激进/合理/保守 | 是否考虑竞争加剧/原材料波动 |
| 费用率假设 | 激进/合理/保守 | 研发/销售费用是否合理 |
| 税率假设 | | 是否有税收优惠到期风险 |
| 非经常性损益 | | 是否剔除/是否可持续 |
| 整体倾向 | 偏乐观/中性/偏悲观 | |
```

**关键指标**：如果盈利预测的实现需要多个乐观假设同时成立，标注为高风险。

---

### Step 4 — 估值方法审查

```markdown
- **使用的估值方法**：PE / PB / DCF / EV-EBITDA / 分部估值 / 其他
- **可比公司选择**：是否合理？是否cherry-pick了估值高的可比公司？
- **估值倍数**：给的是历史中枢、上限还是溢价？理由是否充分？
- **DCF 关键假设**（如适用）：WACC、永续增长率、预测期长度
- **隐含回报**：当前价→目标价的隐含回报率，是否有吸引力？
```

---

### Step 5 — 研报质量评分

```markdown
## 📋 研报质量评分

| 维度 | 分数（1-5） | 说明 |
|------|-----------|------|
| 数据详实度 | | 是否有足够的一手数据/行业数据 |
| 逻辑严密性 | | 因果推理是否完整无跳跃 |
| 假设合理性 | | 盈利预测假设是否经得起推敲 |
| 独立观点 | | 是否有区别于市场共识的独特见解 |
| 风险揭示 | | 是否诚实讨论了风险和不确定性 |
| **综合评分** | **X/5** | |

**评级含义**：
- ⭐⭐⭐⭐⭐：值得反复阅读，逻辑扎实有独立见解
- ⭐⭐⭐⭐：整体可靠，个别假设需自行验证
- ⭐⭐⭐：中规中矩，看看数据就行，结论参考价值有限
- ⭐⭐：逻辑有明显漏洞或过于模板化
- ⭐：纯走过场，不值得花时间
```

---

### Step 6 — 需要自行验证的清单

列出 3-5 个最关键的待验证项：

```markdown
## ✅ 自行验证清单

- [ ] {假设1}：建议通过 xxx 方式验证
- [ ] {假设2}：可查看 xxx 数据
- [ ] {假设3}：关注 xxx 时间节点
```

## 输出格式

```markdown
# 🔬 研报审阅：{标题}

## 卖方话术翻译
（Step 1，表格）

## 逻辑链检验
（Step 2，逐环节）

## 盈利预测审计
（Step 3，如适用）

## 估值审查
（Step 4，如适用）

## 📋 质量评分
（Step 5）

## ✅ 验证清单
（Step 6）
```

---

## 注意事项

- **批判不是否定**。好的研报应该得到认可，不要为了显得"有深度"而硬挑毛病
- **区分"研报质量"和"投资结论"**。一份高质量研报的结论也可能是错的，一份普通研报的结论也可能碰巧对了
- 卖方话术翻译要有分寸，不是所有正面表述都是"忽悠"
- 如果研报确实写得好（数据扎实、逻辑清晰、风险诚实），要明确说出来
