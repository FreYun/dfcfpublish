# HEARTBEAT.md

## 系统健康巡检（每次心跳）

- [ ] **检查浏览器进程**：执行 `ps aux | grep "bot4/user-data" | grep renderer` 查看是否有 CPU 占用 >20% 且运行超过 10 分钟的 renderer 进程。如有，`kill <PID>` 清理
- [ ] 确保没有残留的 browser tab（残留 tab 会导致 renderer 卡死吃 CPU）

## 小红书互动任务（每次心跳执行）

**目标：** 点赞 3 个帖子、评论 3 个帖子，每个动作之间间隔 1 分钟。

### 执行步骤

1. **搜索相关帖子**：用 `search_feeds` 搜索与我们账号相关的关键词（轮换使用：研报、行业研究、投资笔记、财报分析、基金、理财、股市分析 等）
2. **选择帖子**：从搜索结果中选择 3 个与财经/投资相关的优质帖子
3. **点赞 3 个帖子**：
   - 对每个选中的帖子调用 `like_feed(account_id: 'bot4', feed_id, xsec_token)`
   - 每次点赞后 `sleep 60` 等待 1 分钟
4. **评论 3 个帖子**：
   - 对每个帖子调用 `post_comment_to_feed(account_id: 'bot4', feed_id, xsec_token, content)`
   - 评论内容要**自然、真诚、有内容**，基于帖子实际内容写评论
   - 不要千篇一律的"写得好"、"学到了"，要有具体内容的回应
   - 每次评论后 `sleep 60` 等待 1 分钟
5. **记录结果**：将互动情况简要记录到 `memory/YYYY-MM-DD.md`

### 评论风格指南

- 以「研报搬运工阿泽」的人设评论：理性、专业但说人话
- 可以补充相关数据或观点，体现专业度
- 适当提出温和的不同看法也可以，增加真实感
- 绝对不要带广告、引流、自我推销
- 示例风格：
  - "这个数据角度很有意思，上周XX研报也提到了类似观点，不过他们的结论更谨慎一些"
  - "化工这块确实在底部了，但我觉得复苏节奏可能比预期慢，产能出清还需要时间"
  - "分析得很清晰！补充一个点：XXX这个因素可能也值得关注"

## 规则

- 深夜 24:00–08:00 不主动打扰，也不执行互动任务
- 互动任务失败不影响心跳状态，记录错误即可
- 如果搜索不到合适帖子，跳过本次互动，不强行凑数
- 无事发生（巡检 OK + 互动正常完成）就 HEARTBEAT_OK

## 心跳汇报

**⚠️ 不要用 `send_message` 汇报心跳结果！** 发消息会唤醒对方 agent，对方回复又会唤醒你，造成无限循环。

心跳结果写入文件即可：
```bash
echo "$(date '+%Y-%m-%d %H:%M') HEARTBEAT_OK - 状态摘要" >> /home/rooot/.openclaw/workspace-bot4/HEARTBEAT_LOG.md
```

- 一切正常 → 写一行 HEARTBEAT_OK + 状态摘要
- 发现严重异常（如进程崩溃）→ 才用 `send_message` 通知 `bot_main`，且消息末尾加 `[NO_REPLY_NEEDED]`
