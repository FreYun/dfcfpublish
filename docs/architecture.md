# OpenClaw 系统架构

## 总览

OpenClaw 是一个**多智能体自治系统**，管理 11 个内容创作 Bot + 多个系统 Agent，在小红书平台上进行自主内容创作、研究分析和社交互动。

## 核心组件说明

### 1. 内容 Bot (bot1 - bot11)

每个 Bot 是一个**独立人格的 AI 智能体**，拥有：

| 组件 | 说明 |
|------|------|
| SOUL.md | 核心价值观、行为底线、人格定义 |
| IDENTITY.md | 公开身份（名字、人设、Emoji） |
| AGENTS.md | 工作流程、记忆规则、发布权限 |
| TOOLS.md | 专属 MCP 端口、工具配置 |
| MEMORY.md + memory/ | 长期记忆 + 每日日记 |
| skills/ | 技能目录（symlink 共享 + 独有技能） |
| Chrome Profile | 独立浏览器身份 (`/home/rooot/.xhs-profiles/botN/`) |

**Bot 定位示例**：

| Bot | 人设 | 特色技能 |
|-----|------|---------|
| bot1 | 内容复盘型 | `fupan`（复盘） |
| bot3 | 视觉优先 | `visual-first-content` |
| bot5 | 宣妈慢慢变富 | 基金理财、黄金话题 |
| bot7 | 老K投资笔记 | 行业研究全栈（10+ 研究技能） |
| bot11 | 行业追踪 | 知识星球监控、板块追踪 |

### 2. 系统 Agent

| Agent | 角色 | 职责 |
|-------|------|------|
| **bot_main** | 研究部 HQ | 全局调度、健康监控、系统管理 |
| **mcp_publisher** | 印务局 | 内容合规审核 + 小红书发布执行 |
| **skills** | 技能部 | 技能库维护、状态巡检 |
| **coder** | 开发者 | 代码生成、系统维护 |

### 3. 技能系统 (Skill)

```
workspace/skills/                    ← 共享技能源文件
├── xiaohongshu-mcp/SKILL.md        ← 小红书浏览/互动
├── submit-to-publisher/SKILL.md    ← 发布提交流程
├── xhs-nurture/SKILL.md            ← 养号互动
├── xhs-op/                     ← 小红书全流程运营
├── research-mcp/SKILL.md           ← 金融数据查询
├── stock-watcher/SKILL.md          ← 个股监控
└── ...（25+ 技能）

workspace-botN/skills/
├── xhs-op → ../../workspace/skills/xhs-op                    (symlink)
└── fupan/SKILL.md                                            (bot 独有)
```

**改一处，全生效** — 通用技能通过 symlink 共享，修改源文件即可更新所有 Bot。

### 4. 消息通信

```
┌───────┐  send_message   ┌─────────────┐  飞书 API   ┌──────────┐
│ Bot A │ ──────────────→ │ Redis 消息线 │ ──────────→ │  飞书群   │
└───────┘                 │ inbox/outbox │             └──────────┘
                          └──────┬──────┘
                                 │ reply_message
                          ┌──────▼──────┐
                          │   Bot B     │
                          └─────────────┘
```

### 5. 发布流水线

```
Bot 创作内容
    │
    ▼
submit-to-publisher 技能
    │  写入 publish-queue/pending/
    ▼
mcp_publisher 拾取
    │  合规审核（标题/内容/标签/图片）
    ├─── ✓ 通过 → 调用小红书 MCP 发布 → published/
    └─── ✗ 拒绝 → 修改意见返回 Bot → Bot 修改后重新提交
```

### 6. 定时任务 (Cron)

23 个定时任务驱动系统自动运行：

| 类别 | 频率 | 示例 |
|------|------|------|
| 养号互动 | 每 3 小时 | bot1-7 浏览/点赞/评论 |
| 内容巡视 | 每日 2 次 | bot5 话题采集 |
| 行情速递 | 工作日早盘 | bot7 市场分析 |
| 系统巡检 | 每 3 小时 | bot_main 健康检查 |
| 发布监控 | 每 3 小时 | mcp_publisher 队列监控 |

## 技术栈

| 层级 | 技术 |
|------|------|
| Agent 引擎 | Claude Code Agent SDK (Node.js) |
| 浏览器自动化 | Rod (Go) + Chrome Headless |
| 工具协议 | MCP (Model Context Protocol) |
| 消息队列 | Redis Streams |
| 通知 | 飞书 API |
| AI 模型 | Claude / Kimi / Qwen / GLM / MiniMax |
| 监控 | 自建 Dashboard (Express + HTML5) |
| 调度 | 自建 Cron (JSON 配置) |
| 数据 | 文件系统 (JSONL / Markdown / YAML) |
