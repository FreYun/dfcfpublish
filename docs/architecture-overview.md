# OpenClaw — 多智能体内容运营平台

## 平台定位

OpenClaw 是一个**多智能体自治运营平台**，通过统一编排多个具备独立人格、记忆和专业技能的 AI Agent，实现社交媒体内容的规模化自主创作、研究分析与社区互动。

---

## 系统架构

![架构图](../media/architecture-diagram.png)

### 四层架构

**第一层 · 调度中枢**
- 全局任务编排与健康监控
- 跨 Agent 协作与消息路由
- 定时任务调度引擎（支持 cron 表达式，灵活配置各类周期任务）

**第二层 · 编辑部（智能体集群）**
- 20+ 个独立运营的智能体，各自拥有完整人格、记忆系统和专属技能
- 每个 Agent 同时具备**内容撰写**与**研究分析**能力，按人设侧重不同方向

**第三层 · 装备系统（Skill 集群）& MCP 能力服务**
- **装备系统**：模块化技能库，通过槽位机制为 Agent 灵活配置能力组合
- **MCP 能力服务**：社交平台引擎、金融数据网关、合规审核、AI 图片生成、组合管理

**第四层 · 数据与存储**
- Agent 配置与人格定义（Markdown 驱动）
- 分层记忆系统（日记 → 长期记忆 → 专题笔记）
- 发布流水线（待发布 → 审核 → 已发布 / 退回修改）
- 消息投递与去重机制

## 核心设计理念

### 1. 独立人格，统一管理

每个 Agent 拥有独立的身份定义、行为准则、记忆空间和技能组合，但共享统一的工具规范和通信协议。像管理一个团队一样管理 AI — 每个成员有个性，但遵循相同的工作流程。

### 2. 技能即插即用

采用**共享技能 + 专属技能**的混合架构：
- 通用技能一处修改、全员生效
- 专属技能赋予 Agent 独特的专业能力（如金融研究、视觉设计）
- 技能以声明式文件定义，无需重启即可热更新

### 3. 自主运营，人工兜底

- Agent 可自主完成日常内容创作、社区互动、数据研究
- 关键操作（如公开发布）经合规审核流水线把关
- 管理员可随时介入调整策略，Agent 保持学习和记忆

### 4. 多模型协同

平台支持接入多家 AI 模型提供商，根据任务类型智能选择最优模型，兼顾成本与效果。

## 装备系统

OpenClaw 借鉴 RPG 游戏的装备概念，将 Agent 的能力模块化为**技能装备**，通过"槽位"机制实现灵活配置。

### 七大槽位

| 槽位 | 名称 | 数量 | 说明 |
|------|------|------|------|
| ⛑️ 头盔 | 工种 | 1 | 决定 Agent 角色：前台（面向用户）或管理（内部运营） |
| 👔 护甲 | 职业 | 1 | 定义核心职能域：小红书运营、公众号发布、全栈开发等 |
| 💍 饰品 | 风格 | 2 | 写作风格 + 封面图风格，各一个槽位，决定内容调性 |
| 🔧 工具 | 通用技能 | 4 | 浏览器操作、合规审核、事件上报等基础能力 |
| ⚔️ 武器 | 研究技能 | 6 | 财报解读、资金流向、行业脉搏等专业研究能力 |
| 👢 靴子 | 策略 | 1 | 运营策略与优化方向 |
| ⏰ 定时 | 定时任务 | 6+ | 养号互动、定时发帖、市场早报等自动化任务 |

### 工作机制

- **装备即能力**：Agent 只能使用已装备的技能，未装备的能力自动屏蔽
- **一键换装**：通过 Dashboard 可视化界面拖拽装备，实时生效
- **依赖校验**：装备时自动检查前置依赖（如"定时发帖"需要先装备"发帖基础"）
- **共享技能库**：通用技能全员共享，修改源文件一处生效；专属技能仅特定 Agent 拥有
- **自动生成清单**：每次装备变更后，系统自动更新 Agent 的技能清单文件，确保 Agent 对自身能力有准确认知

## Agent 通讯系统

### 消息总线

所有 Agent 间通讯基于统一的**消息总线**，支持持久化、可追溯、异步投递。

**三种通讯原语**：
- **send_message** — 发起新对话（火忘式，不等待回复）
- **reply_message** — 回复消息（自动沿原路由链返回）
- **forward_message** — 转发给下一个 Agent（自动追加路由记录）

### Trace 路由机制

每条消息携带完整的**路由追踪链（Trace）**，记录消息经过的每一跳。回复时系统自动沿 Trace 逆向路由，无需硬编码任何地址。

```
管理员 → 魏忠贤(调度中枢) → bot7(研究员) → bot11(数据助手)
                                                    │
              ← 自动逆向路由 ← ← ← ← ← ← ← ← ← ←┘
```

这意味着：无论消息经过多少中间节点，回复总能准确送达发起者 — 包括外部即时通讯渠道。

### 外部消息通道

系统集成飞书等即时通讯平台，Agent 的消息可直接投递到管理员的聊天窗口：
- 定时任务结果自动推送
- 异常事件实时告警
- Agent 间协作过程可旁听
- 消息去重机制防止重复推送

## 系统 Agent 与职能分工

除内容创作 Agent 外，平台设有专职**系统 Agent**，各司其职：

### 调度中枢（总管）

平台的全局管理者，职责包括：
- 监控所有 Agent 状态（登录、发帖、异常）
- 派发任务、协调跨 Agent 协作
- 问题诊断与根因分析
- 关键事件上报管理员

**权限边界**：可自主执行诊断和告警，但代码变更、服务重启等操作需管理员授权。

### 印务局（发布执行中心）

内容发布的"最后一公里"，负责整个发布流水线：

```
Agent 提交内容
  ↓ 解析标题、正文、标签、配图
  ↓ 格式校验（标题长度、标签数量、发布间隔）
  ↓ 合规审核（AI 驱动的内容安全检查）
  ├── 通过 → 调用社交平台接口发布 → 归档
  └── 未通过 → 退回修改意见 → Agent 修改后重新提交
```

特色能力：
- 自动标签归一化（去重、格式统一、数量限制）
- 同账号发布频率控制（防止触发平台风控）
- 文字转卡片图（多种视觉风格可选）
- 发布日志与数据归档

### 技能部（Skill 进化）

技能生态的进化引擎：
- 通过沙箱环境孵化、迭代和优化技能
- 追踪技能版本演进历史
- 监控各 Agent 的技能装备完整性
- 维护全局技能库（100+ 技能），按类别组织
- 每日自动生成变更简报

### Coder（研发工程师）

平台的技术执行者，负责将需求转化为代码：
- **不直接写代码** — 而是编写精确的需求提示词，交由 AI 编程工具执行
- **沙箱优先** — 所有代码变更在隔离环境中进行，验证通过后合并
- **质量把关** — 每次交付前执行完整检查清单（差异审查、功能完整性、安全性）
- **严格重试策略** — 两轮未成功则停止，上报管理员，避免无效循环

## 能力服务层（MCP 服务）

平台的能力通过一组**微服务**提供，每个服务专注一个领域：

| 服务 | 能力 | 说明 |
|------|------|------|
| **社交平台引擎** | 内容发布、信息采集、社区互动、账号管理 | 单进程多租户架构，按 Agent 身份路由，每个 Agent 拥有独立的浏览器环境 |
| **金融数据网关** | A股/港股/美股行情、基金分析、债券监控、宏观数据、研报搜索 | 92+ 数据工具，按角色分级授权（全权限 / 基金顾问 / 内容创作者） |
| **合规审核引擎** | 内容安全检查、敏感词检测、规则校验 | AI 驱动 + 规则引擎双重把关，发布前必经环节 |
| **AI 图片生成** | 文生图、多风格封面、参考图风格迁移 | 支持多种生成模型，图片自动保存到 Agent 工作空间 |
| **组合管理** | 持仓追踪、净值计算、调仓记录、冷静期检查 | 投研类 Agent 的核心数据支撑 |
| **技能沙箱** | 隔离环境中开发和测试新技能 | 技能部的研发工具，保障主系统稳定性 |

## 技术亮点

- **Agent 数量弹性扩展**：新增 Agent 仅需创建配置文件，分钟级上线
- **全链路可观测**：Dashboard 实时监控所有 Agent 状态、任务执行与发布数据
- **记忆持久化**：Agent 跨会话保持记忆，持续积累经验和领域知识
- **多平台就绪**：工具层抽象设计，可扩展至其他社交平台
