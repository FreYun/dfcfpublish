# OpenClaw 记忆系统

> 简洁介绍 OpenClaw 如何存储、更新和召回记忆

## 1. 记忆如何存储

### 存储位置

所有记忆存储在 **workspace 目录**（默认 `~/.openclaw/workspace/`）：

```
~/.openclaw/workspace/
├── SOUL.md                    # Agent 的"灵魂"（性格、原则）
├── USER.md                    # 用户信息
├── AGENTS.md                  # 工作空间说明
├── MEMORY.md                  # 长期记忆（可选，需要创建）
└── memory/                    # 每日日志目录（需要创建）
    └── YYYY-MM-DD.md          # 每日日志
```

### 存储格式

- **Markdown 文件**：人类可读，可直接编辑
- **JSONL 会话记录**：`~/.openclaw/agents/main/sessions/*.jsonl`（自动保存，完整对话历史）
- **SQLite 索引**：`~/.openclaw/memory/main.sqlite`（向量索引，用于搜索）

## 2. 记忆如何更新

### 自动更新

#### 方式 1: `/new` 命令触发
- 使用 `/new` 命令时，`session-memory` hook 自动执行
- 将上一个会话的最后 15 条消息保存到 `memory/YYYY-MM-DD-slug.md`
- 文件名由 LLM 生成描述性 slug（如 `2026-02-06-api-design.md`）

#### 方式 2: 压缩前记忆刷新
- 当会话 token 接近压缩阈值时自动触发
- 触发条件：`contextTokens > contextWindow - 20000 - 4000`
- 系统静默提示模型写入重要记忆到 `memory/YYYY-MM-DD.md`
- 用户看不到这个回合（模型通常回复 `NO_REPLY`）

### 手动更新

#### 模型主动写入
- 模型使用 `write` 工具创建/更新记忆文件
- 例如：用户说"记住这个"时，模型会写入

#### 文件监控自动索引
- 系统监控 `MEMORY.md` 和 `memory/` 目录的文件变化
- 文件修改后自动更新 SQLite 索引（防抖 1.5 秒）

---

## 3. 记忆如何召回

### 搜索工具

#### `memory_search` - 语义搜索
- **功能**：在 `MEMORY.md` + `memory/*.md` 中语义搜索
- **方式**：混合搜索（向量搜索 + 关键词搜索）
- **返回**：相关片段、文件路径、行范围、相似度分数
- **使用场景**：回答关于历史工作、决策、日期、人物、偏好等问题

#### `memory_get` - 读取文件
- **功能**：读取特定记忆文件的片段
- **参数**：文件路径、起始行、行数
- **使用场景**：在 `memory_search` 找到相关文件后，读取完整内容

### 搜索机制

#### 混合搜索（Hybrid Search）
- **向量搜索**：语义理解，找到意思相同但用词不同的内容
- **关键词搜索**：精确匹配标识符、代码符号、错误信息等
- **综合分数**：`finalScore = 0.7 * vectorScore + 0.3 * textScore`

#### 索引更新
- **增量同步**：只处理变化的文件（基于 hash 检测）
- **批量嵌入**：支持批量处理，提高效率
- **嵌入缓存**：相同文本只计算一次嵌入向量

## 4. 最终的记忆结构

### 核心文件

#### `SOUL.md` - Agent 的"灵魂"
- **作用**：定义 Agent 的性格、原则、边界
- **内容**：核心真理、行为准则、沟通风格
- **特点**：每个会话开始时读取，Agent 可以更新

#### `USER.md` - 用户信息
- **作用**：记录用户的基本信息和偏好
- **内容**：姓名、时区、偏好、项目上下文
- **特点**：逐步完善，随着交互更新

#### `AGENTS.md` - 工作空间说明
- **作用**：工作空间的指南和规则
- **内容**：会话启动流程、记忆使用规则、工具说明
- **特点**：框架文件，定义工作流程

### 记忆文件

#### `MEMORY.md` - 长期记忆（可选）
- **作用**：存储持久性事实、决策、偏好
- **内容**：重要决策、项目上下文、个人笔记
- **特点**：
  - 只在主会话（私聊）中加载
  - 不在群组上下文中加载（安全考虑）
  - 需要模型主动创建和更新

#### `memory/YYYY-MM-DD.md` - 每日日志
- **作用**：记录当天的活动和上下文
- **内容**：会话摘要、活动记录、技术细节
- **特点**：
  - 追加模式（append-only）
  - 会话开始时读取今天和昨天的内容
  - 需要模型或 hook 创建

### 文件关系

```
会话启动流程：
1. 读取 SOUL.md（我是谁）
2. 读取 USER.md（用户是谁）
3. 读取 memory/YYYY-MM-DD.md（最近发生了什么）
4. 如果是主会话，读取 MEMORY.md（长期记忆）

记忆更新流程：
1. 日常笔记 → memory/YYYY-MM-DD.md
2. 重要决策 → MEMORY.md
3. 文件变化 → 自动索引到 SQLite
4. 搜索时 → 从 SQLite 索引召回
```

---

## 总结

### 存储
- **Markdown 文件**：人类可读，Git 友好
- **JSONL 会话**：完整对话历史（自动保存）
- **SQLite 索引**：向量索引，支持搜索

### 更新
- **自动**：`/new` 命令、压缩前刷新、文件监控
- **手动**：模型使用 `write` 工具

### 召回
- **语义搜索**：`memory_search` 工具
- **文件读取**：`memory_get` 工具
- **混合搜索**：向量 + 关键词，兼顾语义和精确性

### 结构
- **SOUL.md**：Agent 性格
- **USER.md**：用户信息
- **MEMORY.md**：长期记忆
- **memory/YYYY-MM-DD.md**：每日日志

---

**关键点**：
- 记忆文件**不是自动创建的**，需要模型或 hook 创建
- 会话记录（JSONL）**自动保存**，完整记录所有对话
- 记忆系统支持**混合搜索**，兼顾语义理解和精确匹配
- 所有数据存储在 `~/.openclaw/` 下，与源代码分离
