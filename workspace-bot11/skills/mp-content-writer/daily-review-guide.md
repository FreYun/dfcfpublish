# 日复盘公众号文章：从数据到成文

本指南将 research-mcp 采集的结构化数据，转化为一篇 1500-2500 字的公众号日复盘文章。

> 数据采集（research-mcp 工具调用、板块逻辑解读、外部信息搜索）的完整流程见 SKILL.md "A. 日复盘素材" 部分，此处不再重复。

---

## 一、从数据到观点

**拿到数据后不要急着写，先做三件事：**

### 1.1 找今天的"主故事"

从复盘数据中找出今天最有解读空间的 1-2 个信号：

| 信号来源 | 怎么找 | 什么算好信号 |
|---------|--------|------------|
| 指数走势分化 | 上证 vs 创业板涨跌方向相反 | 风格切换、资金迁移 |
| 日内画像反转 | 上午跌下午涨（V）或相反（倒V） | 盘中发生了什么事？ |
| 板块异动 | TOP5 有新面孔，或某板块连续多日出现 | 新主线还是一日游？ |
| 情绪极端值 | 情绪评分 > 75（过热）或 < 30（冰点） | 拐点信号 |
| 涨跌停结构 | 涨停数暴增/暴减、跌停扎堆某行业 | 情绪拐点或风险暴露 |
| 资金方向 | 北向大进大出、融资余额突变 | 增量/减量信号 |
| 成交额变化 | 对比前一日（放量/缩量显著时） | 量能是否支持当前走势 |

**选择标准：选"读者明天开盘前最想知道的事"，而非"今天数据最大的变化"。**

### 1.2 形成观点

围绕主故事回答三个问题：
1. **发生了什么？**（盘面现象，1-2 句话描述）
2. **为什么发生？**（背后逻辑，政策/资金/情绪/事件驱动？）
3. **意味着什么？**（对后市的影响，对操作的启示）

### 1.3 选新闻

从 WebSearch 结果中选 3-5 条满足以下条件的新闻：
- 与主故事相关，或与投资决策直接相关
- 能给出差异化点评（不能只转述）
- 排除纯情绪性标题党、非财经新闻

## 二、文章结构模板

```
┌─────────────────────────────────────────────┐
│ 标题（5-15 字）                              │
│ ← 口语化、有情绪、不透露全部                  │
├─────────────────────────────────────────────┤
│ 核心观点引语（blockquote，1 句话）             │
│ ← 今天最重要的一个判断                        │
├─────────────────────────────────────────────┤
│ 【一句话概览】                                │
│ ← 指数涨跌 + 成交额 + 涨跌家数，3 行内搞定    │
├─────────────────────────────────────────────┤
│ 【主题段】（占全文 60%，800-1500 字）          │
│ ├── 主题 1：核心故事展开                      │
│ │   现象 → 原因 → 推演 → 操作含义             │
│ │   穿插数据（板块涨跌、个股、资金流向）       │
│ └── 主题 2（可选）：次要但重要的信号           │
├─────────────────────────────────────────────┤
│ 【新闻简评】（3-5 条，300-500 字）             │
│ ← 每条：一句话新闻 + 2-3 句点评               │
│   点评角度：对投资有什么影响？                  │
├─────────────────────────────────────────────┤
│ 结尾（2-3 句话）                              │
│ ← 总结今天的调性 + 明天值得关注的事            │
│   不强行给结论，可以留问题                     │
├─────────────────────────────────────────────┤
│ 风险提示（固定模板）                          │
└─────────────────────────────────────────────┘
```

## 三、数据 → 文章段落的映射

### 3.1 一句话概览

**写法模板：**
```
今天大 A {涨跌描述}。{主要指数涨跌}，成交 {金额} 亿，{涨跌家数描述}。
```

**示例：**
> 今天大 A 继续分化。上证微跌 0.26%，创业板涨 1.41%，成交 2.3 万亿，涨多跌少。

注意：不要写成数据报表，用自然语言一句话概括。

### 3.2 主题段

**数据来源组合（按信号类型）：**

| 信号类型 | 需要的数据 | research-mcp 工具 |
|---------|-----------|-------------------|
| 风格切换 | 指数对比 + 板块概况 | `market_overview` + `get_ashares_index_quote` |
| 板块异动 | 板块排名 + 龙头股走势 | `market_overview` + `get_stock_daily_quote`（龙头股近30日） |
| 情绪拐点 | 恐慌指数 + 成交额变化 | `get_ashares_gvix` + `get_ashares_turnover` |
| 量能变化 | 成交额走势 | `get_ashares_turnover`（拉近期对比） |
| 资金信号 | 北向 + 国债收益率 + 指数估值 | `get_stock_northbound_holding` + `get_cn_bond_yield` + `get_ashares_index_val`（算 ERP） |
| 政策驱动 | 新闻 + 研报 + 相关个股走势 | `news_search` + `research_search` + `get_stock_daily_quote` |

**主题段写作要点：**
- 不是把数据翻译成文字，而是用数据支撑你的判断
- 数据嵌入叙述中，不要单独列表格（那是复盘数据的事，不是文章的事）
- 引用具体数字时，给出简短解读："创业板涨了 1.4%，这已经是连续第三天跑赢大盘了"
- 讨论板块逻辑时，必须查龙头股近期走势（`get_stock_daily_quote`）和相关指数（`get_ashares_index_quote`），读者需要知道"从哪来、现在在哪、可能往哪去"
- 板块逻辑不许瞎猜，必须用 `research_search` 搜研报 + `news_search` 搜新闻 + 知识星球交叉验证（详见 SKILL.md）

### 3.3 新闻简评

**格式模板（每条新闻）：**
```
**{新闻标题/事件}。** {2-3 句点评，角度：这事对投资有什么影响}
```

**示例：**
> **央行今日净投放 2000 亿逆回购。** 跨季资金面本来就紧，这个量不算小，说明央行在主动呵护。短期利率中枢大概率稳住，对债市偏利好。

### 3.4 结尾

- 总结今天的盘面调性（一句话）
- 提一个明天值得关注的变量
- 可以用问句收尾（制造期待感）

**示例：**
> 今天的分化说明市场还在选方向，资金从周期往科技搬家的趋势第三天了。明天重点看创业板能不能放量站稳 3350，如果缩量回踩，可能只是反抽。

## 四、标题生成

从主题段中提炼标题，套用以下模板：

| 今日盘面特征 | 标题方向 | 示例 |
|------------|---------|------|
| 分化明显 | 口语反问 | "科技和周期，今天选了谁？" |
| 大涨 | 情绪+克制 | "久违的大阳。" |
| 大跌 | 自嘲/调侃 | "又学到了。" |
| 窄幅震荡 | 悬念 | "等一个信号" |
| 板块异动 | 聚焦 | "种子股突然涨疯了" |
| 趋势延续 | 判断 | "搬家还没搬完" |
| 政策驱动 | 反问 | "降准之后呢？" |

## 五、质量自检

写完后过一遍：

- [ ] 有没有主题？（不是流水账）
- [ ] 有没有观点？（不是数据翻译）
- [ ] 开头 3 行能不能留住人？
- [ ] 数据引用准确吗？（核对 research-mcp 返回的原始数据）
- [ ] 没有具体个股买卖建议？
- [ ] 有风险提示？
- [ ] 字数 1500-2500？
- [ ] 标题 5-15 字？像人说的话？
