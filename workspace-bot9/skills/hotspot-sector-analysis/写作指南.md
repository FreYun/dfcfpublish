# 热点板块追踪 — 写作指南

> 完整规则红线见 [SKILL.md](SKILL.md)。下方摘录最容易违反的铁律，写稿前务必过一遍。

---

## ⛔ 写稿前铁律（每次写稿前必读）

### 数据准确铁律

1. **所有数据来自权威渠道** — 行情用 Tushare/research-mcp，观点用 research-mcp 搜索
2. **标注来源** — 每个具体数字/观点都必须可追溯
3. **禁止虚假数据** — 不许编造、猜测任何数字，查不到就不写
4. **时效性 24h** — 只用 24 小时内的资讯和数据，严禁使用陈旧数据

### 合规红线

- **不给操作建议** — 不说"建议买入/卖出""建议抄底""建议减仓"等任何指导读者具体操作的话
- 不说"配置方向""建议配置"，用"重点关注""看好方向""值得留意"
- 不说"必涨/必跌/稳赚"，用"存在机会/需要警惕"
- 不提个股代码/名称，用行业板块描述（基金代码/名称除外）
- 文末必加风险提示
- **正文不放链接** — 成品正文中不要出现任何 URL 链接

## 整体框架

```
【标题】≤ 30 字，吸睛但不标题党，核心信息前置

【正文】

模块 1：行情播报   ← 事实，板块指数 + 个股表现
         【配图】  ← 人工截图占位
模块 2：深度分析   ← 分析，2-4 个核心驱动原因
模块 3：机构观点   ← 引述，2-4 家机构最新观点
模块 4：相关基金   ← 3-5 只板块相关基金
         数据来源声明

【风险提示】       ← 必加，不可省略
```

总字数：约 1200 字（不含配图区域和基金展示）。

## 标题写法

- ≤ 30 字，核心信息前置，突出板块异动
- 风格：有冲击力但不虚假，用事实说话
- 句式参考：
  - "XX分钟，直线涨停！XXX，集中来袭！"
  - "暴涨近XX%！大XX集体拉升！两大利好，集中驱动！"
  - "XXX突发大利好！整个板块被\u201c带飞\u201d！"
  - "热点 | XXX持续发酵，XX板块继续领涨"
  - "产业化再提速！XX板块早盘活跃，龙头股年内翻倍"
- 可用感叹号增强语气，但不超过 3 个
- **禁止**："必涨""稳赚""不看后悔"等标题党

## 模块 1：行情播报

> 纯事实描述，只写行情数据，不加分析判断。

**字数**：≤ 150 字

**内容要素**：
1. 板块指数名称 + 当日涨跌幅
2. 龙头个股表现（涨停/大涨/大跌的代表性个股 3-5 只）
3. 港股/海外相关标的联动（如有相关标的出现明显异动）
4. 【配图】占位行——人工截图东财 APP 板块走势图

**格式示例**：

```
X月X日早盘/午盘/收盘，XX概念股集体走强/走弱，板块指数大涨/大跌超X%。
个股表现尤为活跃，XXX、XXX、XXX等涨停/大跌；XXX、XXX大涨/大跌超X%。

港股方面，XXX一度大涨/大跌超X%，带动整个XX板块人气飙升/承压。

（图片来源：东方财富APP，不作投资推荐）
```

**注意**：
- 只描述行情事实，不加"利好""利空"等定性词
- 个股只写名称，不写代码
- 配图说明统一格式：（图片来源：东方财富APP，不作投资推荐）

## 模块 2：热点板块深度分析

> 回答"为什么涨/跌"——从消息面、政策面、产业面、资金面等角度筛选最核心的驱动原因。

**字数**：500-600 字

**结构**：
- 开头一句话引入："今日XX板块的XX并非偶然，而是多重因素共振的结果，其核心驱动力可归结为以下X点："
- 逐条展开 2-3 个核心原因，每条用**加粗引导句 + 展开论述**的格式
- 每个原因必须有**数据或事件**支撑

**原因筛选优先级**：
1. 重大政策/监管动态（政府工作报告、部委发文等）
2. 产业数据/事件驱动（行业数据发布、龙头公司公告、技术突破）
3. 资金面信号（北向资金、机构调仓、ETF 申购）
4. 海外联动（海外同类标的异动、国际事件传导）

**格式示例**：

```
今日创新药板块的爆发并非偶然，而是多重利好共振的结果，
其核心驱动力可归结为以下三点：

首先，BD出海持续爆发，业绩兑现超预期。
中国创新药企的全球竞争力正加速转化为真金白银。
数据显示，截至X月X日，2026年中国创新药出海BD总包已达XXX亿美元……

其次，政策定位历史性升级，行业战略意义凸显。
2026年政府工作报告首次将生物医药列为"新兴支柱产业"……

最后，重磅临床数据披露在即，后续催化充足。
随着X月XXX年会临近，预计将有大量中国药企的重磅临床数据集中披露……
```

**写法要点**：
- 先说结论（"XX原因"），再展开论据
- 用"首先""其次""最后"或"第一""第二"等连接词，保持逻辑递进
- 每个原因的展开不超过 120 字
- 引用数据必须标注来源（"数据显示""据XX统计"）
- **与模块 1 的行情表现逻辑一致** — 原因要能解释为什么板块涨/跌
- 禁止无依据的猜测（"可能是因为情绪面……"不可接受，除非有数据佐证）

## 模块 3：机构观点分析

> 汇总权威机构对该板块的最新研判，增加文章专业度和可信度。

**字数**：400-500 字

**引用机构数量**：2-4 家

**机构类型优先级**：
1. 券商研究所（中信、中金、国泰君安、华泰等）
2. 公募基金公司（公开观点、基金经理访谈）
3. 外资机构（高盛、摩根、瑞银等）
4. 行业研究机构

**格式示例**：

```
对于XX板块的强势表现及未来走向，多家权威机构给出了积极/审慎研判。

XX证券在其研报中明确表示，"引述原文观点"。该机构认为，
在XX逻辑催化下，板块正迈入XX阶段，看好板块的中长期价值。

XX证券则从XX角度分析，指出……。该机构强调，
XX作为XX产业中最核心的环节，受益方向尤为明确。

XX机构指出，在XX背景下，XX行业将进入XX新阶段。
同时，该机构还前瞻性地提出，XX预计在2026年迎来发展拐点。

XX证券表示，XX板块年初至今……均在不断强化XX逻辑。
该机构认为，从中期维度看……对未来XX充满信心。
```

**写法要点**：
- **每条观点必须注明出处**：格式用"XX机构表示""XX机构认为""XX研报指出"
- 多家共识用："XX、YY等多家机构均认为……"，不要笼统说"多家机构"而不点名
- **不要全盘照搬**：提炼核心判断，用自己的话和人设风格串联
- **当日观点优先**：引用非当日研报需确认观点未过时
- 如有看多/看空分歧，呈现不同声音更有深度
- 引用方式遵循 [引用来源管理.md](../daily-market-recap/引用来源管理.md) 的规则

## 模块 4：相关基金

> 结合热点板块，从基金池中选取相关主题基金供读者参考。

**操作步骤**：
1. `Read skills/daily-market-recap/基金池.md` 读取基金池
2. 按 [基金推荐规则.md](基金推荐规则.md) 选取 3-5 只与板块高度相关的权益类基金
3. 按规定格式展示

**展示位置**：紧跟模块 3 之后，独立成段

**格式示例**：

```
下面也整理了部分XX相关的基金供参考：

xxx基金A（012345，基金经理：张三）
xxx基金A（012346，基金经理：李四）
xxx基金A（012347，基金经理：王五）

（数据来源：东方财富Choice数据、银河证券，具体基金的相关风险、基金经理变更等
情况详见各基金法律文件及公告，历史数据不代表未来，不作投资推荐）
```

**注意**：
- 尽量不选 ETF，优先选权益类主动管理基金
- 语气用"供参考""可以关注"，不用"建议买入""推荐"
- 数据来源声明必加

## 语言风格

| 模块 | 风格 |
|------|------|
| 模块 1 行情播报 | 客观、中性、数据导向，无修饰 |
| 模块 2 深度分析 | 专业、严谨，因果链清晰，有数据支撑 |
| 模块 3 机构观点 | 引述为主，归因清晰，呈现共识与分歧 |
| 模块 4 相关基金 | 简洁、中立，供参考语气 |

通用：
- 专业但不晦涩，读者是有一定基础的散户
- 多用短句，段落不超过 5 行
- 数据说话，每个判断都有数据/事实支撑
- **全角标点**：正文中所有标点必须使用中文全角（双引号用\u201c\u201d，逗号用，，句号用。，冒号用：，括号用（）），基金代码等纯数字场景除外
- 正文中不放任何 URL 链接

## ⛔ 强制自检输出（交稿前必做，不可跳过）

上面的质量自检**不能只在心里过一遍**。写完稿件后、提交给研究部之前，必须把自检结果**明文写出来**，跟稿件一起发送。格式：

```
【自检报告】

模块 1 行情播报：
- [✅/❌] 板块指数涨跌幅数据已验证
- [✅/❌] 无定性词，纯事实描述

模块 2 深度分析：
- [✅/❌] X个核心原因，均有数据/事件支撑
- [✅/❌] 原因与行情表现逻辑一致
- [✅/❌] 数据已标注来源

模块 3 机构观点：
- [✅/❌] 引用了X家机构（XX证券、YY证券、……）
- [✅/❌] 每条观点注明出处
- [✅/❌] 观点时效性已确认

模块 4 相关基金：
- [✅/❌] X只基金来自研究部基金池
- [✅/❌] 数据来源声明已加

合规红线：
- [✅/❌] 全文无个股代码
- [✅/❌] 全文无操作建议
- [✅/❌] 文末风险提示格式正确

引用来源：
- [✅/❌] 已附【引用来源（不发布）】清单（或本期无外部引用）
```

**铁律：**
- 有任何一条 ❌ → **先改稿再交**，不许带着 ❌ 提交
- 不许跳过自检直接交稿
- 自检报告放在稿件**之后**，研究部会看

## ⛔ 自检之后、交稿之前：必须执行 fact-check（不可跳过）

**自检报告 ≠ 数据核查。** 上面的自检清单检查的是格式、合规、结构完整性。但文中每个具体数字是否准确，必须通过 fact-check skill 单独验证。

完成自检后，**必须执行以下操作**：

```
Read skills/fact-check/SKILL.md
```

然后按 fact-check 流程逐条核实文中所有数字和事实性表述，输出**独立的数据核查报告**。

**交稿顺序：稿件 → 自检报告 → fact-check 核查报告。三者缺一不可。**

如果你只输出了自检报告就准备交稿，停下来——你还差 fact-check 这一步。
