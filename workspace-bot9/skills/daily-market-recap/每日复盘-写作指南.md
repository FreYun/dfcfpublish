# 每日盘面复盘 — 写作指南

> 规则红线（新闻摘录铁律、浏览器铁律、合规红线、行情时效性例外、研报引用规则）见 [SKILL.md](SKILL.md)，不在此重复。

固定结构，不要自由发挥：`01 重磅资讯 → 02 数据复盘 → 03 盘面解读 → 04 后市展望`。

---

## 整体框架

```
【标题】≤ 30 字，从当日最重磅的一条资讯中提炼
        例："央行发声，事关A股！"
        例："美联储按兵不动，A股怎么走？"

【正文】

01 重磅资讯   ← 事实，从研究部提供的链接中摘录
02 数据复盘   ← 配图建议，研究部后期人工插图
03 盘面解读   ← 分析，bot 自己写
04 后市展望   ← 观点，bot 自己写

【风险提示】  ← 必加，不可省略
```

总字数：2000-3500 字（不含 02 配图区域）。

---

## 标题写法

- ≤ 30 字，核心信息前置
- 从当日最重磅的一条资讯中提炼
- 句式参考：
  - "央行发声，事关A股！"
  - "美联储按兵不动，A股怎么走？"
  - "油价飙升，谁在受益？"
  - "XXX数据出炉，市场如何反应？"
- 不用标题党，但要有信息量和吸引力

## 01 重磅资讯

### 工作流程

1. 接收研究部发来的 3-5 条新闻链接
2. 用浏览器逐条打开，阅读全文
3. 从每条新闻中**摘取至多 2 段内容**——核心在于新闻事实本身，可适当摘取原文中的解读部分，但不要喧宾夺主
4. 按下方格式组织输出

### 格式

每条新闻：**编号 + 加粗标题 + 摘录段落**

```
1.央行：坚定维护股票、债券、外汇等金融市场平稳运行

X月X日，中国人民银行党委召开扩大会议。会议指出，积极稳妥化解
重点领域金融风险。……（摘录关键段落）……

会议强调，……（摘录关键段落）……

2.鲍威尔：美经济前景"极不确定" 联储内部倾向减少降息次数

在周三的新闻发布会上，鲍威尔直言……（摘录关键段落）……

3.市场监管总局：加力破除地方保护与市场分割

据市场监管总局网站，……（摘录关键段落）……
```

### 选材优先级

1. 央行/银保监/证监会等监管动态
2. 美联储/海外央行决议、发言
3. 国务院/发改委/工信部等产业政策
4. 地缘政治重大事件（影响市场的）
5. 行业重大数据发布（PMI、CPI、社融等）

## 02 数据复盘

### 定位

当日行情数据的可视化呈现。**此区域以插图为主，由研究部后期人工编辑插入。** bot 只需写配图建议即可。

### bot 输出内容

```
02 数据复盘

【配图建议】
1. 主要指数日K走势截图（上证/深成/创业板），数据来源：Choice
2. 板块涨跌幅排行截图（涨幅前5 + 跌幅前5），数据来源：Choice
3. 北向资金近期净流入走势图，数据来源：Choice

数据来源：Choice，数据时间：YYYY-MM-DD
```

### 规则

- bot 不插图、不画表、不填数据，只写配图建议
- 建议 2-3 张图即可，根据当日情况选择
- 标注数据来源和日期

## 行情数据获取（03/04 写作前必做）

写 03 盘面解读和 04 后市展望前，先通过 **research-mcp** 获取当日行情数据（详见 `skills/research-mcp/SKILL.md`）。

**推荐调用顺序：**

```
# 1. A股大盘总览（指数涨跌、成交额）
npx mcporter call "research-mcp.market_overview()"

# 2. 主要指数行情（如需更详细）
npx mcporter call 'research-mcp.get_ashares_index_quote(symbol: "000001.SH,000300.SH,399006.SZ", start_date: "YYYYMMDD", end_date: "YYYYMMDD")'

# 3. 港股总览（如当日港股相关）
npx mcporter call "research-mcp.get_hshares_market_overview()"

# 4. 美股指数（如隔夜美股影响大）
npx mcporter call 'research-mcp.get_usstock_index_quote(symbol: "DJIA.GI,SPX.GI,NDX.GI", start_date: "YYYYMMDD", end_date: "YYYYMMDD")'

# 5. 新闻搜索（补充消息面素材）
npx mcporter call 'research-mcp.news_search(query: "关键词", top_k: 5)'
```

> 日期格式：`YYYYMMDD`（如 `20260324`），不是 `YYYY-MM-DD`。

拿到的数据用于：
- **03 盘面解读**：引用具体涨跌幅、成交额、板块表现
- **04 后市展望**：引用技术面位置、资金面数据、估值水平
- **02 配图建议**：基于数据给出更精准的配图建议

## 03 盘面解读

### 定位

连接 01 的消息面与行情数据，**解释"今天市场为什么这样走"**。400-600 字。

### 结构

```
第一段：一句话概括大势
  → 涨跌幅、成交量变化、大小盘分化、赚钱效应

第二段起：板块结构性分析
  → 哪些板块强/弱
  → 为什么强/弱（对应 01 中的消息面）
  → 板块之间的联动关系（跷跷板效应、资金轮动等）
  → 消息面催化（简述事件，不需要全文再引，01 已有）
```

### 写法要点

- **解释"为什么"**，不要只罗列涨跌数字
- 消息面→板块反应→资金行为，建立因果链
- 引用消息面时一句话简述即可（"消息面上，XXX"），不需要再次全文摘录
- 可以指出市场的结构性特征（如：算力与资源的跷跷板效应）
- 语言节奏：短句为主，段落不超过 5 行

## 04 后市展望

### 定位

全文核心价值部分，**篇幅最长（800-1200 字）**。bot 结合消息面、数据面、技术面输出观点，体现专业水平和差异化。

### 结构

按以下顺序展开，每段自然过渡，不需要再加小标题：

```
a) 短期技术面判断（200-300 字）
   - 指数所处位置（箱体上沿/下沿、支撑位、整数关口）
   - 当前交易情绪（冰点/回暖/过热）
   - 近期关键信号（如：跌破整数关口后的修复预期）
   - 明日/短期需关注的要点

b) 外部环境（150-200 字）
   - 海外市场、地缘局势、油价/汇率
   - 美联储/海外央行政策走向
   - 对 A 股的传导路径和影响程度判断

c) 内部环境（150-200 字）
   - 政策预期（"十五五"、新质生产力等）
   - 资金面（大资金动向、外资回流等）
   - 产业趋势（产能重估、实物属性重定价等）

d) 配置方向（200-300 字）
   - 聚焦 2-3 个核心方向，每个方向说清逻辑链
   - 用行业/板块描述，不提个股代码/名称
   - 区分中线主线 vs 短线题材
   - 给出具体的产业催化线索（如：云服务涨价→AIDC/算力租赁）

e) 题材与风控提示（100-200 字）
   - 轮动方向判断
   - 当前市场流动性环境下的风险提示
   - 不追高、不盲目抄底的具体建议

f) 风险提示（必加）
   "市场有风险，投资需谨慎。"
   "以上仅为个人观点，不构成投资建议。"
```

### 写法要点

- **要有明确观点**，不要两头堵。"牛市进程并未结束，目前处于震荡整固阶段"比"可能涨也可能跌"强一百倍
- 观点必须有逻辑支撑，不能空喊口号
- 用"关注""可以留意"代替"建议买入"
- 用"存在机会""需要警惕"代替"必涨""必跌"
- 配置方向要具体到板块/产业链环节，不要泛泛而谈
- 段落之间自然过渡，读起来是一篇连贯的分析文，不是拼凑的要点列表

## 语言风格

| 区域 | 风格 |
|------|------|
| 01 重磅资讯 | 客观、中性、无修饰，原文摘录 |
| 02 数据复盘 | 配图建议，标注来源 |
| 03 盘面解读 | 专业、简洁，因果链清晰 |
| 04 后市展望 | 可有立场、可体现人设风格、需逻辑支撑 |

通用：
- 专业但不晦涩，读者是有一定基础的散户
- 多用短句，段落不超过 5 行
- 数据说话，每个判断都有数据/事实支撑
- 03/04 可体现人设风格和个性（从 SOUL.md 出发），01/02 保持客观中性

## 质量自检

- [ ] 标题 ≤ 30 字，从当日最重磅资讯提炼
- [ ] 01 新闻全部从研究部链接中摘录，无改写、无杜撰、无语义变更
- [ ] 01 新闻无主观评价词（"利好""利空""重磅"等）
- [ ] 01 中涉及行情数据的段落已用浏览器核实最新行情
- [ ] 01 用浏览器访问链接，未使用爬虫
- [ ] 02 写了配图建议和数据来源
- [ ] 03 解释了"为什么"，不只是罗列数字
- [ ] 04 有明确观点，不两头堵
- [ ] 04 配置方向具体到板块/产业链，不泛泛而谈
- [ ] 04 引用的研报为当日发布，或已确认历史研报观点未过时
- [ ] 全文无个股代码/名称
- [ ] 全文无"建议买入/卖出""必涨/必跌"等违规表述
- [ ] 文末有风险提示
- [ ] **读一遍 03/04，听起来像"我"说的话吗？**（人设一致性）
