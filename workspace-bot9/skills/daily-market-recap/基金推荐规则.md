# 基金推荐规则

## 触发条件（默认必须带）

**写复盘时，基金推荐是必选项，不是可选项。** 在撰写"04 后市展望"之前，必须先 `Read skills/daily-market-recap/基金池.md`，为每个重点关注方向匹配基金。每个方向 3 只。

⛔ **禁止写完复盘后再回头补基金。** 基金推荐必须在初稿中就包含，不能等研究部追问才补。如果你提交的复盘稿件没有基金推荐，视为未完成。

## 基金来源

天天基金研究部基金池，保存在本地文件。当前文件同时包含权益基金和指数型基金：

```
Read skills/daily-market-recap/基金池.md
```

直接读取该文件，从中按规则匹配选取基金，不要只看主题字段。

### 更新基金池

当研究部说「更新一下基金池」或类似指令时，执行以下命令：

```bash
python3 /home/rooot/.openclaw/scripts/sync-fund-pool.py
```

脚本会自动从腾讯文档同步最新的权益基金和指数型基金数据到本地基金池文件。执行完后告知研究部更新结果（基金数量、主题分布）。

### 月份检查

每次读取时，检查文件顶部的 `month` 字段是否为当月：
- **是当月** → 正常使用
- **不是当月** → 提醒研究部更新基金池文件，本次复盘可暂不带基金

## 选取优先级

先按来源层级排序，再在同层候选中做二次筛选。

### 来源优先级

这里的“东财指数 / 东财权益 / 其他指数 / 其他权益”，指的是 **东财指数型基金、东财权益型基金、其他指数型基金、其他权益型基金** 四类候选。

1. **东财指数型基金**
2. **东财权益型基金**
3. **其他指数型基金**
4. **其他权益型基金**

如果高优先级层级里已经能找到合适基金，就不要降级到后面的层级凑数。

### 基金池对应关系

- **指数基金相关候选**：从 `基金池.md` 的**指数基金池**部分查找
- **权益基金相关候选**：从 `基金池.md` 的**权益基金池**部分查找
- 不要把指数基金和权益基金混在同一个候选层级里理解

### 同层筛选规则

1. **优先 C 类** — 同一基金如果同时有 A/C 类，优先选 C 类；没有 C 类时再选 A 类或其他份额
2. **不要直接推荐 ETF** — 如果命中的指数型基金条目本身是 ETF，或基金名称中明确带有 `ETF` 字样，不要直接把 ETF 写进推荐列表；应继续读取其 `可购买代码`，优先落到对应的 **C 类份额** 作为最终推荐结果
3. **同一基金不同份额不要重复推荐** — 同一管理人、同一产品线只保留一个最优份额

### 匹配规则

匹配顺序固定为：

1. **跟踪指数**
2. **基金名称**
3. **主题**

具体执行时：

- 先看 **跟踪指数/指数名称** 是否与当前重点关注方向直接对应，这是最强匹配
- 跟踪指数不够明确时，再看 **基金名称** 是否直接体现该方向
- 前两者都不够明确时，才退到看 **主题字段**
- 仅主题能勉强蹭上但前两层完全不匹配的，宁可不选，不要硬塞

## 展示格式

**基金推荐必须按板块/方向归类，集中在"重点关注方向"段落末尾统一展示，不要散落在各段里。**

格式如下（注意全角标点）：

```
感兴趣的朋友可以关注相关方向的基金：
农业板块：xxx基金A（012345，基金经理：张三）、xxx基金A（012346，基金经理：李四）、xxx基金A（012347，基金经理：王五）
基建板块：xxx基金A（012348，基金经理：赵六）、xxx基金A（012349，基金经理：钱七）
新能源板块：xxx基金A（012350，基金经理：孙八）、xxx基金A（012351，基金经理：周九）、xxx基金A（012352，基金经理：吴十）
```

**要求：**
- 每个板块独占一行，板块名用中文全角冒号"："分隔
- 同板块内基金用中文顿号"、"分隔
- 每个板块推荐 2-3 只基金，不要只推 1 只
- 每只基金展示：基金名称（代码，基金经理：姓名）
- 基金代码用半角括号（纯数字场景）
- 若索引到的是 ETF，最终推荐结果必须替换为其 `可购买代码` 对应的可买份额，优先 C 类
- 同一候选池中优先保留 C 类份额
- 集中放在一个位置，不要把基金散落在文章各段里
- 语气用"感兴趣的朋友可以关注""可以留意"，不用"建议买入""推荐"

## 仅以下情况可以不带基金

只有满足以下**全部条件**时才可以不带：
1. 基金池中确实没有任何主题能匹配上重点关注方向（需在稿件末尾注明"基金池无匹配主题"）
2. 当日行情极端波动（单日跌幅超3%），配置信号完全混乱

**以下不是跳过基金的理由：**
- ❌ "分析比较笼统" — 那就把分析写具体，然后带基金
- ❌ "忘了读基金池" — 流程要求在写后市展望前就读
- ❌ "觉得硬塞" — 参考 bot7 的写法，基金推荐应该是分析的自然延伸，不是硬塞
