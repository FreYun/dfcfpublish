# 行情数据获取（03/04 写作前必做）

## 时间判断

research-mcp 的 A 股指数数据通常在 **16:00 后更新当日收盘数据**。取数前先判断当前时间：

- **16:00 之前**（盘中或刚收盘）：research-mcp 还没有当日数据，**必须走下面的三步流程**（Tushare API 取大盘指数+申万一级行业 + 浏览器校对 + research-mcp 补充历史）
- **16:00 之后**：research-mcp 已更新当日收盘数据，**可以直接用 research-mcp 取所有数据**，跳过第一步 Tushare API 和第二步浏览器。具体：
  - 大盘指数：`market_overview()` 或 `get_ashares_index_quote()`
  - 申万一级行业涨跌：`get_ashares_index_quote(symbol: "801010.SI,801080.SI,...")`（申万一级行业也是指数）
  - 浏览器仅在需要校对数据时使用

---

## 第一步：Tushare API 取当日大盘指数 + 申万一级行业（16:00 前必做，16:00 后可跳过）

通过 Tushare Pro HTTP API 获取**当日实时**大盘指数和申万一级行业数据。如果当前已过 16:00，research-mcp 已有当日数据，可直接用 research-mcp 代替本步骤。

**A 股指数（实时，接口 `rt_idx_k`）：**

```bash
curl -s -X POST https://api.tushare.pro \
  -H 'Content-Type: application/json' \
  -d '{
    "api_name": "rt_idx_k",
    "token": "从 TOOLS.md 获取 Tushare Token",
    "params": {"ts_code": "000001.SH,399001.SZ,399006.SZ,000300.SH,000016.SH,000905.SH,000852.SH,932000.CSI,000688.SH,899050.BJ"},
    "fields": "ts_code,name,trade_time,close,pre_close,high,low,vol,amount"
  }'
```

> 常用 A 股宽基指数代码：
> | 代码 | 名称 |
> |------|------|
> | `000001.SH` | 上证指数 |
> | `399001.SZ` | 深证成指 |
> | `399006.SZ` | 创业板指 |
> | `000016.SH` | 上证50 |
> | `000300.SH` | 沪深300 |
> | `000905.SH` | 中证500 |
> | `000852.SH` | 中证1000 |
> | `932000.CSI` | 中证2000 |
> | `000688.SH` | 科创50 |
> | `899050.BJ` | 北证50 |
>
> - 返回字段：`close`（最新价/收盘价）、`pre_close`（昨收）、`vol`（成交量）、`amount`（成交额）
> - ⚠️ **频率限制：每小时最多 1 次**，一次调用取完所有需要的指数

**申万一级行业（实时，接口 `rt_sw_k`）：**

```bash
curl -s -X POST https://api.tushare.pro \
  -H 'Content-Type: application/json' \
  -d '{
    "api_name": "rt_sw_k",
    "token": "从 TOOLS.md 获取 Tushare Token",
    "params": {"level": "1"},
    "fields": "ts_code,name,trade_time,close,pre_close,high,low,vol,amount"
  }'
```

> - `level: "1"` 表示申万一级行业（共 31 个）
> - 返回字段与 A 股指数一致
> - ⚠️ 如果 `rt_sw_k` 接口无权限或返回失败，**降级到第二步用浏览器取申万行业数据**

**国际指数（当日收盘后可取，接口 `index_global`）：**

```bash
# 日韩港美等国际指数（已收盘市场当日即可取到）
curl -s -X POST https://api.tushare.pro \
  -H 'Content-Type: application/json' \
  -d '{
    "api_name": "index_global",
    "token": "从 TOOLS.md 获取 Tushare Token",
    "params": {"ts_code": "N225", "start_date": "YYYYMMDD", "end_date": "YYYYMMDD"},
    "fields": "ts_code,trade_date,close,pre_close,pct_chg,change"
  }'
```

> 常用国际指数代码：
> | 代码 | 名称 |
> |------|------|
> | `HSI` | 恒生指数 |
> | `HKTECH` | 恒生科技指数 |
> | `N225` | 日经225指数 |
> | `KS11` | 韩国综合指数 |
> | `DJI` | 道琼斯工业指数 |
> | `SPX` | 标普500指数 |
> | `IXIC` | 纳斯达克指数 |
> | `XIN9` | 富时中国A50指数 |
>
> 日期格式：`YYYYMMDD`。可一次查一个代码，按需选取当日相关的国际市场。

## 第二步：浏览器校对 + API 失败时的备用（16:00 前按需，16:00 后可跳过）

浏览器用于两个场景：
1. **校对**：与第一步 API 取的大盘指数和申万一级行业数据做交叉验证
2. **降级备用**：第一步 `rt_sw_k` 接口无权限或失败时，用浏览器取申万一级行业数据

> 16:00 后 research-mcp 已更新，可用 `get_ashares_index_quote(symbol: "申万行业代码")` 直接取申万一级行业涨跌，浏览器仅在需要校对时使用。

```
# 1. 打开东方财富首页，获取大盘指数（校对用）
browser open url="https://www.eastmoney.com" profile="bot9"
browser snapshot profile="bot9"

# 记录以下数据：
# a) 大盘指数：上证、深成指、创业板指的收盘点位和涨跌幅
#    → 与第一步 Tushare API 取的数据做校对
#    → 如果两者差异超过 0.03%，立即向研究部报告，以浏览器数据为准
# b) 两市成交额

# 2. 查看申万一级行业涨跌排行（API 成功时用于校对，API 失败时作为主数据源）
browser open url="https://www.swsresearch.com/institute_sw/allIndex/releasedIndex" profile="bot9"
browser snapshot profile="bot9"
# 记录：涨幅前5和跌幅前5的申万一级行业及涨跌幅

# 3. 用完关闭
browser close profile="bot9"
```

> ⚠️ **校对规则**：浏览器取的数据与 Tushare API 数据必须交叉验证（大盘指数和申万一级行业均适用）。发现不一致时：
> - 差异 ≤ 0.03%：可能是实时 vs 收盘的微小差别，以浏览器收盘数据为准
> - 差异 > 0.03%：**立即向研究部报告**，说明两个数据源的具体数值，等待指示后再写稿

## 第三步：research-mcp 补充历史数据和其他指标

Tushare 和浏览器解决了当日数据，**research-mcp 用于补充近期走势、估值、资金面等**：

```
# 1. A股近期走势（用于技术面分析）
npx mcporter call "research-mcp.market_overview()"

# 2. 恐慌指数（如需）
npx mcporter call 'research-mcp.get_ashares_gvix(start_date: "YYYYMMDD", end_date: "YYYYMMDD")'

# 3. 新闻搜索（补充消息面素材）
npx mcporter call 'research-mcp.news_search(query: "关键词", top_k: 5)'
```

> 日期格式：`YYYYMMDD`（如 `20260324`），不是 `YYYY-MM-DD`。

## 行情数据铁律

- **行业分类只用申万一级行业**（如：电子、计算机、医药生物、有色金属等），不许用同花顺概念板块、东财行业分类、自创分类等其他口径
- 大宗商品行情、概念题材行情仍可正常引用，不受此限制

拿到的数据用于：
- **03 盘面解读**：引用收盘点位、涨跌幅、成交额、板块表现（行业口径：申万一级），灵活组织写作即可
- **04 后市展望**：引用技术面位置（点位是判断支撑/阻力的基础）、资金面数据、估值水平
- **02 配图建议**：基于数据给出更精准的配图建议
