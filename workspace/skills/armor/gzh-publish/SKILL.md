## 铁律

1. **默认保存草稿** — 不直接发表，除非研究部明确要求。发表是不可逆操作。
2. **先检查登录** — 每次操作前先截图确认登录状态，token 是否有效。
3. **人设一致** — 内容符合 SOUL.md / CONTENT_STYLE.md，不暴露研究部。
4. **AI 创作须声明** — AI 辅助内容发表时选择「去声明」标注来源，平台合规要求。
5. **关键步骤截图确认** — 填写内容、保存/发表前后都要截图验证。

## 发文流程

### 写稿前

**必须先读以下两个文件，否则不得开始写稿：**
1. Read [公众号-发文前必读.md](公众号-发文前必读.md) — 红线、禁用词、Checklist
2. Read [公众号-合规速查.md](公众号-合规速查.md) — 金融合规、风险提示模板

然后用 `mem0_search` 查过往发帖、读 `memory/写稿经验.md` + `CONTENT_STYLE.md`（如有），保证连续性。原文档案可在 `memory/posts/` 兜底查阅。

### 三步发布

```
1. 写稿 → 先读「发文前必读」+「合规速查」，再由 gzh-write skill 完成内容生成，按 CONTENT_STYLE.md 检查风格、排版、结构
2. 浏览器操作 → Read 公众号-浏览器发布.md，进入编辑器填写内容
3. 保存草稿 → 截图确认成功，通知研究部审阅
```

### 发布权限

| 场景 | 动作 |
|------|------|
| 日常内容 | 保存草稿，通知研究部 |
| 敏感话题、强投资建议 | **研究部确认**后才可发表 |
| 长文/深度分析 | 保存草稿，建议研究部过目后再发 |

### 首次发文

账号从未发过文章时，按**新账号**方式措辞。后续发文**建立在前一次基础上**，有连续性。

### 发文后

1. 更新 `memory/公众号发帖记录.md`（日期、标题、正文摘要）
2. 更新当日 `memory/YYYY-MM-DD.md`
3. 被打回时更新 `memory/公众号写稿经验.md`
