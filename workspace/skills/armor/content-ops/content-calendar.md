# 📅 选题日历

> 没有规划的账号像随机播放的歌单——每首歌单独听还行，听完记不住是谁的专辑。每个账号是一个**角色**在经历一段**旅程**，每篇帖子都应该能回答："这篇在 ta 的故事里是第几章？"

排期文件位置：`workspace-{self}/queue/botN.md`（例如 `workspace-sys4/queue/bot1.md`）。本文档下文出现的 `queue/botN.md` 都是这个相对路径。

**不要写进 `skills/content-ops/queue/`**——skill 是 symlink 装备的，sandbox 禁止向 symlink 外写入；且排期是 agent 自身状态，应该归属 agent 而非 skill。

## 每日运行流程

> **触发方式**：cron 每日一次（建议上午 9:30）。
> **原则**：轻量快速，有变化才改 queue，无事则跳过。
>
> ⚠️ **本流程不写 `cron/bulletin.jsonl`**。bulletin 是 xhs-topic-collector / skill-evolution 等 skill 的广播通道,排期流程不占它。如果你从昨天的日记里看到"Step 6.5 写 bulletin"的痕迹,那是历史残留,**忽略它**,当前流程到 Step 4 更新 queue 就结束,后续只写今日工作日记 `memory/YYYY-MM-DD.md`,不要 append bulletin.jsonl。

### Step 1 — 数据刷新（🚨 硬性前置，失败即停）

```bash
bash /home/rooot/.openclaw/dashboard/query-stats.sh --summary
```

对有新发布的 bot 追加：

```bash
bash /home/rooot/.openclaw/dashboard/query-stats.sh {botN} --top=3 --with-content
bash /home/rooot/.openclaw/dashboard/query-stats.sh {botN} --titles
```

> `--with-content` 自动从 `workspace-sys1/publish-queue/published/` 匹配标题附正文。数据来自 `dashboard/xhs-stats.json`（cron 9:00/21:00 自动采集）。**用绝对路径**——agent 的 cwd 不固定。

**🚨 FAIL-LOUD 协议**（上次翻车：sys4 声称命令"不存在"就跳过 Step 1，然后在「数据回顾」里**编造**了浏览量/点赞数）：

1. **必须真跑命令**，把 stdout 原文贴日记，不能只用 Glob/LS 判断"文件存不存在"
2. **退出码 ≠ 0 或 stderr 非空 → 立即停止排期**：不写 queue / 在日记记完整错误 / 等研究部介入
3. **绝对禁止**用印象数字、旧日记、诊断 context 填「数据回顾」。查不到就留空 + 注明"本次未拉取到数据"
4. queue 「数据回顾」里每个数字必须能在当次 stdout 原字符串里搜到

### Step 1.5 — 生成/更新「状态告警」块

> **为什么这一步**：draft-review 的"完整"档（新 bot / 风格调整后 / 数据下滑期 / 高风险窗口）**不是审稿时识别的**——审稿时 sys4 既没有时间序列数据，也没有合规词库。识别责任在排期这一步，产物是 `queue/botN.md` 顶部的「状态告警」块（格式见下文「queue/botN.md 格式」节）。draft-review 只读这个块，不自己跑数据。

对每个有发布数据的 bot，逐项判断并更新告警块：

| 告警类 | 判定方式 | 截止日 |
|---|---|---|
| 新 bot 前 2 周 | `stat -c '%Y' workspace-botN`，距今 < 14 天 | 创建 + 14 天 |
| 风格调整后首周 | `stat -c '%Y' workspace-botN/{SOUL.md, skills/*-style/SKILL.md}`，任一 mtime < 7 天 | 修改 + 7 天 |
| 数据下滑期 | 三条粗启发任一命中：① stdout 死亡率 ≥ 30% ② `--top=5` 近 5 篇 likes ≤ 2 ③ 比上次排期日的 likes/notes 比值掉 ≥ 50% | 7 天后复查 |
| 高风险窗口 | 政策/监管敏感、涉个股具体买卖、敏感词边界、数据存在内部冲突。在当日条目加 `⚠️ 高风险` 标签 + 理由 | 窗口结束日 |

**截止日到期清理**：每条告警的截止日 ≤ 今天时，重新检查触发条件——还在 → 续期；不在 → **取消勾选**清空填充值。

**没有任何告警时**：保留空模板**不删**——这是给 draft-review 的"已检查、无告警"明确信号，区分"没检查"和"没告警"。

### Step 2 — 环境扫描

快速 web search：

1. `今日 经济数据` / `今日 财报` — 有确定性事件吗？
2. 各 bot 支柱关键词 + "小红书 热门" — 有热点吗？

搜不到就跳过，不要瞎猜。有就插入/调整对应 bot 的 queue。

### Step 3 — 读取 bot 素材库

`workspace-botN/memory/topic-library.md` 是排期选题的**第一输入源**——bot 自己采集的素材最贴合其人设，自带角度和时效判断。

**bot 的素材库标记是输入，不是最终决策**。bot 在 topic-library 里打的 🔥🔥 / 🔥 / 🌲 标记只是它自己的视角，sys4 作为主编必须**重新判定每条素材的实际优先级**。判定维度（按重要性排序）：

1. **时效窗口**：这条新闻/数据/事件的热度还能持续几天？过期的"紧急"就不紧急
2. **跨 bot 协调**：同天/近 7 天有没有其他 bot 排了类似选题？要避撞或差异化
3. **bot 的节奏**：这个 bot 本周已经排了什么类型？同支柱连发 3 篇会疲劳
4. **数据预期**：类似选题之前的数据如何？历史死亡的话题不要再推
5. **环境契合**：Step 2 的环境扫描里有没有强相关热点能放大这条素材？

**所以会出现这些情况**——都很正常：

- bot 标 🔥🔥 但 sys4 判断只值 🔥（事件已经过气了，或者撞题了）
- bot 标 🌲 常青但 sys4 拉到今天发（环境契合度突然变高）
- bot 标 🔥 但 sys4 直接砍掉（同支柱已经连 2 篇了）
- bot 没标的（素材库里没有）但 sys4 从 Step 2 发现的热点直接补进 queue

不复制素材库内容到 queue，只引用方向和角度。素材库里已有角度分析的，可以采用或基于 5 个维度微调；**素材库为空，或所有素材都被 sys4 判定不发**，再从 Step 2 环境扫描结果自行安排。

> **铁律**：sys4 的判定理由必须能在 queue 条目的「理由」字段或日记里写出来，不能只写"按 bot 标记排的"——这是 sys4 主编层的职责，不能转嫁给 bot。

### Step 4 — 更新 queue

**日常更新（每天）**：
- 已发布的条目：标 📝 已发布 + 实际数据
- 今天要推送的条目：确认选题仍合适（没被热点或新素材取代）
- 紧急素材但 queue 没排 → 插入；强相关热点 → 插入今天或明天

**周度规划（每周一额外）**：
- 读各 bot SOUL.md 确认人设和支柱
- 参考 `patterns/INDEX.md` 近期活跃模式
- 通读素材库，补未来 7 天排期
- 跨 bot 撞题检查（规则见下文「排期规则 · 跨 bot 撞题检查」）
- 更新「数据回顾」和「月度方向」

**月度调整（每月初额外）**：用 `query-stats.sh` 汇总上月数据 / 更新「月度方向」 / 清理过期备选池。

### Step 5 — 推送当日任务（通过 send_message 内联派发）

**重要（2026-04-15 改）**：不再向 `workspace-botN/今日选题.md` 写文件——sys4 的 Claude CLI sandbox 会 block 跨 workspace 写入。改成用 `send_message` 工具把今日选题**内联到消息体**传给 bot。

对今天有 ⏳ 待推送 条目的 bot，调用：

```

  to: "botN",
  content: <下面这段完整文本，把 {占位符} 替换成真实内容>,
  trace: [{"agent": "sys4"}]

```

### 字段铁律

- **状态告警块**永远在文件最顶（紧接 `> 最后更新`），方便 draft-review O(1) 读到
- **数据回顾**节的每个数字必须能在当次 `query-stats.sh` 的 stdout 原字符串里搜到。**禁止**靠印象、旧日记、诊断 context 编造数据
- **👁 = views（浏览），不是 impressions（曝光）**。query-stats.sh 输出的 `impressions` 是曝光（笔记出现在 feed 的次数），`views` 才是浏览（用户点进去看的次数），两者差 3-5 倍。👁 一律填 views 字段，不要填 impressions。描述时"浏览量"对应 views，"曝光量"对应 impressions，不要混用
- **状态字段**只用 `✅ 已推送 / 📝 已发布 / ⏳ 待推送 / ⏳ 待定` 这四个值，不要发明别的
- **高风险标签**：高风险选题在当日条目第一行加 `⚠️ 高风险` + 理由（同时在状态告警块勾选「高风险窗口」）

## 排期规则

### 节奏控制

- **轻重交替** — 不连发深度长文，不连发短评
- **支柱轮转** — 各支柱穿插，不连续 3 篇同支柱
- **每日至少一篇** — 每个 bot 每天至少 1 篇，保持账号活跃度和算法权重。素材库里有常青素材兜底，不存在"没东西发"
- **热点优先** — 强相关热点来了，打破常规节奏，当天可发 2 篇（热点 + 原计划）

### 跨 bot 撞题检查

> 同天同话题两个号发，读者会觉得"这是一个团队在操作"。

打开所有 `queue/botN.md`，检查：

- [ ] 同天内不同 bot 选题是否重合？
- [ ] 同热点各 bot 角度是否足够不同？
- [ ] 近 7 天内是否多个 bot 都在聊同一话题？

**冲突处理**：

1. **角度不同** → 允许，但时间错开至少 2 小时
2. **角度相似** → 保留更契合的那个，另一个换选题或延后
3. **多 bot 同话题** → 各自视角必须不同（宣妈→妈妈怎么看，老K→行业怎么看，来财→小白怎么看）

### 与 bot 素材库的关系

部分 bot 自己维护 `memory/topic-library.md`（素材库）：

- **运营 agent 排期 > bot 自选** — queue 里有安排的日子，bot 按排期写
- **空白日 bot 自选** — queue 里没安排的日子，bot 可以从自己的素材库选题
- **bot 素材库是输入** — 周度规划时参考 bot 素材库选合适选题排进 queue
- **不重复维护** — queue 只管排期和推送，不复制素材库的内容

## 参考知识

### A. 内容支柱设计方法

每个 bot 应该有 **3-4 个内容支柱**，所有内容都能归入某个支柱。

**设计三步法**：

1. **从 SOUL.md 提取**：身份标签、专业领域、性格特征、差异化点
2. **交叉组合**：`内容支柱 = 专业领域 × 内容形式 × 人设角度`
3. **三问检验**：
   - 可持续吗？（能写 20 篇不重复？）
   - 有差异吗？（系统内其他 bot 角度不同？）
   - 有受众吗？（小红书上有人搜？）

### B. 节奏参考

| 账号阶段 | 频率 | 理由 |
|---------|------|------|
| 所有阶段 | 每天至少 1 篇 | 保持算法权重和账号活跃度 |
| 冷启动（<1000粉） | 每天 1-2 篇 | 快速积累，让算法认识你 |
| 成长期（1000-1万粉） | 每天 1 篇 + 热点加发 | 稳定输出，热点时加量 |
| 成熟期（>1万粉） | 每天 1 篇，质量优先 | 保底日更，重质不重量 |

**时间节点意识**：

| 节点 | 内容方向 | 示例 |
|------|---------|------|
| 每周一 | 新周展望/上周复盘 | "新的一周，市场最该关注这几件事" |
| 每月初 | 月度复盘 | "3月操作回顾：对了2次错了1次" |
| 数据日 | CPI、美联储、财报季 | 提前预热，当天即时反应 |
| 季节性 | 年终总结、开年展望 | "2026上半年成绩单" |

### C. 系列化模式

| 模式 | 说明 | 示例 |
|------|------|------|
| 计划型 | 提前规划 3-5 篇 | "黄金入门三部曲" |
| 衍生型 | 从爆款评论区长出来 | 某篇互动高 → 围绕高频问题出延伸 |
| 固定栏目 | 周期性仪式感 | "宣妈每周金价日记"（每周五） |

**系列命名**：包含 bot 名字 + 数字暗示连续性 + ≤ 10 字。

### D. 热点响应三道过滤器

```
过滤器1：相关性 — 跟该 bot 内容支柱有关吗？
         ↓ 有关
过滤器2：独特性 — 该 bot 能提供独特视角吗？
         ↓ 能
过滤器3：时效性 — 现在发还来得及吗？（>24h 需要新角度）
         ↓ 来得及
→ 值得追
```

| 策略 | 场景 | 时效 | 深度 |
|------|------|------|------|
| 闪评 | 突发，有即时观点 | 2h内 | 150-300字 |
| 深度拆解 | 热点背后有逻辑链 | 24h内 | 600+字 |
| 反转视角 | 情绪一边倒 | 48h内 | 300-600字 |

### E. 差异化矩阵

| 维度 | 说明 | 示例 |
