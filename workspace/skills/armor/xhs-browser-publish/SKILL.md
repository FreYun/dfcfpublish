# 使用 OpenClaw 浏览器发布小红书笔记

本 skill 通过 OpenClaw 内置的 browser 工具（`profile: "bot11"`）操作小红书创作者中心网页完成笔记发布。

**触发条件：** 默认小红书操作方式。所有小红书相关操作（发帖、浏览、评论、点赞等）优先使用本 skill。仅当浏览器不可用或小富龙明确要求时才改用 MCP 方式。

**重要：发布笔记时最后一步为「存草稿」，不直接发布。**

## 前置条件

1. OpenClaw 浏览器可用（browser 工具的 `profile: "bot11"`）
2. 已登录小红书创作者中心，或准备好扫码登录
3. **发帖前必须对照 `memory/小红书限流规则备忘.md` 做敏感词与限流自检**

---

## 发布模式一览

| 模式 | 入口 URL | 适用场景 |
|------|----------|----------|
| 长文笔记 | `https://creator.xiaohongshu.com/publish/publish?from=tab_switch&target=article` | 千字以上深度内容，支持一键排版、多模板；需账号已开通长文权限（内测/定向邀请） |
| 图文笔记（直达） | `https://creator.xiaohongshu.com/publish/publish?from=tab_switch&target=image` | 1–9 张图 + 标题正文，封面 3:4 竖版为佳，高频发帖 |
| 视频笔记 | `https://creator.xiaohongshu.com/publish/publish?from=tab_switch&target=video` | 竖屏 9:16 为佳，普通用户≤5 分钟，专业号可更长 |

## Step 0: 登录

### 0.1 打开创作者中心

使用 browser 工具打开 `https://creator.xiaohongshu.com`。

### 0.2 检查登录状态

截图（snapshot）检查页面：
- 如果看到创作者后台界面（有左侧导航栏、用户头像等）→ 已登录，跳到 Step 1
- 如果看到登录页面 → 需要登录

### 0.3 登录流程

1. 页面上应显示二维码或手机号登录选项
2. 使用 `AskUserQuestion` 提示用户："请在浏览器中扫码登录小红书，登录完成后告诉我。"
3. 用户确认后，刷新页面并 snapshot 验证登录状态

---

## Step 1: 内容准备与自检

### 1.1 准备内容

根据用户提供的内容，整理出：
- **标题**：不超过 20 个字
- **正文**：段落之间用双换行分隔，语言自然，简体中文
- **图片**：生成对应的文字片段（30字以内），用于生成“文字配图”的图片；如果指定了图片，则使用本地路径或 URL（图文笔记必须至少 1 张图片）对应的图片文件或视频文件；
- **话题标签**：最多 5 个，但需要逐个配置，并不随正文一起贴入；


### 1.2 发帖前自检
对照1.1 准备内容检查：
- [ ] 标题生成了吗
- [ ] 正文生成了吗
- [ ] 生成文字配图的文字片段生成了吗？
- [ ] 话题标签生成了吗？
- [ ] 用户有指定图片或视频吗？

对照 `memory/小红书限流规则备忘.md` 检查：
- [ ] 标题不超过 20 个字
- [ ] 无敏感词（绝对化用语、医疗功效词、金融承诺词等）
- [ ] 无微信号/手机号/二维码/外链等引流信息
- [ ] 投资内容已加风险提示
- [ ] 内容为原创
- [ ] 如使用 AI 辅助创作，已标明
- [ ] 话题标签格式：`# 标签名`（`#` 与文字之间有空格）

### 1.3 用户确认内容

通过 `AskUserQuestion` 向用户展示即将发布的标题、正文和图片，获得明确确认后才继续。

## Step 2: 发布图文笔记

### 2.1 内容规格与建议

- **标题**：≤ 20 字；前 10 字要吸睛，可含数字、关键词，避免「最」「第一」等绝对化用语。
- **正文**：建议 500–800 字，段落短（每段 1–3 行）、段间空一行；可适量用 emoji（约 10–15 个）和 ①②③ 分点；开头 3 行要抓人，结尾可引导互动。
- **图片**：至少 1 张，最多 9 张；**首张即封面**，建议按「封面 → 内容图 → 自我介绍/结尾图」顺序上传。
- **封面建议**：竖版 3:4 比例，宽度 1080px 为佳；文字简洁、突出核心词，清晰度高，让用户一秒看懂主题。

### 2.2 导航到发布页

使用 browser 工具打开 `https://creator.xiaohongshu.com/publish/publish?from=tab_switch&target=image`。

若页面跳转到登录页，返回 Step 0.3 处理登录。

### 2.3 等待页面加载

snapshot 确认页面已完全加载，能看到图片上传区域（或文字配图入口）。

### 2.4 文字配图

若用户指定使用图片，改用下方「上传图片」添加图片后再继续。 

1. 在页面中找到文字配图区域（通常是一个虚线框，提示"文字配图"）；
2. 点击文字配图区域；
3. 将用于文字配图的文字贴入；
4. 点击生成图片，等待片刻；
5. snapshot 确认图片成功生成，页面有“选择一个喜欢的卡片”区域，该区域可以选择相关的卡片；
6. 选择卡片完成后，点击“下一步”按键区域； 


### 2.5 上传图片（指定使用图片时使用）

1. 在页面中找到图片上传区域（通常是一个虚线框，提示"上传图片"或含 `input[type="file"]` 元素）；
2. 点击上传区域，或直接操作文件上传控件
3. 选择本地图片文件上传（单张不超过 20MB，支持 JPG/PNG 格式）
4. 如果图片是 URL，先下载到本地临时目录再上传
5. snapshot 确认图片已成功上传（应能看到图片缩略图预览）
6. 系统默认首张图为封面，如需更换可点击封面缩略图的"替换"按钮

**图片要求汇总：** 格式 JPG/PNG，单张 ≤ 20MB，数量 1–9 张。




### 2.6 填写标题

1. 找到标题输入框（placeholder 多为「填写标题」，选择器：`input[placeholder*="填写标题"]` 或 `input.d-text`）
2. 输入标题（≤ 20 字）

### 2.7 填写正文（不含话题标签）

1. 找到正文编辑区（TipTap/ProseMirror，选择器：`div.tiptap.ProseMirror` 或 `div.ProseMirror[contenteditable="true"]`）
2. 输入或粘贴**纯正文内容**，可插入表情；段落间用换行分隔，避免大段堆砌
3. **注意：正文中不要包含 `# 话题` 标签**，话题标签必须在正文输入完成后，通过下面 2.8 的流程逐个添加
4. 可选：在发布页添加地点信息（若内容与地点相关）

### 2.8 添加话题标签（必须逐个添加，不能批量粘贴）

**重要：话题标签不能跟随正文一起粘贴，必须在正文输入完成后，在正文编辑区末尾逐个操作。**

操作流程（每个话题重复一次）：

1. 将光标定位到正文编辑区的**末尾**
2. 输入 `#` ，再输入话题关键词（如「旅行」）
3. 等待页面弹出**话题选择浮层**（类似搜索建议的下拉框）
4. 在弹出的话题列表中，**点击选择**匹配的话题（不要只打字不点选，必须点击才能生效）
5. 选中后，话题会变成带样式的标签块，表示添加成功
6. **重复步骤 1-5** 添加下一个话题，直到所有话题添加完毕

注意事项：
- 最多 5 个话题，建议兼顾宽泛词、精准词和热点词
- 如果弹出列表中没有完全匹配的话题，选最接近的一个
- 如果输入后没有弹出话题选择框，snapshot 检查页面状态，可能需要等待加载
- **绝不要**把 `# 话题1 # 话题2 # 话题3` 一次性写在正文里或一次性粘贴——这样不会被小红书识别为话题标签

### 2.9 预览确认

1. snapshot 截取当前页面（含封面、标题、正文、话题）
2. 通过 `AskUserQuestion` 请用户确认预览无误
3. **绝不自动发布** — 必须获得用户明确同意

### 2.10 存草稿与结果确认

用户确认后，找到并点击「存草稿」按钮（通常在「发布」按钮旁边，页面右上或右下区域）。snapshot 检查是否出现「已保存为草稿」等提示，向用户报告结果。提醒用户可以到创作者中心的草稿箱检查内容，确认无误后手动发布。

## Step 3: 发布视频笔记

### 3.1 内容规格与建议

- **时长**：普通用户一般 ≤ 5 分钟，专业号/认证创作者可更长（平台技术上限约 30 分钟）；算法更偏好 1–3 分钟高信息密度，**前 5 秒**决定是否被划走，完播率是关键指标。
- **比例**：竖屏 9:16 推荐（有流量倾斜），横屏 16:9 可用。
- **标题**：≤ 20 字，前 10 字要抓人；可套用「痛点 + 解决方案 + 情绪词」，避免绝对化用语。
- **封面**：清晰、有文字提示（如「干货总结」），避免纯图无信息；选最有冲击力或最能代表内容的一帧，或单独制作文字封面上传。
- **描述与话题**：正文可补充干货、@ 官方账号；主标签 1 个 + 辅标签 2–3 个，避免堆砌；可加 1–2 个热门话题 + 2–3 个精准话题。
- **合规**：不引导「点赞/评论/私信」、不留二维码或外链；敏感词用错别字/拼音等方式规避，避免过度美颜导致误判非原创。

### 3.2 导航到发布页

使用 browser 工具打开 `https://creator.xiaohongshu.com/publish/publish?from=tab_switch&target=video`。

若跳转到登录页，返回 Step 0.3。

### 3.3 选择视频发布并上传

1. 在发布类型面板中选择「发布视频」，进入视频上传界面
2. 点击「选择文件」或将视频拖入上传区域，选择本地视频文件

**视频要求：**
- 格式：MP4（推荐）/ MOV
- 分辨率：≥ 1080p（1920×1080）为佳，至少 720p
- 编码：H.264，帧率 25–30fps
- 时长：普通用户 ≤ 5 分钟，专业号可更长（技术上限约 30 分钟）
- 大小：单文件一般 ≤ 2GB（部分场景支持更大，建议 500MB 内更稳）

3. 等待上传完成（观察进度条），上传完成后系统自动转码，需等待转码完成再继续

### 3.4 设置封面

转码完成后：
- 在预览区拖动时间轴截取一帧作为封面，或
- 点击「重新选择封面」/「生成封面」上传自定义封面图
- 系统会从视频中提取候选帧，可选其一；封面需清晰、有信息量（建议带简短文字）

### 3.5 填写标题和描述

1. 在标题栏输入视频标题（≤ 20 字）
2. 在描述框填写说明文字，可补充干货、引导收藏，勿出现引流或违规话术

### 3.6 添加话题标签

同图文笔记 2.8 的流程：在正文/描述编辑区末尾，**逐个**输入 `# ` + 关键词 → 等待话题选择浮层弹出 → 点击选中 → 重复添加下一个。不要批量粘贴，最多 5 个。

### 3.7 预览确认并存草稿

1. snapshot 截取当前页面（含封面、标题、描述、话题）
2. 通过 `AskUserQuestion` 请用户确认
3. 用户确认后，点击「存草稿」按钮（在「发布」按钮旁边）
4. snapshot 确认「已保存为草稿」或成功态提示，向用户报告结果。提醒用户可到草稿箱检查后手动发布。

## Step 4: 发布长文笔记

长文功能需账号已开通（平台内测/定向邀请）。发布后读者仍以右划图片方式阅读，与图文浏览逻辑一致。

### 4.1 内容规格

- **标题**：≤ 20 字（与图文/视频一致）
- **正文**：支持千字以上（通常上限约 6000 字），可插入图片、表情
- **描述/摘要**：发布页可选填，建议 ≤ 1000 字；不填时系统可能取正文前约 800 字

### 4.2 导航到长文发布页

使用 browser 工具打开：

`https://creator.xiaohongshu.com/publish/publish?from=tab_switch&target=article`

若未开通长文权限，页面可能提示申请或不可用，需告知用户。若跳转到登录页，回到 Step 0.3。

### 4.3 填写标题与正文

1. 找到标题输入框，输入标题（≤ 20 字）
2. 在正文编辑区（富文本）输入或粘贴正文，段落间用换行分隔
3. 可选：在正文中插入图片、表情
4. snapshot 确认内容已正确填入

### 4.4 一键排版

1. 点击页面上的「一键排版」按钮
2. 系统将长文自动切分为多页/多图，并进入排版预览
3. 等待排版完成（可能数秒至十数秒），snapshot 确认进入模板选择界面

### 4.5 选择模板与自定义

1. 后台通常提供约 6 种排版模板（如简约基础、清晰明朗、文艺清新等），选择一种
2. 可调整：主题配色、封面配图、标题与摘要的展示
3. 若有「实时预览」，可查看最终效果后再下一步
4. 点击「下一步」进入发布前描述/设置页

### 4.6 填写描述（摘要）

1. 在发布前页找到「描述」或「摘要」输入框
2. 填写用于展示的简短描述（建议 ≤ 1000 字）；可不填，由系统截取正文前段
3. 确认可见范围等设置（若页面提供）

### 4.7 预览确认并存草稿

1. snapshot 截取当前页面（含描述与预览效果）
2. 通过 `AskUserQuestion` 请用户确认标题、正文分段效果、描述无误
3. **绝不自动发布** — 获得用户明确同意后，点击「存草稿」按钮（在「发布」按钮旁边）
4. 等待保存完成，snapshot 确认「已保存为草稿」或成功态提示，向用户报告结果

### 4.8 长文发布注意

- 全程约 50 秒左右（含排版与多步点击），操作失败可重试一次
- 发布前同样需对照 `memory/小红书限流规则备忘.md` 做敏感词与限流自检
- 长文适合知识分享、攻略、复盘、深度观点等，注意段落清晰、重点突出，便于一键排版后阅读

## DOM 选择器参考

> 小红书前端可能随时更新，以下选择器基于 2026 年初的页面结构。如果操作失败，需要通过 snapshot 重新观察页面元素。

| 元素 | 选择器 | 说明 |
|------|--------|------|
| 图片上传 | `input.upload-input` 或 `input[type="file"]` | 隐藏的文件输入；图文 1–9 张 |
| 文字配图 | 虚线框/「文字配图」入口 | 无图时生成封面/内容图，选卡片后「下一步」返回 |
| 标题输入（图文） | `input[placeholder*="填写标题"]` 或 `input.d-text` | 标题 ≤20 字 |
| 正文编辑 | `div.tiptap.ProseMirror` | TipTap 富文本编辑器 |
| 存草稿按钮 | 文本匹配"存草稿" | 在发布按钮旁边，遍历按钮文本定位 |
| 登录检测 | URL 包含 "login" | 重定向到登录页 |
| 长文标题/正文 | 长文页内标题框、富文本区域 | 以 snapshot 为准，结构可能与图文不同 |
| 一键排版 | 按钮文本「一键排版」 | 长文页内，点击后进入模板选择 |
| 长文描述 | 发布前页描述/摘要输入框 | 可选，≤1000 字 |

## 重要提示

1. **绝不自动发布** — 最后一步只存草稿，不直接发布；必须获得用户确认后才点击存草稿
2. **图文笔记必须有图片** — 至少 1 张；**长文笔记** 无图片数量要求，正文可千字以上，标题仍 ≤ 20 字
3. **始终使用 openclaw 浏览器** — `profile: "bot11"`，不要用 `profile: "chrome"`
4. **发帖前必须自检** — 对照限流规则备忘录检查敏感词（图文/视频/长文均适用）
5. **操作失败时** — snapshot 观察当前页面状态，根据实际 DOM 结构调整操作
6. **距上一篇发布间隔** — 建议 ≥ 3 小时
7. **每日发帖量** — 建议 1-2 篇，不超过 3 篇
8. **长文权限** — 长文为内测功能，若入口打开后无写长文界面，需账号先申请/开通

## 故障排除

| 问题 | 处理方式 |
|------|----------|
| 页面跳转到登录页 | 回到 Step 0 重新登录 |
| 图片上传失败 | 检查格式 JPG/PNG、单张 ≤20MB，重试；URL 需先下载到本地再传 |
| 文字配图生成失败 | 检查网络与文案长度，重试；仍失败可改用本地上传图片 |
| 视频上传/转码失败 | 检查格式 MP4/MOV、分辨率≥720p、大小与时长符合要求，转码需等待完成后再操作 |
| 找不到上传/发布按钮 | snapshot 观察当前页面，根据实际布局定位元素 |
| 发布后提示违规 | 对照限流规则备忘录检查内容，修改后重新发布 |
| 页面加载缓慢 | 等待几秒后再 snapshot，确认加载完成 |
| 长文入口无写长文界面 | 可能未开通长文权限，提示用户到创作者中心查看/申请 |
| 一键排版后卡住或失败 | 等待 10–20 秒后重试；仍失败则 snapshot 看当前步骤，按页面实际按钮继续 |
