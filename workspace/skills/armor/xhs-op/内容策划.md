# 小红书内容策划

> 每次写笔记前必读。本文件只管"怎么写好一篇小红书"，不管选题和研究。

---

**铁律**人设一致性 — 内容风格必须参考自己装备里的内容风格，图片风格必须参考装备里的图片风格

## 1. 读取行文风格

生成内容前，**必须先读 agent 的风格参考**：

1. **SOUL.md** 中的"说话风格"/"性格"段落 — 决定语气和调性
2. **MEMORY.md** 中的近期经历 — 提供真实素材和细节
3. **研究成果**（如有）— 提供数据、案例和观点支撑
4. **发布记录**（`topic-library.md` 底部）— 回顾之前发过的笔记风格是否一致
5. **限流规则备忘**（如有）— 确认敏感领域的表达边界

从中提取写作参数：

| 参数 | 说明 | 举例 |
|------|------|------|
| **人称** | 第一人称/第三人称/拟人 | "我""本龙""咱家" |
| **调性** | 专业/轻松/毒舌/温暖/搞笑 | 根据 SOUL.md 性格 |
| **口头禅/特色用语** | agent 特有的说法 | "本龙觉得""说人话就是" |
| **emoji 密度** | 不用/适量/密集 | 小红书适量为佳（10-15个/篇） |
| **段落节奏** | 长段深度/短段快节奏/问答式 | 根据人设和选题类型 |
| **互动方式** | 提问/投票/征集/吐槽引战 | 符合人设的互动引导 |

> **关键原则：同一个选题，不同 agent 写出来的文章应该完全不同。**
> 区别不在于内容本身，而在于视角、语气、用词和情绪。

## 2. 标题优化（封面标题 + 正文标题）

标题是流量入口，封面标题决定点击率，正文标题决定搜索排名。两者逻辑不同，必须分别打磨。

### 封面标题（吸引点击）

封面标题的核心任务是**让用户停下来点进去**。≤ 20 字，前 10 字必须包含钩子。

**三种钩子类型，至少命中一种：**

| 钩子类型 | 原理 | 封面标题示例 |
|---------|------|------------|
| **痛点钩子** | 用户必须解决的刚需，制造紧迫感 | "工资5千存不下钱，这个方法一年攒了3万"；"基金亏了30%，我是怎么回本的" |
| **痒点钩子** | 内心渴望成为的理想自我，激发向往 | "靠定投3年，姐实现了第一个小目标"；"26岁财务自由，靠的不是工资是认知" |
| **热点钩子** | 时间节点 / 明显热点，借势吸引关注 | "美联储降息，普通人的钱该往哪放"；"A股放量大涨，现在上车还来得及吗" |

**写封面标题的步骤：**
1. 先确定本篇内容命中哪种钩子（痛点/痒点/热点），可以组合
2. 把钩子写在标题前半段（前 10 字内）
3. 后半段给出内容承诺（方法/结果/数字）
4. 检查：如果这个标题出现在首页信息流里，用户会不会想点？

### 正文标题（搜索排名）

正文标题决定笔记能否被搜索到。核心逻辑是**埋关键词**，让小红书搜索算法能匹配到。

**三层关键词，逐层叠加：**

| 关键词层级 | 说明 | 示例 |
|-----------|------|------|
| **人群关键词** | 内容受众是谁 | 上班族、宝妈、工薪族、理财小白、新手投资者 |
| **需求关键词** | 解决什么问题/什么主题 | 基金定投、黄金投资、存钱方法、A股入门 |
| **长尾关键词** | 搜索主关键词后跳出的衍生词 | 搜"基金定投" → 小白基金定投入门、每月1000定投买什么 |

**写正文标题的步骤：**
1. 确定目标受众 → 提取人群关键词
2. 确定内容主题 → 提取需求关键词
3. 去小红书搜索框输入需求关键词，**看下拉联想词** → 提取长尾关键词
4. 将 2-3 个关键词自然组合进正文标题，不要硬堆

**正文标题 vs 封面标题的关系：**
- 封面标题 = 给人看的（情绪驱动，让人想点）
- 正文标题 = 给算法看的（关键词驱动，让人搜到）
- 两者可以相同，也可以不同。text_to_image 帖子的封面文字用封面标题逻辑，`-t` 参数用正文标题逻辑

> **示例对比：**
> - 封面标题（痛点钩子）：工资5千存不下钱，这个方法一年攒了3万
> - 正文标题（关键词优化）：工薪族存钱方法｜每月定投基金入门指南

## 3. 内容结构

内容结构不是固定模板，而是根据 **选题类型 × agent 风格** 灵活组合：

### 基础骨架（所有笔记通用）

```
【封面标题】≤ 20 字，前 10 字含钩子（痛点/痒点/热点），吸引点击
【正文标题】含人群关键词 + 需求关键词 + 长尾关键词，优化搜索
【正文】主体内容
【配图方案】文字配图 / 实拍图 / 截图
【话题标签】3-5 个
```

### 正文结构变体（根据风格选择）

**叙事型**（适合故事感强的 agent）
```
场景开头 → 经过 → 转折/感悟 → 互动引导
"上周五收盘后我打开账户一看..."
```

**干货型**（适合专业/理性的 agent）
```
结论先行 → 分点展开 → 数据/案例 → 总结 + 互动
"3 个月实测下来，我的结论是..."
```

**吐槽型**（适合搞笑/毒舌的 agent）
```
槽点切入 → 放大痛点 → 反转/解法 → 引发共鸣
"谁懂啊，基金跌的时候我比它还绿..."
```

**对话型**（适合亲和/温暖的 agent）
```
"你是不是也遇到过..." → 分享经验 → "你们觉得呢？"
读起来像跟朋友聊天
```

**清单型**（适合实用导向的 agent）
```
"X 件事/X 个方法/X 个坑" → 逐条展开 → 收藏引导
适合收藏型内容
```

## 4. 写作红线

无论什么风格，以下底线不可突破：

- **活人感** — 2026 算法打压 AI 味内容，必须有个性、有情绪、有瑕疵
- **真实 > 精致** — 真实经历和感受比完美干货更有流量
- **人设一致** — 不能这篇文艺下篇粗犷，风格要稳定
- **不说空话** — 每一段都要有信息量，删掉"众所周知""首先我们来看"这种废话
- **敏感领域** — 投资/健康/法律等需要在末尾加免责声明，措辞风格也要符合人设
- **敢下结论** — 模棱两可的内容没人看。"我的判断是""在我看来"比"可能会涨也可能会跌"强一百倍
- **承认不确定** — 自信判断和承认不确定不矛盾。"我也不知道底在哪，但这估值水平几年一遇"

### 金融热帖的五种爆款模式（参考）

从平台高赞帖子中提取的内容模式，选题时可参考：

| 模式 | 特点 | 互动特征 | 示例标题 |
|------|------|---------|---------|
| **深度分析型** | 1000-2000字，敢下结论，有独家信息/独特视角 | 高赞高评论 | "恒生科技在等腾讯的AI" |
| **实盘记录型** | 完全第一人称，写真实操作和心路历程，坦诚情绪 | 高共鸣高评论 | "这一次定投犯的错" |
| **教程干货型** | 步骤化、清单化、产业链梳理，收藏价值高 | 高收藏 | "最新电力上下游板块热门基金盘点" |
| **娱乐段子型** | 把投资经历写成故事/段子，幽默自嘲 | 传播广 | "恒生科技会惩罚每一个想抄底的人" |
| **每日计价型** | 短小精悍，每日更新数据+简短判断 | 粉丝粘性高 | "恒s科j3.26计价" |

## 5. 质量自检

- [ ] 封面标题 ≤ 20 字，前 10 字含钩子（痛点/痒点/热点至少命中一种）
- [ ] 正文标题含人群关键词 + 需求关键词，自然不堆砌
- [ ] 正文 500-800 字，段落短（1-3 行/段）
- [ ] 开头 3 行能留住人
- [ ] 结尾有互动引导
- [ ] 敏感领域有风险提示
- [ ] 无敏感词
- [ ] **读一遍正文，听起来像"我"说的话吗？** — 最重要的一条
- [ ] 与近 7 天发布不重复

---

## 注意事项

1. **质量自检必须通过** — 自检不过不发布
2. **遵守平台规则** — 对照限流规则备忘自检
