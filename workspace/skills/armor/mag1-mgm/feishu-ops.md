# 飞书群管理

> 飞书群是研究部获取系统状态的主窗口。印务局负责基础设施告警，不在群里发日常发布结果。

## 飞书群信息

> 群 ID 和用户 open_id 请查 📇 **通讯小本本**（`workspace/skills/contact-book/SKILL.md`）

## 发送群消息

```
# 管理群（疯狂小龙虾）— 查通讯小本本获取 chat_id
message(action="send", channel="feishu", target="{管理群chat_id}", message="告警内容")
```

## 什么该发群 vs 什么不该发群

### ✅ 该发群（基础设施异常）

| 场景 | 消息示例 |
|------|----------|
| MCP 批量离线 | `🚨 MCP 异常 \| bot1/bot3/bot5 的 MCP 服务无响应，已尝试自动重启` |
| Cookie 批量失效 | `🚨 登录异常 \| bot2/bot6/bot7 需要重新扫码登录` |
| 合规服务宕机 | `⚠️ 合规服务离线 \| compliance-mcp 无响应，发布队列暂停` |
| 发布队列长时间积压 | `⚠️ 队列积压 \| 当前 pending 队列有 5+ 任务超过 30 分钟未处理` |

### ❌ 不该发群

| 场景 | 正确做法 |
|------|----------|
| 单个帖子发布成功/失败 | `reply_message` 回复提交者 |
| 单个 bot 登录过期 | `reply_message` 通知提交者，不发群 |
| heartbeat 一切正常 | 不通知 |
| 日常队列处理进度 | 不通知 |

## 消息格式规范

### 告警消息

```
🚨 {严重级别} | {问题摘要}

影响范围：{哪些 bot/服务受影响}
当前状态：{已做了什么处理}
需要操作：{研究部需要做什么，如"请扫码登录 bot6"}
```

### 状态恢复

```
✅ 已恢复 | {之前的问题}

恢复时间：{timestamp}
处理方式：{如何恢复的}
```

## 告警级别

| 级别 | 图标 | 触发条件 | 是否发群 |
|------|------|----------|----------|
| P0 紧急 | 🚨 | 3+ 个 MCP 同时离线 / 全线发布失败 | ✅ 立即发群 |
| P1 重要 | ⚠️ | 合规服务离线 / 队列积压 | ✅ 发群 |
| P2 常规 | 📋 | 单个 bot 登录过期 / 单次发布失败 | ❌ 只 reply_message |
| P3 信息 | ℹ️ | heartbeat 正常 / 队列为空 | ❌ 不通知 |

## 避免群消息轰炸

- **同一问题 30 分钟内不重复告警** — 记录到 `memory/YYYY-MM-DD.md`，已告警的不重发
- **批量问题合并** — 多个 bot 同时出问题时，一条消息汇总，不逐个发
- **恢复通知可选** — 小问题自动恢复后不必发群通知
