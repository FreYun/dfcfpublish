# 知识星球 Skill

所有 MCP 工具都已在 `tools/list` 里自描述（参数、用途 bot 可自行查询）。此文档只记录 **bot 无法推断的约定和规则**。

- 土狗和他的朋友们 — pay

## 节流与重试（已内置，外层不要再加 sleep）

MCP 进程内置 0.8-2.5s 随机间隔 + 每分钟 20 次上限 + 1059 签名失败自动重签。**不要并发调用**（串行排队，并发只会堵着）。

---

## 搜索能力边界

`search_topics` 的 `group_id` 是**客户端过滤**（zsxq 服务端忽略该参数）。过滤发生在翻页之后——目标星球命中稀疏时 `max_pages` 要开到 3-5 才能凑齐。

---

## 附件下载与阅读

- 共享目录：`~/.openclaw/shared/zsxq-files/{group_id}/{file_id}_{文件名}`
- 幂等：同 file_id + size 一致 → 自动跳过下载，返回缓存路径
- **阅读 PDF：** bot 用 `Read(file_path=..., pages='1-10')` 直接读。**超过 10 页必须带 pages 参数**否则会失败

---

## Cookie 失效

MCP 返回 `{error: "cookie_expired", ...}` 时，**只能 bot11 重新登录**（登录态存在 bot11 的 Chrome profile）：

```
1. openclaw browser navigate https://wx.zsxq.com --browser-profile bot11
2. 手机扫码登录
3. cd ~/MCP/zsxq-mcp && python3 refresh_cookie.py
4. 调 MCP 的 reload_cookie 工具热加载
```

其他 bot 遇到此错误时，只需在回复里提示研究部按上述步骤处理，**不要尝试自行修复**。

## 定时巡检输出格式（scheduled 场景必须遵守）

巡检完成后按此格式汇总（**不是逐条列举原始话题**）：

```
📊 {星球名} {日期范围} 话题汇总

📍 滚动范围
从 {最新话题时间} 到 {最早话题时间}，共 {N} 条话题。

📈 话题分类统计

| 方向 | 话题数 | 核心内容 |
|------|--------|----------|
| {方向1} | N 条 | {关键词1}、{关键词2}、{关键词3} |

🔥 核心主线聚焦
1️⃣ {主线名}（{定位描述}，{N} 条）
逻辑链：{完整传导链，用 → 连接}
核心标的：{相关股票}

📊 关键数据速览（有数据才列）
- 布油 103.9 美元，+42%
- DRAM 合约价 Q1 涨幅 90%-95%

📎 重点研报/附件（有附件才列）
- {文件名}（本地：/home/rooot/.openclaw/shared/zsxq-files/.../xxx.pdf）
```

### 必含板块自检

**发送前自检，缺一个就补上再发：**

| # | 板块 | 标识 | 可否省略 |
|---|------|------|---------|
| 1 | 滚动范围 | 📍 | 不可 |
| 2 | **话题分类统计** | 📈 | **不可** |
| 3 | 核心主线聚焦 | 🔥 | 不可 |
| 4 | 关键数据速览 | 📊 | 无数据可省 |
| 5 | 重点研报/附件 | 📎 | 无附件可省 |

### 输出规则

1. **分类归纳**，不是逐条列举 — 按投资/行业方向归类统计，每方向的话题数+核心关键词
2. **主线有深度** — 每条主线含：逻辑链/传导链（→ 连接）、关键数据、核心标的（含目标价则注明）
3. **主线按重要性排序** — 话题数最多、市场影响最大的排第一
4. **研报附件列本地路径** — 方便追查或用 Read 直接读
5. **不同星球分开输出** — 每个星球一个独立汇总块
6. **关键数据单独汇总** — 价格、涨跌幅、成交额、融资额、政策目标等提到「📊 关键数据速览」
7. **风格参考** — 卖方晨会纪要/投研早报，简洁、有判断、有标的、有数据
