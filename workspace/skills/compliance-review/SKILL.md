# 小红书合规审核 — compliance-mcp

## 概述

独立的合规审核服务，内置 Claude Sonnet 作为审核引擎，自动检查小红书帖子和评论是否符合平台规则。

**发帖前必须调用此服务进行合规审核，审核通过后才能发布。**

## 可用工具

### `review_content` — 审核帖子内容

发布帖子前调用，检查标题、正文、话题标签是否合规。

```bash
npx mcporter call "compliance-mcp.review_content(title: '标题', content: '正文内容', tags: '话题1,话题2')"
```

**参数：**
| 参数 | 必填 | 说明 |
|------|------|------|
| `title` | 是 | 帖子标题 |
| `content` | 是 | 帖子正文 |
| `tags` | 否 | 话题标签，逗号分隔 |
| `content_type` | 否 | 内容类型：图文/视频/长文（默认图文）|
| `bot_identity` | 否 | Bot 人设简述，用于检查人设一致性 |

### `review_comment` — 审核评论/回复

发表评论前调用，检查评论内容是否合规。

```bash
npx mcporter call "compliance-mcp.review_comment(comment: '评论内容')"
```

**参数：**
| 参数 | 必填 | 说明 |
|------|------|------|
| `comment` | 是 | 评论/回复内容 |
| `context` | 否 | 原帖内容，提供上下文以便更准确审核 |

## 返回格式

```json
{
  "passed": true,
  "score": 92,
  "violations": [],
  "summary": "内容合规，可以发布"
}
```

- `passed`: 是否通过审核
- `score`: 合规评分（0-100）
- `violations`: 违规项列表，每项包含 `level`（critical/warning/info）、`category`、`detail`、`suggestion`
- `summary`: 一句话总结

## 使用流程

1. 准备好帖子内容（标题、正文、话题）
2. 调用 `review_content` 获取审核结果
3. 如果 `passed: false`，根据 `violations` 中的 `suggestion` 修改内容
4. 修改后重新审核，直到 `passed: true`
5. 审核通过后调用 `xiaohongshu-mcp.publish_content` 发布

## 合规红线分级标准

### critical（必须修改，否则拒绝发布）
- 明确收益率承诺：「保证年化 8%」「稳赚不赔」「100% 获利」
- 无资质推荐具体股票/基金/合约买卖操作
- 夸大收益截图（未标注「模拟盘」或「历史不代表未来」）
- 涉及非法集资、传销、地下钱庄
- 无免责声明的投资建议帖（金融类内容必须含免责声明）
- 仿冒监管机构、知名机构背书

### warning（建议修改，可能触发限流或降权）
- 模糊收益暗示：「轻松月入过万」「躺赚」「财富自由」
- 未标注内容为个人观点/非专业建议
- 话题标签与内容不符（堆砌热门标签）
- 含竞品名称（可能被平台过滤）
- 引导站外成交或私聊「详细说」

### info（提示性问题，不影响发布）
- 标题党但未造假
- 格式建议（段落过长、无分段）
- 话题标签数量超过 10 个

## 金融内容合规要点清单

发布金融类帖子前，逐项核查：

1. **免责声明**：正文末尾必须包含「以上内容仅供参考，不构成投资建议，投资有风险，入市需谨慎」或等效表述
2. **收益率表述**：历史收益必须标注时间区间，禁止正向预测未来收益
3. **截图规范**：盈利截图须标注「历史业绩，不代表未来」；若为模拟盘须注明
4. **资质说明**：推荐具体标的时须注明「本人非持牌投资顾问」
5. **数据来源**：引用数据/排名须注明来源机构和日期
6. **话题标签**：避免使用「稳赚」「躺赚」「无风险」等标签

## 审核不通过时的修改策略

### 通用流程
1. 优先修复所有 critical 违规，再处理 warning
2. 按每条 `suggestion` 逐项修改，不要只删除文字而不补充合规表述
3. 修改后重新调用 `review_content`，直到 `passed: true`

### 常见违规与修改示例

| 原始表述 | 违规类型 | 合规改写 |
|---------|---------|---------|
| 「买入 XX 基金，年化稳定 12%」 | critical：收益承诺 | 「XX 基金近三年年化约 12%（数据来源：基金官网，历史业绩不代表未来）」 |
| 「跟着我操作，轻松月入 5 万」 | critical：收益承诺 | 删除，替换为策略逻辑说明 |
| 「我重仓了 XX 股，快跟上」 | critical：无资质荐股 | 「分享我的持仓思路，仅代表个人观点，非投资建议」 |
| 「躺赚攻略」（标题） | warning：模糊收益暗示 | 「2024 年我的基金配置复盘」 |
| 正文无免责声明 | critical（金融类） | 末尾添加标准免责声明 |

## 批量审核

需要批量审核多条内容时，按顺序逐条调用 `review_content`，汇总所有结果后统一处理：
- 所有条目 `passed: true` → 可批量发布
- 任意条目 `passed: false` → 修复该条目后重新审核，不要跳过直接发布其他条目

批量场景建议先审核，再统一修改，最后统一发布，避免已发布与未发布内容口径不一致。

## 注意事项

- 审核结果仅供参考，最终发布责任由 bot 承担
- 金融类内容执行最严格审核标准，critical 违规必须全部清零才能发布
- 不要跳过审核直接发布
