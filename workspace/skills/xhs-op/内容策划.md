# 小红书内容策划 Skill

通用 skill，适用于任何需要运营小红书的 agent。选题基于 **agent 自身人设 + 日常积累**，不硬编码内容方向。

---

## 触发条件

- **每日定时推荐 3 次**（如 9:00、13:00、18:00），通过 heartbeat 触发
- 用户说"推荐选题""今天发什么""给我推几个选题"等
- 用户说"发一篇笔记"但没指定主题

### 智能跳过规则

每次定时触发前，先检查当天是否已经成功发布过笔记（查 `topic-library.md` 发布记录 或 当天日记）：
- **当天尚未发帖** → 正常推荐选题
- **当天已发帖** → 跳过本次推荐，不打扰用户
- **用户主动要求推荐**（"今天发什么"等） → 无论是否已发帖，都正常推荐

> 核心逻辑：定时推荐的目的是提醒发帖，已经发了就不用再提醒。但用户主动问的时候永远响应。

---

## Step 1: 确定"我是谁"——从人设出发

**每次策划选题前，先读自己的身份文件：**

1. **SOUL.md** — 我的核心性格、说话风格、价值观
2. **IDENTITY.md** — 我的名字、擅长领域、人设定位
3. **MEMORY.md** — 我的长期记忆、近期经历、关注的事

从中提取：
- **内容方向**：我擅长什么、关注什么领域（如投资、旅行、美食、技术...）
- **人设调性**：我说话的风格是什么（专业、搞笑、温暖、毒舌...）
- **真实经历**：最近发生了什么可以写的事（做了什么决策、去了哪里、学到了什么）

> 选题的核心逻辑：**我是什么样的"人"，就写什么样的内容。**
> 不是找热门话题硬蹭，而是从自身经历和视角出发，与热点产生自然交集。

---

## Step 2: 收集选题素材

从以下素材源并行收集，**不是每个都要用，根据 agent 的内容方向选择相关的源**：

### 2.1 选题素材库（优先）

查阅 `topic-library.md`，这是 agent 平时积累的选题池。优先从中挑选：
- 标记为 `💡 灵感` 的新鲜素材（来自日常浏览积累）
- 尚未发过的常备选题
- 与当前热点能碰撞的旧选题

### 2.2 小红书热点 & 自身爆款复盘

```bash
# 刷首页看平台在推什么
npx mcporter call xiaohongshu-mcp.list_feeds

# 搜索与自身领域相关的热门内容（关键词根据人设动态决定）
npx mcporter call "xiaohongshu-mcp.search_feeds(keyword: '<根据人设填>', filters: {sort_by: '最多点赞', publish_time: '一周内'})"

# 查看自己发过的笔记，找互动数据好的方向
npx mcporter call xiaohongshu-mcp.get_self_notes
```

**自身爆款复盘**：查看自己历史笔记的互动数据（点赞、收藏、评论），找出效果好的内容方向：
- 互动高的方向 → 可以继续深挖、出系列、换角度再写
- 某篇评论区讨论热烈 → 评论区的问题本身就是新选题

### 2.3 账号数据复盘

```bash
# 看粉丝在评论什么、关心什么
npx mcporter call xiaohongshu-mcp.get_notification_comments
```

分析：哪类内容互动多？评论区在问什么？

### 2.4 外部信息源

根据 agent 的领域，从日常使用的信息源获取素材：
- 浏览器（雪球、金十、知乎、微博等）
- Web 搜索（时事、行业动态）
- agent 自身的数据和工具输出（如持仓变动、跟踪任务触发等）

### 2.5 时间维度

- **季节/节气/节日**：当前时令有什么自然话题
- **星期几**：工作日 vs 周末，内容调性不同
- **近期日记**：`memory/` 目录下最近 7 天的日记，避免重复 & 寻找可写的经历

## Step 3: 生成 3 个选题推荐

基于 Step 1 的人设 + Step 2 的素材，生成 **3 个差异化选题**：

```
📋 今日选题推荐（YYYY-MM-DD）

【选题 A】<内容方向>
标题草案：xxxxxxxxxx
类型：图文 / 文字配图
切入角度：一句话说为什么现在写这个
来源：💡素材库积累 / 🔥热点 / 📖自身经历 / 💬粉丝提问
适合标签：#xxx #yyy #zzz

【选题 B】<内容方向>
...

【选题 C】<内容方向>
...

你想发哪个？可以直接选 A/B/C，也可以告诉我想改改。
```

### 选题原则

1. **从人设出发**：选题必须是"我"会写的内容，不是硬蹭不相关的热点
2. **3 个选题要有差异**：不同方向、不同内容形式、不同切入角度
3. **至少 1 个来自素材库积累**：用掉平时攒下的灵感，别让素材库白积累
4. **标题先行**：标题 ≤ 20 字，前 10 字要吸引人
5. **优先能引发讨论的话题**：CES 公式中评论权重 x4
6. **避免重复**：检查近 7 天日记和素材库发布记录
7. **遵守安全红线**：对照限流规则自检

---

## Step 4: 用户选定后生成内容

### 4.1 读取行文风格

生成内容前，**必须先读 agent 的风格参考**：

1. **SOUL.md** 中的"说话风格"/"性格"段落 — 决定语气和调性
2. **MEMORY.md** 中的近期经历 — 提供真实素材和细节
3. **发布记录**（`topic-library.md` 底部）— 回顾之前发过的笔记风格是否一致
4. **限流规则备忘**（如有）— 确认敏感领域的表达边界

从中提取写作参数：

| 参数 | 说明 | 举例 |
|------|------|------|
| **人称** | 第一人称/第三人称/拟人 | "我""本龙""咱家" |
| **调性** | 专业/轻松/毒舌/温暖/搞笑 | 根据 SOUL.md 性格 |
| **口头禅/特色用语** | agent 特有的说法 | "本龙觉得""说人话就是" |
| **emoji 密度** | 不用/适量/密集 | 小红书适量为佳（10-15个/篇） |
| **段落节奏** | 长段深度/短段快节奏/问答式 | 根据人设和选题类型 |
| **互动方式** | 提问/投票/征集/吐槽引战 | 符合人设的互动引导 |

> **关键原则：同一个选题，不同 agent 写出来的文章应该完全不同。**
> 区别不在于内容本身，而在于视角、语气、用词和情绪。

### 4.2 内容结构

内容结构不是固定模板，而是根据 **选题类型 × agent 风格** 灵活组合：

#### 基础骨架（所有笔记通用）

```
【标题】≤ 20 字，含关键词，前 10 字抓人
【正文】主体内容
【配图方案】文字配图 / 实拍图 / 截图
【话题标签】3-5 个
```

#### 正文结构变体（根据风格选择）

**叙事型**（适合故事感强的 agent）
```
场景开头 → 经过 → 转折/感悟 → 互动引导
"上周五收盘后我打开账户一看..."
```

**干货型**（适合专业/理性的 agent）
```
结论先行 → 分点展开 → 数据/案例 → 总结 + 互动
"3 个月实测下来，我的结论是..."
```

**吐槽型**（适合搞笑/毒舌的 agent）
```
槽点切入 → 放大痛点 → 反转/解法 → 引发共鸣
"谁懂啊，基金跌的时候我比它还绿..."
```

**对话型**（适合亲和/温暖的 agent）
```
"你是不是也遇到过..." → 分享经验 → "你们觉得呢？"
读起来像跟朋友聊天
```

**清单型**（适合实用导向的 agent）
```
"X 件事/X 个方法/X 个坑" → 逐条展开 → 收藏引导
适合收藏型内容
```

### 4.3 写作红线

无论什么风格，以下底线不可突破：

- **活人感** — 2026 算法打压 AI 味内容，必须有个性、有情绪、有瑕疵
- **真实 > 精致** — 真实经历和感受比完美干货更有流量
- **人设一致** — 不能这篇文艺下篇粗犷，风格要稳定
- **不说空话** — 每一段都要有信息量，删掉"众所周知""首先我们来看"这种废话
- **敏感领域** — 投资/健康/法律等需要在末尾加免责声明，措辞风格也要符合人设

### 4.4 质量自检

- [ ] 标题 ≤ 20 字，含搜索关键词
- [ ] 正文 500-800 字，段落短（1-3 行/段）
- [ ] 开头 3 行能留住人
- [ ] 结尾有互动引导
- [ ] 敏感领域有风险提示
- [ ] 无敏感词
- [ ] **读一遍正文，听起来像"我"说的话吗？** — 最重要的一条
- [ ] 与近 7 天发布不重复

### 4.5 展示给用户确认

将完整内容展示给用户，确认后调用发布 skill。

---

## Step 5: 发布与记录

1. 调用小红书发布 skill（MCP 或浏览器方式）
2. 发布后记录到当天日记
3. 更新 `topic-library.md` 中的发布记录

## 素材积累

> 素材积累由 **`素材积累.md`** 子技能负责，定时巡逻 + 随手记录，将素材写入 `topic-library.md`。
> 本子技能专注于从素材库中**选题 → 生成内容 → 发布**。

---

## 每周内容节奏建议

| 维度 | 建议 |
|------|------|
| 频率 | 每天 1 篇，最多 2 篇（每日 3 次推荐窗口，发帖后自动停止推荐） |
| 间隔 | 两篇之间 ≥ 3 小时 |
| 方向轮换 | 不连续发同一方向，穿插不同内容降低营销号风险 |
| 周末 | 可以轻松一些，也可以休息 |
| 系列化 | 有固定系列的话，每周固定一天发（如"周三科普"） |

---

## 注意事项

1. **选题推荐是建议，不是命令** — 用户可以修改、替换、跳过
2. **不发半成品** — 所有内容必须用户确认后才发布
3. **人设一致性** — 内容风格必须和 agent 的 SOUL.md 一致
4. **遵守平台规则** — 对照限流规则备忘自检
5. **记录一切** — 每次发布记入日记和素材库，便于复盘
6. **素材库是核心资产** — 平时多积累，用时不愁没东西写
