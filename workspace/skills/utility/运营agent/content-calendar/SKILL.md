# 📅 选题日历

> 没有规划的账号像随机播放的歌单——每首歌单独听还行，听完记不住是谁的专辑。有规划的账号像一部连续剧，观众追着看下一集。

---

## 核心理念

每个账号是一个**角色**在经历一段**旅程**。读者关注你，不是因为某一篇好，是想看"这个人接下来会怎样"。

**每篇帖子都应该能回答：这篇在ta的故事里是第几章？**

## 每日运行流程

> **触发方式**：cron 每日一次（建议上午 9:30）。
> **原则**：轻量快速，有变化才改 queue，无事则跳过。

### Step 1 — 数据刷新

```bash
bash dashboard/query-stats.sh --summary
```

快速看哪些 bot 有新数据。对有新发布的 bot：

```bash
bash dashboard/query-stats.sh {botN} --top=3 --with-content
```

> `--with-content` 自动从 publish-queue/published/ 匹配标题附带正文。
> 数据来自 `dashboard/xhs-stats.json`（cron 每天 9:00/21:00 自动采集）。

**检查项**：
- 昨天发的帖子数据怎么样？
- 有没有意外的亮点/低谷？
- 需不需要更新 queue 里的「数据回顾」区块？

### Step 2 — 环境扫描

快速搜索当日环境（web search）：
1. `今日 经济数据` / `今日 财报` — 有确定性事件吗？
2. 各 bot 支柱相关关键词 + "小红书 热门" — 有热点吗？

> 搜不到就跳过，不要瞎猜。

**检查项**：
- 今天有没有必须响应的事件？→ 如果有，插入/调整对应 bot 的 queue
- 有没有新热点适合某个 bot？→ 如果有，加入 queue 或备选池

### Step 3 — 读取 bot 素材库

各 bot 自己会采集和维护素材库，路径：`workspace-botN/memory/topic-library.md`

这是排期选题的**第一输入源** — bot 自己采集的素材最贴合其人设，自带角度和时效判断。

**每日读取**（对有排期需求的 bot）：

```bash
cat /home/rooot/.openclaw/workspace-botN/memory/topic-library.md
```

**从素材库中挑选的优先级**：
1. 🔥🔥 紧急素材（今日必用）→ 直接排入当天
2. 🔥 时效素材（本周内用）→ 排入近几天的空位
3. 🌲 常青素材 → 填充没有热点的空白日
4. Step 2 环境扫描发现的新热点但 bot 素材库里没有 → 补充进 queue，同时可提醒 bot 更新素材库

**注意**：
- 不要复制素材库内容到 queue，只引用选题方向和角度
- bot 的素材库里已有角度分析，排期时直接采用或微调，不要另起炉灶
- 如果素材库为空或过期，才需要运营 agent 自行从环境扫描结果中安排选题

### Step 4 — 更新 queue

根据 Step 1-3 的结果，更新 `queue/botN.md`：

**日常更新（每天都做）**：
- 已发布的条目：标记 📝 已发布，补上实际数据
- 今天要推送的条目：确认选题仍然合适（没被热点或新素材取代）
- 素材库里有紧急素材但 queue 里没排 → 插入
- 热点插队：强相关热点来了，插入今天或明天

**周度规划（每周一额外做）**：
- 读取各 bot 的 `SOUL.md`（确认人设和内容支柱）
- 通读各 bot 素材库，从中挑选本周选题，补充未来 7 天排期
- 跨 bot 撞题检查（见下方规则）
- 更新「数据回顾」和「月度方向」

**月度调整（每月初额外做）**：
- 用 `query-stats.sh` 汇总上月全部数据
- 更新「月度方向」：支柱权重、主题、教训
- 清理过期的备选池条目

### Step 5 — 推送当日任务

对每个今天有 ⏳ 待推送 条目的 bot，生成推送内容，写入该 bot 的 workspace：

**推送文件**：`workspace-botN/今日选题.md`（覆盖写，bot 启动时读）

```markdown
# 今日选题（{YYYY-MM-DD}）

> 来自运营排期，写之前先看完。

## 选题
{方向}

## 角度
{具体切入}

## 为什么今天发这个
{理由：数据、热点、支柱轮转等}

## 参考你的旧作
《{标题}》：
> {正文关键段落}

这篇好在：{具体分析}。这次可以延续 {xxx}，但 {yyy} 换个方式试试。

## 注意
- {上次这个支柱的教训}
- {跟其他号的差异化要求}
- 优先级：🔴必发 / 🟡弹性（有更好的热点可以替换）
```

推送后更新 queue 状态为 ✅ 已推送。

### Step 6 — 静默判断

如果今天所有 bot 都没有待推送条目，且没有新数据变化、没有热点事件：

```
CALENDAR_OK — 无需更新
```

## 排期规则

### 节奏控制
- **轻重交替** — 不连发深度长文，不连发短评
- **支柱轮转** — 各支柱穿插，不连续 3 篇同支柱
- **每日至少一篇** — 每个 bot 每天至少安排 1 篇，保持账号活跃度和算法权重。素材库里有常青素材兜底，不存在"没东西发"的情况
- **热点优先** — 强相关热点来了，打破常规节奏，当天可以发 2 篇（热点 + 原计划）

### 跨 bot 撞题检查

> 同天同话题两个号发，读者会觉得"这是一个团队在操作"。

打开所有 `queue/botN.md`，检查：
- [ ] 同天内不同 bot 选题是否重合？
- [ ] 同热点各 bot 角度是否足够不同？
- [ ] 近 7 天内是否多个 bot 都在聊同一话题？

**冲突处理**：
1. **角度不同** → 允许，但时间错开至少 2 小时
2. **角度相似** → 保留更契合的那个，另一个换选题或延后
3. **多 bot 同话题** → 各自视角必须不同（宣妈→妈妈怎么看，老K→行业怎么看，来财→小白怎么看）

### 与 bot 素材库的关系

部分 bot 自己维护 `memory/topic-library.md`（素材库/灵感池）。关系：
- **运营 agent 排期 > bot 自选** — queue 里有安排的日子，bot 按排期写
- **空白日 bot 自选** — queue 里没安排的日子，bot 可以从自己的素材库选题
- **bot 素材库是输入** — 运营 agent 做周度规划时，可以参考 bot 的素材库选合适的选题排进 queue
- **不重复维护** — queue 只管排期和推送，不复制素材库的内容

## 附录：参考知识

> 以下是排期时用到的背景知识。不需要每次都读，有疑问时查阅。

### A. 内容支柱设计方法

每个 bot 应该有 **3-4 个内容支柱**，所有内容都能归入某个支柱。

**设计三步法**：

1. **从 SOUL.md 提取**：身份标签、专业领域、性格特征、差异化点
2. **交叉组合**：`内容支柱 = 专业领域 × 内容形式 × 人设角度`
3. **三问检验**：
   - 可持续吗？（能写 20 篇不重复？）
   - 有差异吗？（系统内其他 bot 角度不同？）
   - 有受众吗？（小红书上有人搜？）

### B. 节奏参考

| 账号阶段 | 频率 | 理由 |
|---------|------|------|
| 所有阶段 | 每天至少 1 篇 | 保持算法权重和账号活跃度 |
| 冷启动（<1000粉） | 每天 1-2 篇 | 快速积累，让算法认识你 |
| 成长期（1000-1万粉） | 每天 1 篇 + 热点加发 | 稳定输出，热点时加量 |
| 成熟期（>1万粉） | 每天 1 篇，质量优先 | 保底日更，重质不重量 |

**时间节点意识**：

| 节点 | 内容方向 | 示例 |
|------|---------|------|
| 每周一 | 新周展望/上周复盘 | "新的一周，市场最该关注这几件事" |
| 每月初 | 月度复盘 | "3月操作回顾：对了2次错了1次" |
| 数据日 | CPI、美联储、财报季 | 提前预热，当天即时反应 |
| 季节性 | 年终总结、开年展望 | "2026上半年成绩单" |

### C. 系列化模式

| 模式 | 说明 | 示例 |
|------|------|------|
| 计划型 | 提前规划 3-5 篇 | "黄金入门三部曲" |
| 衍生型 | 从爆款评论区长出来 | 某篇互动高 → 围绕高频问题出延伸 |
| 固定栏目 | 周期性仪式感 | "宣妈每周金价日记"（每周五） |

**系列命名**：包含 bot 名字 + 数字暗示连续性 + ≤ 10 字。

### D. 热点响应三道过滤器

```
过滤器1：相关性 — 跟该 bot 内容支柱有关吗？
         ↓ 有关
过滤器2：独特性 — 该 bot 能提供独特视角吗？
         ↓ 能
过滤器3：时效性 — 现在发还来得及吗？（>24h 需要新角度）
         ↓ 来得及
→ 值得追
```

| 策略 | 场景 | 时效 | 深度 |
|------|------|------|------|
| 闪评 | 突发，有即时观点 | 2h内 | 150-300字 |
| 深度拆解 | 热点背后有逻辑链 | 24h内 | 600+字 |
| 反转视角 | 情绪一边倒 | 48h内 | 300-600字 |

### E. 差异化矩阵

| 维度 | 说明 | 示例 |
|------|------|------|
| 受众 | 面向不同人群 | 宣妈→理财妈妈，老K→专业投资者 |
| 深度 | 不同信息密度 | 来财→入门，老K→深度 |
| 语气 | 不同表达风格 | 来财→可爱，老K→犀利 |
| 角度 | 同话题不同切入 | 黄金涨→宣妈聊情绪，老K聊逻辑 |
| 形式 | 不同内容载体 | 有的擅短评，有的擅长文 |
