# 🔍 爆款拆解

> 不是"抄爆款"，是"读懂爆款的结构"。一篇帖子火了，不是因为运气，是因为它在标题、开头、情绪、互动设计上同时做对了好几件事。拆出来这些"做对的事"，才能复用。

---

## 核心理念

爆款的底层特征是**跨领域通用**的。理财帖和美妆帖的话题不同，但让人点进去的标题钩子、让人读完的情绪曲线、让人评论的互动设计，底层逻辑是一样的。

**本技能不关心"写什么"，只关心"怎么写才有人看"。**

## 采样流程

> **触发方式**：每周一次深度采样（配合选题日历周度规划），或审稿/日常发现好帖子时随时补充。

### Step 1 — 关键词采样

用 `search_feeds` 按关键词矩阵搜索，**不限于 bot 的垂类**。

**关键词三层矩阵**：

| 层级 | 目的 | 示例关键词 |
|------|------|-----------|
| 泛流量词 | 看小红书整体什么内容在跑量 | "搞钱"、"副业"、"踩坑"、"干货"、"翻车" |
| 情绪词 | 找情绪驱动型爆款 | "崩溃"、"后悔"、"真香"、"破防"、"绷不住" |
| 生活词 | 找人感强、有生活质感的爆款 | "带娃"、"下班后"、"租房"、"独居"、"30岁"、"辞职" |
| 垂类词 | 各 bot 相关领域 | "理财"、"基金"、"黄金"、"股票"、"投资" |

**每次采样**：
1. 从每层各选 2-3 个关键词
2. 每个关键词 `search_feeds` 搜索
3. 从结果中筛选高互动帖子（见下方筛选标准）
4. 对命中的帖子用 `get_feed_detail` 获取完整正文 + 评论

**筛选标准**（初始值，随数据积累调整）：

| 指标 | 下限 | 上限 | 说明 |
|------|------|------|------|
| 点赞 | ≥ 500 | ≤ 5000 | 太高的多靠博主影响力，结构参考价值低 |
| 评论 | ≥ 50 | — | 互动深度指标 |
| 收藏 | ≥ 200 | — | 实用价值指标 |
| 点赞/评论比 | — | < 10 | 比值越低说明互动质量越高 |
| 博主粉丝数 | — | — | 必须记录，用于判断"内容力"vs"影响力" |

> 不是所有指标都要满足，满足任意两项即可入选。
> **最有参考价值的区间是"中等爆款"**——500-5000 赞的帖子说明内容本身有力量，而不是靠粉丝基数撑起来的。这个阶段的帖子跟 bot 当前体量最接近，模式最可复用。

**博主粉丝量分级**：

| 粉丝量 | 标签 | 参考权重 |
|--------|------|---------|
| < 1000 | 素人 | ⭐⭐⭐ 最高——粉丝少还能爆，纯靠内容力 |
| 1000 - 1万 | 小博主 | ⭐⭐⭐ 高——跟 bot 体量接近，最可复用 |
| 1万 - 10万 | 中腰部 | ⭐⭐ 中——有一定粉丝基数加成，结构仍有参考价值 |
| > 10万 | 头部 | ⭐ 低——数据大概率靠影响力撑，除非内容结构确实特别 |

> 采样时用 `user_profile` 获取博主粉丝数。同样的 1000 赞，素人发的和 10 万粉博主发的，参考价值完全不同——**素人爆款才是真正的内容力标本**。

**评论抓取策略**：
- `get_feed_detail` 时**不加载全部评论**，只取默认首屏（约 10-20 条高赞评论）
- 目的不是统计评论量，而是看**用户在讨论什么**——哪句话引发了共鸣、哪个观点有争议、哪段内容被反复提及
- 高赞评论往往就是读者的"投票"，指出了正文中最有力量的部分

**每次采样目标**：15-20 篇，覆盖至少 3 个不同领域。

### Step 2 — 单篇拆解

对每篇入选帖子做 **6 维度结构化拆解**。

#### 拆解卡片模板

```markdown
## 拆解卡片 #{序号}

**基础信息**
- 标题：{原标题}
- 作者：{昵称}（粉丝量 {N}）
- 类型：{图文 / 视频}
- 数据：👍{N} 💬{N} ⭐{N}
- 领域：{理财 / 生活 / 职场 / 情感 / ...}
- 链接：{笔记ID}

---

### 维度一：标题

- **钩子类型**：{痛点 / 反直觉 / 数字 / 悬念 / 身份认同 / 情绪共鸣 / 争议}
- **字数**：{N} 字
- **情绪词**：{提取标题中的情绪关键词}
- **分析**：{为什么这个标题让人想点——一句话}

### 维度二：开头（前 3 行）

- **抓人方式**：{故事开场 / 直接亮观点 / 制造冲突 / 自嘲 / 提问 / 场景代入}
- **原文摘录**：
  > {帖子前 3 行原文}
- **分析**：{开头做对了什么}

### 维度三：结构

- **结构类型**：{故事型 / 清单型 / 对比型 / 自述型 / 教程型 / 争议型 / 反转型}
- **段落数**：{N} 段
- **骨架**：
  1. {第一段在干什么——引入/铺垫/...}
  2. {第二段在干什么——展开/转折/...}
  3. {第三段...}
  ...
- **分析**：{结构上的亮点}

### 维度四：情绪曲线

- **曲线类型**：{开高走平 / 低开高走 / 过山车 / 平铺有转折 / 持续升温}
- **关键转折点**：{在第几段，什么事件/话语触发了情绪变化}
- **分析**：{情绪设计做对了什么}

### 维度五：互动设计

- **结尾钩子**：{提问 / 征集经历 / 抛争议 / 开放结局 / 晒自己等结果 / 无}
- **评论区特征**：{评论主要在聊什么——共鸣 / 争论 / 补充经历 / 求教程}
- **高赞评论摘录**（前 3 条）：
  1. "{评论内容}" — 👍{N}
  2. ...
- **分析**：{为什么这篇评论多}

### 维度六：视觉与排版

- **封面风格**：{纯文字 / 实拍 / 拼图 / 截图 / 手写体 / ...}
- **正文排版**：{密集长文 / 短段落 / 带emoji分隔 / 列表式 / ...}
- **emoji 密度**：{高 / 中 / 低 / 无}

---

### 一句话总结

> {这篇帖子最值得学的一件事是什么}

### 可提炼模式

- {标题模式：xxx}
- {结构模式：xxx}
- {互动模式：xxx}
```

### Step 3 — 模式归纳

> 单篇拆解是素材，模式归纳才是资产。5 篇以上同类拆解才值得归纳为一个"模式"。

**归纳流程**：
1. 通读 `samples/` 下所有拆解卡片
2. 按 6 个维度分别聚类：哪些帖子在某个维度上用了类似手法？
3. 出现 ≥3 次的手法 → 提炼为一个"模式"
4. 每个模式写入 `patterns.md`

**模式的结构**：

```markdown
### 模式名：{简短命名}

> {一句话定义这个模式}

**适用场景**：{什么类型的内容适合用这个模式}

**案例**：
1. 《{帖子标题}》— 👍{N} 💬{N}
   > {关键摘录}
2. 《{帖子标题}》— 👍{N} 💬{N}
   > {关键摘录}

**反面案例**（做不好是什么样）：
> {常见的失败版本}

**使用要点**：
- {要点1}
- {要点2}
```

## 与现有 skill 的集成

### → draft-review（审稿）

Phase 3 标题/内容打磨时，参照 `patterns.md`：
- 标题不够吸引 → 推荐匹配的标题模式 + 案例
- 开头抓力不足 → 推荐开头模式
- 互动潜力低 → 推荐互动设计模式

审稿意见从"我觉得标题不行"变成"你的标题是信息摘要型，但这个领域跑量的是悬念型——看这个案例"。

### → content-calendar（选题日历）

周度规划时参考：
- 最近哪类结构/情绪模式在跑量 → 排期时倾向这类
- 哪些领域的互动模式在变化 → 调整 bot 的互动引导策略

### → bot 写稿

今日选题.md 可附带推荐参考模式：
```markdown
## 推荐参考模式
- 标题：试试「反直觉钩子」模式（见 patterns.md 1.3）
- 结构：这个话题适合「过山车型」情绪曲线
```

## 运行频率与维护

| 动作 | 频率 | 说明 |
|------|------|------|
| 深度采样 | 每周一次 | 配合选题日历周度规划，周一做 |
| 随手补充 | 随时 | 审稿或日常发现好帖子时，补一张拆解卡片 |
| 模式归纳 | 每两周一次 | 积累够了再归纳，不要3篇就急着提模式 |
| 模式修订 | 每月一次 | 检查已有模式是否过时，小红书风向变化快 |
| 阈值校准 | 每月一次 | 根据采样经验调整筛选标准 |

## 注意事项

1. **不要复制内容**：拆解是提炼结构和手法，不是搬运正文。patterns.md 里的案例只保留关键摘录
2. **跨领域优先**：不要只采自己 bot 的垂类。美食帖的标题技巧可能比理财帖更值得学
3. **数据会骗人**：高赞不等于值得学。有些帖子纯靠颜值/争议，结构上没什么可学的——跳过
4. **模式会过时**：小红书的算法和用户口味在变，半年前的爆款模式现在可能已经失效。patterns.md 要标注观察时间
5. **量变到质变**：前几次采样可能看不出规律，坚持积累到 30+ 篇拆解后模式会自然浮现
