# 热门趋势播报（trend-broadcast）

> 抓取外部热门内容，整理成简洁的趋势播报。所有命令均为 PUBLIC 策略，直接 HTTP 请求，无需登录。

## 可用数据源

| 数据源 | 命令 | 内容类型 | 速度 |
|--------|------|---------|------|
| **新浪财经 7x24** | `opencli sinafinance news` | 实时快讯，带阅读量 | ~0.2s |
| **36氪** | `opencli 36kr news` | 科技/商业新闻，带摘要 | ~0.4s |
| **微信读书排行** | `opencli weread ranking` | 热门书籍排行榜 | ~0.1s |
| **微信读书搜索** | `opencli weread search "<关键词>"` | 书籍搜索 | ~0.2s |

## 通用参数

- `--limit N` — 返回条数（默认 20）
- `-f json` — JSON 格式输出（推荐，方便解析）
- `-f table` — 表格格式（默认，适合直接阅读）

## 使用场景

### 1. 获取实时热点快讯

```bash
opencli sinafinance news --limit 10 -f json
```

返回字段：`id`, `time`, `content`, `views`

适用：捕捉突发新闻、财经热点、社会事件，判断当下什么话题有热度。

### 2. 科技商业选题

```bash
opencli 36kr news --limit 10 -f json
```

返回字段：`rank`, `title`, `summary`, `date`, `url`

适用：了解科技/创业/商业领域最新动态，寻找内容角度。

### 3. 热门书籍话题

```bash
# 总榜
opencli weread ranking --limit 10 -f json

# 按关键词找书
opencli weread search "职场" --limit 5 -f json
```

ranking 返回字段：`rank`, `title`, `author`, `category`, `readingCount`, `bookId`
search 返回字段：`rank`, `title`, `author`, `bookId`, `url`

适用：发现当下读者关注的书籍话题，书评/读书笔记类内容选题。

## 选题工作流

当需要寻找选题灵感时，按以下顺序执行：

1. **扫快讯**：`opencli sinafinance news --limit 15 -f json` — 看当下什么事件正在发酵
2. **看科技**：`opencli 36kr news --limit 10 -f json` — 看有没有适合自己人设的科技/商业话题
3. **翻书榜**：`opencli weread ranking --limit 10 -f json` — 看热门书籍有没有可以结合的内容角度

从以上结果中筛选与自己人设匹配的话题，结合 SOUL.md 和 CONTENT_STYLE.md 判断是否适合发帖。

## 播报输出格式

执行完三个数据源后，整理成以下格式播报：

```
【热门趋势播报】{日期} {时段}

📰 实时快讯（新浪财经）
- {时间} {内容摘要}（{阅读量}）
- ...
（取阅读量最高的 5 条）

💡 科技商业（36氪）
- {标题}：{一句话摘要}
- ...
（取最新 5 条）

📚 热门书籍（微信读书）
- TOP{N} 《{书名}》{作者} — {分类}
- ...
（取 TOP 5）
```

播报要简洁，每条一行，不展开分析。目的是快速扫描当前热点。

## 注意事项

## 异常处理策略
当 `opencli` 命令执行失败时，按以下方式处理：
| 异常类型 | 判断依据 | 处理方式 |
|---------|---------|---------|
| **超时** | 命令执行 > 5s 无响应 | 跳过该数据源，继续执行其余命令；在播报末尾注明"[新浪/36氪/微信读书] 数据获取超时，已跳过" |
| **返回空** | JSON 返回 `[]` 或 `{}` | 跳过该数据源；注明"[数据源名] 暂无数据" |
| **格式异常** | 输出无法解析为预期字段 | 尝试切换 `-f table` 重试一次；仍失败则跳过并注明 |
| **命令不存在** | `command not found` 类报错 | 停止执行，提示用户检查 opencli 安装 |
**最低可用原则**：三个数据源中只要有一个成功返回数据，仍可输出播报；若全部失败，告知用户网络或服务异常，不输出空播报。
## 选题筛选优先级
从抓取结果中判断哪些值得作为选题，按以下标准排序：
### 快讯优先级（新浪财经）
1. **阅读量 > 5000** — 优先考虑，说明话题有广泛关注度
2. **阅读量 1000–5000** — 次选，结合内容判断是否与自身人设相关
3. **阅读量 < 1000** — 一般跳过，除非内容与自身领域高度相关
4. **负面/灾难类内容**（事故、死亡、自然灾害等）— 即使阅读量高也跳过，不适合用于创作选题
### 36氪优先级
1. 标题涉及 **AI、效率工具、创业方法论、职场趋势** — 优先，通用人设适配度高
2. 标题涉及 **融资/上市/公司八卦** — 次选，需确认与自身内容方向相关
3. 纯财经数据/股市类 — 一般跳过
### 书榜优先级
1. **TOP 1–5** 且分类为 **个人成长、职场、心理、商业** — 优先
2. **TOP 1–5** 但分类为小说/漫画 — 次选，内容创作角度有限
3. **TOP 6–10** — 备选，仅在前五无合适内容时考虑
### 热度衰减判断
- 同一快讯在连续两天的播报中均出现 → 话题进入尾声，不建议再作为新选题
- 36氪文章发布日期超过 **3 天** → 时效性降低，慎用
- 书榜排名连续 **7 天无变化** → 长青内容，适合长期选题池，但不作为当下热点
## 数据时效性
| 数据源 | 更新频率 | 有效时间窗口 | 过期判断 |
|--------|---------|------------|---------|
| 新浪财经 7x24 | 实时（分钟级） | 当天有效，超过 **6 小时**的快讯降低优先级 | 快讯时间戳距当前 > 6h 标记为「旧」 |
| 36氪 | 每日多次更新 | 发布后 **3 天内**有效 | `date` 字段超过 3 天视为过期 |
| 微信读书榜 | 每日更新一次（通常凌晨） | 当天榜单全天有效 | 无时间戳，默认当天数据有效 |
执行播报时，若快讯中出现大量 6 小时前的内容，说明当前时段新闻较少，可在播报中注明「近期快讯较少，以下为今日较早内容」。
## 注意事项
- **不要原样搬运**：这些数据是灵感来源，不是内容本身。如果用于创作，必须用自己的人设和风格重新加工。
- **时效性**：快讯数据实时更新，适合当天使用；书籍排行变化较慢，可作为长期选题池。
- **网络限制**：当前服务器部分海外站点不可达，以上数据源均已验证可用。
- **播报不是创作**：趋势播报只做信息整理，不需要加入个人观点或人设语气。
