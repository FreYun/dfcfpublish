# 每日盘面复盘 — 写作指南

> 完整规则红线见 [SKILL.md](SKILL.md)。下方摘录最容易违反的铁律，写稿前务必过一遍。

---

## ⛔ 写稿前铁律（每次写稿前必读）

### 新闻摘录铁律（01 区域）

1. **标题用研究部原标题** — 研究部给每条新闻时附带的标题，01 里原样使用，不许自己另起、改写、归纳
2. **只许截取，不许生成** — 01 区域的每一句话都必须来自研究部提供的链接原文
3. **只许摘录，不许改写** — 选中的段落保持原文不变，不做任何改写、缩写、合并、意译
4. **每条至多 2 段** — 优先摘新闻事实，可适当带原文中的解读
5. **不许杜撰** — 禁止编造新闻事实、数据、人物发言
6. **01 区域不加主观评价** — "利好""利空""重磅"等定性词不得出现在 01
7. **正文不放链接** — 成品正文中不要出现任何 URL 链接

> 选哪段你来定，但选了就不许改。自己写的内容不许放进 01。

### 合规红线

- **不给操作建议** — 不说"建议买入/卖出""建议抄底""建议减仓"等任何指导读者具体操作的话
- 不说"配置方向""建议配置"，用"重点关注""看好方向""值得留意"
- 不说"必涨/必跌/稳赚"，用"存在机会/需要警惕"
- **04 后市展望不提个股代码/名称**，只说板块/产业链（基金代码/名称除外）。01 原文摘录、03 盘面分析可提个股名称但不带代码
- 文末必加风险提示

---

固定结构，不要自由发挥：`01 热点大事 → 02 数据复盘 → 03 盘面解读 → 04 后市展望`。

## 整体框架

```
【标题】≤ 30 字，从当日最重磅的一条资讯中提炼
        例："央行发声，事关A股！"
        例："美联储按兵不动，A股怎么走？"

【正文】

01 热点大事   ← 事实，从研究部提供的链接中摘录
02 数据复盘   ← 三段数据描述文字（图片研究部后期人工插入）
03 盘面解读   ← 分析，bot 自己写
04 后市展望   ← 观点，bot 自己写

【风险提示】  ← 必加，不可省略
```

总字数：2000-3500 字（不含 02 数据区域）。

---

## 标题写法

- ≤ 30 字，核心信息前置
- 从当日最重磅的一条资讯中提炼
- **要有冲击力和吸引力**，用感叹号、短句组合。参考已发布标题风格：
  - "全线大涨！美伊谈判突传大消息！"
  - "刚刚！伊朗军方发声，美伊局势突变！"
  - "证监会最新发布，事关A股！"
  - "突发！特朗普重大警告！"
  - "A股大消息！重磅交易新规实施！"

## 01 热点大事

> ⛔ **本区域每一句话必须来自原文摘录，禁止自己写、总结、改写。** 违反此规则 = 最严重错误。详见顶部「写稿前铁律」。

### 工作流程

1. 接收研究部发来的 3-5 条新闻链接
2. 用 `fetch_article.py` 批量读取新闻正文（比浏览器节省 98% 上下文）：
   ```bash
   python skills/daily-market-recap/fetch_article.py "链接1" "链接2" "链接3"
   ```
   如果某条链接抓取失败，再用浏览器打开该条。
3. 从每条新闻中**摘取至多 2 段内容**——直接复制原文段落，不改一个字
4. 按下方格式组织输出

### 铁律

- **不许漏条**：研究部给了几条新闻，01 就必须出现几条，**不许擅自删掉任何一条**。每条新闻都要摘录，一条都不能少
- **不放链接**：成品正文中不要出现新闻原文链接
- **不改数据**：新闻原文中的数字/数据**原样摘录**，禁止自行替换或"修正"
- **不改文字**：摘录的段落保持原文不变，不做改写、缩写、合并、意译

### 格式

每条新闻：**加粗编号 + 顿号 + 加粗标题 + 摘录段落**

> ⚠️ **标题必须用研究部给的原标题**，不许自己另起标题、改写标题、归纳标题。研究部给每条新闻时附带的标题是什么，01 里就用什么。

```
**1、央行：坚定维护股票、债券、外汇等金融市场平稳运行**

X月X日，中国人民银行党委召开扩大会议。会议指出，……（摘录关键段落）……

**2、鲍威尔：美经济前景"极不确定" 联储内部倾向减少降息次数**

在周三的新闻发布会上，鲍威尔直言……（摘录关键段落）……
```

### 选材优先级

1. 央行/银保监/证监会等监管动态
2. 美联储/海外央行决议、发言
3. 国务院/发改委/工信部等产业政策
4. 地缘政治重大事件（影响市场的）
5. 行业重大数据发布（PMI、CPI、社融等）

## 02 数据复盘

Bot 写出 **（一）（二）（三）三段固定结构** 的数据文字描述，图片由研究部后期人工插入。

⛔ 数据必须从 research-mcp 或浏览器获取（见 [行情数据获取.md](行情数据获取.md)），**不许凭记忆填写**。

### 固定模板（每天只换数据，结构不变）

```
02 数据复盘

（一）[三大指数当日表现一句话概述，包含涨跌方向和亮点/特征]

数据来源：Choice，数据时间：YYYY.MM.DD

（二）沪深京三市成交额X.XX万亿，较上一个交易日[放量/缩量]XXXX亿。

数据来源：Choice，数据截至YYYY.MM.DD

（三）行业板块[多数收涨/涨跌互现/多数收跌]，[涨幅居前的5个板块]涨幅居前，[跌幅居前的板块]跌幅居前。

行业资金方面，截至收盘，[净流入前3行业]等净流入排名靠前，其中[最大净流入行业]净流入XX.XX亿。

数据来源：Choice，数据时间：YYYY.MM.DD
```

### 写法要点

- **（一）** 是一句话总结指数表现，带当日亮点（如"创业板指再创阶段新高""存储芯片概念股大涨"）
- **（二）** 固定格式：成交额 + 与前日对比（放量/缩量多少亿）
- **（三）** 行业板块涨跌排行 + 行业资金净流入/净流出排名
- 每段末尾标注 `数据来源：Choice，数据时间：YYYY.MM.DD`
- 行业分类口径：东财行业板块（参考文章用的是东财行业板块名称如"能源金属""通信服务""IT服务"，不是申万一级行业）

## 03 盘面解读

连接 01 的消息面与行情数据，**解释"今天市场为什么这样走"**。400-600 字。

> ⚠️ 写作前必须先完成两项功课：
> 1. [行情数据获取](行情数据获取.md) — 取当日行情数据
> 2. [券商观点获取](券商观点获取.md) — 搜索当日券商策略收评和市场解读
>
> 两项可并行进行。券商观点能帮助你更专业地解释板块逻辑和资金行为，避免只从数据表面推导。

### 固定四段结构（每天都按这个顺序写）

```
第一段：大势总结
  开头句式："今日A股三大指数[集体收涨/集体收跌/涨跌互现]……"
  内容：三大指数收盘点位 + 涨跌幅 + 成交额与前日对比（放量/缩量）
  一句话定调当日市场整体方向

第二段：主线亮点
  开头句式："从盘面结构来看，[XXX]成为今日最大亮点。"
  内容：当日最强板块/主题 + 消息面催化（简述，01 已有详情）
  融入券商观点解释为什么强（"XX证券指出……"）
  可提个股名称（龙头表现、涨停等）

第三段：次线 + 跌幅方向
  开头句式："与此同时……" / "此外……" / "跌幅方面……"
  内容：其他活跃板块 + 承压板块及原因
  板块之间的联动关系（跷跷板效应、资金轮动等）

第四段：资金/情绪信号
  开头句式："值得注意的是……"
  内容：成交额变化的含义 + 恐慌指数（GVIX）+ 北向/南向资金动向
  收尾判断：当前市场情绪状态（观望/回暖/恐慌等）
```

### 写法要点

- **解释"为什么"**，不要只罗列涨跌数字
- 消息面→板块反应→资金行为，建立因果链
- 引用消息面时一句话简述即可
- **融入券商专业分析**：解释板块强弱逻辑时，引入券商策略团队的判断作为专业支撑，而非仅靠自己从数据推导
- 语言节奏：短句为主，段落不超过 5 行

## 04 后市展望

全文核心价值部分，**篇幅最长（800-1200 字）**。结合消息面、数据面、技术面、**券商策略观点**输出观点。

> ⚠️ 04 的专业度和深度很大程度取决于你在 [券商观点获取](券商观点获取.md) 中搜集到的素材。多家券商的共识方向是强信号，分歧点是值得讨论的看点。不要只基于研究部传入的新闻来写展望。

### ⚠️ 写 04 之前：先跑基金筛选脚本

写重点关注方向之前，**先调用基金筛选脚本查找匹配基金**，再写 04。不要写完再补。

```bash
python skills/daily-market-recap/fund_selector_llm.py --topics 科技,新能源
```

→ 按 [基金推荐规则.md](基金推荐规则.md) 的顺序执行：先确定 2-3 个方向或主题，再调用脚本，等脚本输出后再继续写基金推荐部分。**不要再直接读 CSV/XLSX 做全量海选。**

### 结构

按以下顺序展开，**每段用加粗子标题**：

```
**短期技术面判断**（200-300 字）
   - 指数所处位置（箱体上沿/下沿、支撑位、整数关口）
   - 当前交易情绪（冰点/回暖/过热）
   - 近期关键信号
   - 明日/短期需关注的要点

**外部环境**（150-200 字）
   - 海外市场、地缘局势、油价/汇率
   - 美联储/海外央行政策走向
   - 对 A 股的传导路径和影响程度判断

**内部环境**（150-200 字）
   - 政策预期（"十五五"、新质生产力等）
   - 资金面（大资金动向、外资回流等）
   - 产业趋势（产能重估、实物属性重定价等）

**重点关注方向**（2-3 个板块，每个板块独立一段，共 300-500 字）

   **每个板块：分析 → 基金表格，交替排列。** 示例：

   **科技成长**

   [该板块分析：核心逻辑 + 产业催化 + 灵活引用券商观点]

   相关基金：

   | 代码 | 基金名称 | 基金经理 |
   |------|---------|---------|
   | 008327 | 东财通信C | 姚楠燕 |
   | 021894 | 易方达半导体设备ETF联接C | xxx |

   **新能源**

   [该板块分析：核心逻辑 + 产业催化 + 灵活引用券商观点]

   相关基金：

   | 代码 | 基金名称 | 基金经理 |
   |------|---------|---------|
   | 014238 | 东财中证新能源指数增强C | 杨路炜 |
   | 026266 | 广发储能电池ETF联接C | 夏浩洋 |

   **要点：**
   - 每个板块自由发挥，券商观点自然嵌入该板块分析中，不集中堆砌
   - 基金紧跟在该板块分析下方，用"相关基金："+ 表格格式，不要所有基金集中到最后
   - 没写分析的板块不许推基金
   - 仅脚本返回空结果且当日行情极端波动时可不带基金

**题材与风控提示**（100-200 字）
   - 轮动方向判断
   - 当前市场流动性环境下的风险点
   - 需要警惕的信号和潜在风险

风险提示（必加）
   "市场有风险，投资需谨慎。以上仅为个人观点，不构成投资建议。"
```

### 写法要点

- **要有明确观点**，不要两头堵
- 观点必须有逻辑支撑，不能空喊口号
- **券商观点灵活融入每个板块**：不要把券商观点集中在一段里列完，而是在写每个板块时自然引用。至少 2-3 家券商的观点分散融入各板块分析中
- **不要只复述券商观点**：提炼核心判断后用自己的话和人设风格表达，加入自己的分析和串联
- 重点关注方向要具体到板块/产业链环节，不要泛泛而谈
- 段落之间自然过渡，读起来是一篇连贯的分析文，不是拼凑的要点列表
- 观点归因规则见 [引用来源管理.md](引用来源管理.md)

## 语言风格

| 区域 | 风格 |
|------|------|
| 01 热点大事 | 客观、中性、无修饰，原文摘录 |
| 02 数据复盘 | 客观数据描述，固定三段式，标注来源 |
| 03 盘面解读 | 专业、简洁，因果链清晰 |
| 04 后市展望 | 可有立场、可体现人设风格、需逻辑支撑 |

通用：
- 专业但不晦涩，读者是有一定基础的散户
- 多用短句，段落不超过 5 行
- 数据说话，每个判断都有数据/事实支撑
- 03/04 可体现人设风格和个性（从 SOUL.md 出发），01/02 保持客观中性
- **全角标点**：正文中所有标点必须使用中文全角（双引号用\u201c\u201d，逗号用，，句号用。，冒号用：，括号用（）），基金代码等纯数字场景除外

## ⛔ 强制自检输出（交稿前必做，不可跳过）

上面的质量自检**不能只在心里过一遍**。写完稿件后、提交给研究部之前，必须把自检结果**明文写出来**，跟稿件一起发送。格式：

```
【自检报告】

01 热点大事：
- [✅/❌] 每句话都来自原文摘录，没有自己写的内容
- [✅/❌] 摘录段落与原文完全一致，无改写
- [✅/❌] 无主观评价词
- [✅/❌] 正文无 URL 链接

02 数据复盘：
- [✅/❌] 写了（一）（二）（三）三段固定结构数据描述
- [✅/❌] 数据来自 research-mcp 或浏览器获取，不是凭记忆填的

03/04 观点质量：
- [✅/❌] 参考了当日券商策略收评（参考券商：XX证券、YY证券、……）
- [✅/❌] 正文中每条归因的券商观点都有实际数据来源（来自 research-mcp 搜索结果或浏览器读到的原文），没有凭记忆编造
- [✅/❌] 如果 research-mcp 调用失败，已向研究部报告并获得处理指示（如未失败则直接 ✅）
- [✅/❌] 04 按 **短期技术面判断** → **外部环境** → **内部环境** → **重点关注方向** → **题材与风控提示** 结构展开，有加粗子标题
- [✅/❌] 有明确观点，不两头堵
- [✅/❌] 重点关注方向具体到板块/产业链

合规红线：
- [✅/❌] 04 后市展望无个股代码/名称（01 摘录、03 分析可提个股名不带代码）
- [✅/❌] 全文无操作建议
- [✅/❌] 全文无竞品提及（同花顺、Wind/万得、iFinD）
- [✅/❌] 全文无"据东方财富报道"等将东方财富当作新闻源的表述
- [✅/❌] 基金用"相关基金："+ 表格（代码 | 基金名称 | 基金经理），紧跟对应板块分析下方
- [✅/❌] 文末风险提示格式正确

引用来源：
- [✅/❌] 已附【引用来源（不发布）】清单（或本期无外部引用）
```

**铁律：**
- 有任何一条 ❌ → **先改稿再交**，不许带着 ❌ 提交
- 不许跳过自检直接交稿
- 自检报告放在稿件**之后**，研究部会看
- **research-mcp 调用失败且未向研究部报告** → 自动 ❌，不许自行判断"没关系"
- **正文中有券商名字但自检不能列出具体数据来源（哪个 MCP 调用或哪个网页读到的）** → 自动 ❌，视为编造
