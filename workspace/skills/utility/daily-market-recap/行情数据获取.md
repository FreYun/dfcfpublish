# 行情数据获取（03/04 写作前必做）

## 时间判断

research-mcp 的 A 股指数数据通常在 **16:00 后更新当日收盘数据**。取数前先判断当前时间：

- **16:00 之后**：research-mcp 已更新当日收盘数据，**可以直接用 research-mcp 取所有数据**，浏览器仅在需要校对时使用
- **16:00 之前**（盘中或刚收盘）：research-mcp 还没有当日数据，**用浏览器获取当日实时数据，再用 research-mcp 补充历史**

---

## 第一步：research-mcp 取行情数据

### 16:00 之后（直接取当日收盘数据）

```
# 1. A股大盘指数
npx mcporter call "research-mcp.market_overview()"

# 或指定指数
npx mcporter call "research-mcp.get_ashares_index_quote()"

# 2. 申万一级行业涨跌
npx mcporter call "research-mcp.get_ashares_index_quote(symbol: '801010.SI,801080.SI,801150.SI,801750.SI,801730.SI,801780.SI,801790.SI,801120.SI,801110.SI,801880.SI,801050.SI,801950.SI,801960.SI,801040.SI,801030.SI,801160.SI,801170.SI,801180.SI,801200.SI,801210.SI,801130.SI,801140.SI,801710.SI,801720.SI,801740.SI,801760.SI,801770.SI,801890.SI,801970.SI,801980.SI,801230.SI,801010.SI')"
```

## 第二步：浏览器校对 / 实时数据获取

### 16:00 之后（校对 research-mcp 数据）

浏览器用于**校对**：与第一步 research-mcp 取的数据做交叉验证。

```
# 1. 打开东方财富首页，获取大盘指数（校对用）
browser open url="https://www.eastmoney.com"
browser snapshot

# 记录以下数据：
# a) 大盘指数：上证、深成指、创业板指的收盘点位和涨跌幅
#    → 与第一步 research-mcp 取的数据做校对
#    → 如果两者差异超过 0.03%，立即向研究部报告，以浏览器数据为准
# b) 两市成交额

# 2. 申万一级行业涨跌排行
browser open url="https://www.swsresearch.com/institute_sw/allIndex/releasedIndex"
browser snapshot
# 记录：涨幅前5和跌幅前5的申万一级行业及涨跌幅

# 3. 用完关闭
browser close
```

### 16:00 之前（浏览器作为主数据源）

此时 research-mcp 无当日数据，**浏览器是当日实时数据的主要来源**：

```
# 1. 大盘指数（实时）
browser open url="https://www.eastmoney.com"
browser snapshot
# 记录：上证、深成指、创业板指的实时点位和涨跌幅，两市成交额

# 2. 申万一级行业涨跌排行（收盘后）
browser open url="https://www.swsresearch.com/institute_sw/allIndex/releasedIndex"
browser snapshot
# 记录：涨幅前5和跌幅前5的申万一级行业及涨跌幅

# 3. 用完关闭
browser close
```

> ⚠️ **校对规则**：当两个数据源都有数据时，必须交叉验证。发现不一致时：
> - 差异 ≤ 0.03%：可能是实时 vs 收盘的微小差别，以浏览器收盘数据为准
> - 差异 > 0.03%：**立即向研究部报告**，说明两个数据源的具体数值，等待指示后再写稿

## 第三步：research-mcp 补充历史数据和其他指标

无论什么时间段，**research-mcp 始终用于补充近期走势、估值、资金面等历史数据**：

```
# 1. A股近期走势（用于技术面分析）
npx mcporter call "research-mcp.market_overview()"

# 2. 恐慌指数（如需）
npx mcporter call 'research-mcp.get_ashares_gvix(start_date: "YYYYMMDD", end_date: "YYYYMMDD")'

# 3. 新闻搜索（补充消息面素材）
npx mcporter call 'research-mcp.news_search(query: "关键词", top_k: 5)'
```

> 日期格式：`YYYYMMDD`（如 `20260324`），不是 `YYYY-MM-DD`。

## ⛔ 数据获取失败处理

### research-mcp 调用失败

如果 `npx mcporter call` 返回 "MCP call failed"、空结果 `{}`、超时或报错：

1. **不要跳过，不要凭记忆填数据**
2. **用浏览器作为替代数据源**：打开东方财富首页获取当日行情数据
3. 如果浏览器也无法获取（被拦截、网络异常等），**立即向研究部报告**，说明：
   - 失败的具体调用命令和返回信息
   - 已尝试的替代方案
   - 当前缺少哪些数据，影响哪些区域
4. **等研究部指示后再继续**

### 浏览器被拦截

如果浏览器打开页面时遇到验证码、登录要求或空白页：

1. 换一个数据源网站重试（东方财富 → 新浪财经 → 金十数据）
2. 都失败则向研究部报告

> ⚠️ **绝对禁止在数据获取全部失败的情况下继续写稿。** 没有数据就不写，报告研究部等待指示。

## 行情数据铁律

- **行业分类只用申万一级行业**（如：电子、计算机、医药生物、有色金属等），不许用同花顺概念板块、东财行业分类、自创分类等其他口径
- 大宗商品行情、概念题材行情仍可正常引用，不受此限制

拿到的数据用于：
- **03 盘面解读**：引用收盘点位、涨跌幅、成交额、板块表现（行业口径：申万一级），灵活组织写作即可
- **04 后市展望**：引用技术面位置（点位是判断支撑/阻力的基础）、资金面数据、估值水平
- **02 配图建议**：基于数据给出更精准的配图建议
