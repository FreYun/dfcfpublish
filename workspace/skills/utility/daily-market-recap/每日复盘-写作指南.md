# 每日盘面复盘 — 写作指南

> 完整规则红线见 [SKILL.md](SKILL.md)。下方摘录最容易违反的铁律，写稿前务必过一遍。

---

## ⛔ 写稿前铁律（每次写稿前必读）

### 新闻摘录铁律（01 区域）

1. **标题用研究部原标题** — 研究部给每条新闻时附带的标题，01 里原样使用，不许自己另起、改写、归纳
2. **只许截取，不许生成** — 01 区域的每一句话都必须来自研究部提供的链接原文
3. **只许摘录，不许改写** — 选中的段落保持原文不变，不做任何改写、缩写、合并、意译
4. **每条至多 2 段** — 优先摘新闻事实，可适当带原文中的解读
5. **不许杜撰** — 禁止编造新闻事实、数据、人物发言
6. **01 区域不加主观评价** — "利好""利空""重磅"等定性词不得出现在 01
7. **正文不放链接** — 成品正文中不要出现任何 URL 链接

> 选哪段你来定，但选了就不许改。自己写的内容不许放进 01。

### 合规红线

- **不给操作建议** — 不说"建议买入/卖出""建议抄底""建议减仓"等任何指导读者具体操作的话
- 不说"配置方向""建议配置"，用"重点关注""看好方向""值得留意"
- 不说"必涨/必跌/稳赚"，用"存在机会/需要警惕"
- 不提个股代码/名称，用行业板块描述（基金代码/名称除外）
- 文末必加风险提示

---

固定结构，不要自由发挥：`01 重磅资讯 → 02 数据复盘 → 03 盘面解读 → 04 后市展望`。

## 整体框架

```
【标题】≤ 25 字，从当日最重磅的一条资讯中提炼
        例："央行发声，事关A股！"
        例："美联储按兵不动，A股怎么走？"

【正文】

01 重磅资讯   ← 事实，从研究部提供的链接中摘录
02 数据复盘   ← 配图建议，研究部后期人工插图
03 盘面解读   ← 分析，bot 自己写
04 后市展望   ← 观点，bot 自己写

【风险提示】  ← 必加，不可省略
```

总字数：2000-3500 字（不含 02 配图区域）。

## 标题写法

- ≤ 30 字，核心信息前置
- 从当日最重磅的一条资讯中提炼
- 句式参考："央行发声，事关A股！" / "油价飙升，谁在受益？" / "XXX数据出炉，市场如何反应？"
- 不用标题党，但要有信息量和吸引力

## 01 重磅资讯

> ⛔ **本区域每一句话必须来自原文摘录，禁止自己写、总结、改写。** 违反此规则 = 最严重错误。详见顶部「写稿前铁律」。

### 工作流程

1. 接收研究部发来的 3-5 条新闻链接
2. 用浏览器逐条打开，阅读全文
3. 从每条新闻中**摘取至多 2 段内容**——直接复制原文段落，不改一个字
4. 按下方格式组织输出

### 铁律

- **不许漏条**：研究部给了几条新闻，01 就必须出现几条，**不许擅自删掉任何一条**。每条新闻都要摘录，一条都不能少
- **不放链接**：成品正文中不要出现新闻原文链接
- **不改数据**：新闻原文中的数字/数据**原样摘录**，禁止自行替换或"修正"
- **不改文字**：摘录的段落保持原文不变，不做改写、缩写、合并、意译

### 格式

每条新闻：**编号 + 加粗标题 + 摘录段落**

> ⚠️ **标题必须用研究部给的原标题**，不许自己另起标题、改写标题、归纳标题。研究部给每条新闻时附带的标题是什么，01 里就用什么。

```
1.央行：坚定维护股票、债券、外汇等金融市场平稳运行

X月X日，中国人民银行党委召开扩大会议。会议指出，……（摘录关键段落）……

2.鲍威尔：美经济前景"极不确定" 联储内部倾向减少降息次数

在周三的新闻发布会上，鲍威尔直言……（摘录关键段落）……
```

### 选材优先级

1. 央行/银保监/证监会等监管动态
2. 美联储/海外央行决议、发言
3. 国务院/发改委/工信部等产业政策
4. 地缘政治重大事件（影响市场的）
5. 行业重大数据发布（PMI、CPI、社融等）

## 02 数据复盘

**此区域只写配图建议，由研究部后期人工插图。** ⛔ 不要在 02 里写指数点位、涨跌幅、行业排名等行情数据——这些数据在 03/04 中引用即可。

```

3. 北向资金近期净流入走势图，数据来源：Choice

数据来源：Choice，数据时间：YYYY-MM-DD
```

- 建议 2-3 张图，标注数据来源和日期
- **02 区域只有配图建议，不写其他内容**

## 03 盘面解读

连接 01 的消息面与行情数据，**解释"今天市场为什么这样走"**。400-600 字。

> ⚠️ 写作前必须先完成两项功课：
> 1. [行情数据获取](行情数据获取.md) — 取当日行情数据
> 2. [券商观点获取](券商观点获取.md) — 搜索当日券商策略收评和市场解读
>
> 两项可并行进行。券商观点能帮助你更专业地解释板块逻辑和资金行为，避免只从数据表面推导。

### 结构

```
第一段：一句话概括大势
  → 收盘点位 + 涨跌幅、成交量变化、大小盘分化、赚钱效应

第二段起：板块结构性分析
  → 哪些板块强/弱
  → 为什么强/弱（对应 01 中的消息面）
  → 板块之间的联动关系（跷跷板效应、资金轮动等）
  → 消息面催化（简述事件，不需要全文再引，01 已有）
```

### 写法要点

- **解释"为什么"**，不要只罗列涨跌数字
- 消息面→板块反应→资金行为，建立因果链
- 引用消息面时一句话简述即可
- **融入券商专业分析**：解释板块强弱逻辑时，引入券商策略团队的判断作为专业支撑，而非仅靠自己从数据推导
- 语言节奏：短句为主，段落不超过 5 行

## 04 后市展望

全文核心价值部分，**篇幅最长（800-1200 字）**。结合消息面、数据面、技术面、**券商策略观点**输出观点。

> ⚠️ 04 的专业度和深度很大程度取决于你在 [券商观点获取](券商观点获取.md) 中搜集到的素材。多家券商的共识方向是强信号，分歧点是值得讨论的看点。不要只基于研究部传入的新闻来写展望。

### ⚠️ 写 04 之前：先跑基金筛选脚本

写重点关注方向之前，**先调用基金筛选脚本查找匹配基金**，再写 04。不要写完再补。

```bash
python skills/daily-market-recap/fund_selector_llm.py --topics 科技,新能源
```

→ 按 [基金推荐规则.md](基金推荐规则.md) 的顺序执行：先确定 2-3 个方向或主题，再调用脚本，等脚本输出后再继续写基金推荐部分。**不要再直接读 CSV/XLSX 做全量海选。**

### 结构

按以下顺序展开，每段自然过渡，不需要再加小标题：

```
a) 短期技术面判断（200-300 字）
   - 指数所处位置（箱体上沿/下沿、支撑位、整数关口）
   - 当前交易情绪（冰点/回暖/过热）
   - 近期关键信号
   - 明日/短期需关注的要点

b) 外部环境（150-200 字）
   - 海外市场、地缘局势、油价/汇率
   - 美联储/海外央行政策走向
   - 对 A 股的传导路径和影响程度判断

c) 内部环境（150-200 字）
   - 政策预期（"十五五"、新质生产力等）
   - 资金面（大资金动向、外资回流等）
   - 产业趋势（产能重估、实物属性重定价等）

d) 重点关注方向（200-300 字）
   - 聚焦 2-3 个看好的方向，每个方向说清逻辑链
   - 用行业/板块描述，不提个股代码/名称
   - 区分中线主线 vs 短线题材
   - 给出具体的产业催化线索
   - ⚠️ 不要用"配置方向""建议配置"等措辞，改用"重点关注""看好方向""值得留意"
   - ⚠️ **基金推荐必须紧跟观点** — 只有 04 中明确看好并给出逻辑链的方向，才可以推荐对应板块的基金。没有在 04 中写观点的板块，**不许推荐该板块基金**，不许为了凑基金而硬造观点。
  - ⚠️ **展示格式固定** — 基金推荐必须按板块/方向归类，集中在"重点关注方向"段落末尾统一展示，不要散落在各段里。每个板块独占一行，板块名后用中文全角冒号"："；同板块内基金用中文顿号"、"分隔；每个板块推荐 2-3 只；每只基金展示为"基金名称（代码，基金经理：姓名）"；语气用"感兴趣的朋友可以关注""可以留意"，不用"建议买入""推荐"。
  - ⚠️ **仅以下情况可以不带基金** — 只有脚本返回空结果/明确无合适候选，且当日行情极端波动（单日跌幅超 3%）、配置信号完全混乱时，才可以不带基金；"分析比较笼统""忘了调脚本""觉得硬塞"都不是跳过理由。

e) 题材与风控提示（100-200 字）
   - 轮动方向判断
   - 当前市场流动性环境下的风险点
   - 需要警惕的信号和潜在风险

f) 风险提示（必加）
   "市场有风险，投资需谨慎。"
   "以上仅为个人观点，不构成投资建议。"
```

### 写法要点

- **要有明确观点**，不要两头堵
- 观点必须有逻辑支撑，不能空喊口号
- **券商观点是核心素材**：至少引用 2-3 家券商的当日策略观点，展示专业共识和分歧。多家共同看好的方向说服力更强；存在分歧时呈现不同声音更有深度
- **不要只复述券商观点**：提炼核心判断后用自己的话和人设风格表达，加入自己的分析和串联
- 用"关注""可以留意"代替"建议买入"
- 用"存在机会""需要警惕"代替"必涨""必跌"
- 重点关注方向要具体到板块/产业链环节，不要泛泛而谈
- **不说"配置方向""建议配置"**，用"重点关注""看好""值得留意"
- 段落之间自然过渡，读起来是一篇连贯的分析文，不是拼凑的要点列表
- 观点归因规则见 [引用来源管理.md](引用来源管理.md)

## 语言风格

| 区域 | 风格 |
|------|------|
| 01 重磅资讯 | 客观、中性、无修饰，原文摘录 |
| 02 数据复盘 | 配图建议，标注来源 |
| 03 盘面解读 | 专业、简洁，因果链清晰 |
| 04 后市展望 | 可有立场、可体现人设风格、需逻辑支撑 |

通用：
- 专业但不晦涩，读者是有一定基础的散户
- 多用短句，段落不超过 5 行
- 数据说话，每个判断都有数据/事实支撑
- 03/04 可体现人设风格和个性（从 SOUL.md 出发），01/02 保持客观中性
- **全角标点**：正文中所有标点必须使用中文全角（双引号用\u201c\u201d，逗号用，，句号用。，冒号用：，括号用（）），基金代码等纯数字场景除外
