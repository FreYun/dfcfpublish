# 每日盘面复盘（daily-market-recap）

> 交易日每日一篇，固定结构，建立读者"每天来看一眼"的习惯。

## ⛔ 铁律（全程有效）

1. **无来源不落笔** — 正文中的每一个数字、每一条事实、每一个观点都必须有明确的、可追溯的来源。找不到来源的内容一律不写，没有例外
2. **绝对禁止编造** — 没有读到的券商报告，不许写"XX证券认为..."；没有拿到的行情数据，不许凭记忆填写
3. **数据获取失败 → 停下来报告** — 不许在缺少关键数据的情况下继续写作，向研究部说明失败情况，等待指示
4. **全角标点** — 正文中所有标点必须中文全角（，。：""（）！？），基金代码等纯数字场景除外

## 分步工作流（⛔ 不要一次读完全部文件，按步骤走）

收到写复盘的指令后，**按以下 4 步依次执行**。每一步开头标注了"现在读"的文件，到该步骤时再读，不要提前全部加载。

---

### Step 1：写 01 热点大事 + 02 数据复盘

**现在读：** `Read skills/daily-market-recap/每日复盘-写作指南.md` + `Read skills/daily-market-recap/行情数据获取.md`

> ⛔ **01 铁律（本步最易违反）**：01 区域的**每一句话**都必须从研究部提供的链接原文中摘录，**禁止自己生成、概括、改写、意译**。标题用研究部原标题。选哪段你来定，但选了就不许改。
>
> ⛔ **02 铁律**：02 区域写（一）（二）（三）三段固定结构的数据描述，**数据必须从 research-mcp 或浏览器获取，不许凭记忆填**。图片由研究部后期人工插入，你只写文字。

**执行：**
1. 用 `fetch_article.py` 批量读取研究部给的新闻链接（⭐ 比浏览器 snapshot 节省 98% 上下文）：
   ```bash
   python skills/daily-market-recap/fetch_article.py "链接1" "链接2" "链接3" "链接4"
   ```
   如果某条链接抓取失败（微信公众号偶尔被拦截），再用浏览器访问该条链接。
2. 从每条新闻中摘取至多 2 段原文（不改一个字），写出 01
3. 获取当日行情数据（按行情数据获取.md 流程），写出 02 三段数据描述
4. 写完 01 和 02 后，进入 Step 2

---

### Step 2：搜集 03/04 素材（可并行）

**现在读：** `Read skills/daily-market-recap/行情数据获取.md` + `Read skills/daily-market-recap/券商观点获取.md`

**执行：**
1. **取行情数据**：按行情数据获取.md 的流程，用 research-mcp 或浏览器获取当日指数、行业涨跌等数据
2. **搜券商观点**：用 `research-mcp.research_search` 搜索当日券商策略收评、后市展望（返回 `[研报ID, 摘要, 券商名, score]`，券商名可直接使用）
3. 整理素材草稿，标注来源，准备用于 03/04 写作
4. 素材搜集完成后，进入 Step 3

---

### Step 3：写 03 盘面解读 + 04 后市展望

**现在读：** `Read skills/daily-market-recap/引用来源管理.md`

> ⛔ **归因铁律（本步最易违反）**：03/04 正文中必须**在文中明确写出券商名归因**（"长江证券指出……""中金公司认为……"），至少引用 2-3 家券商。不能只在文末来源列表里提，正文里不写。
>
> ⛔ **无来源不落笔**：没有来源的观点、数字、事实一律不写进正文。

**执行：**
1. **写 03 盘面解读**（400-600 字）：融入 Step 2 搜集的券商分析，解释"市场为什么这样走"
2. **现在读：** `Read skills/daily-market-recap/基金推荐规则.md`
3. **确定 2-3 个重点关注板块**，然后 **为每个板块分别调用基金筛选脚本**：
   ```bash
   python skills/daily-market-recap/fund_selector_llm.py --topic 科技
   python skills/daily-market-recap/fund_selector_llm.py --topic 新能源
   python skills/daily-market-recap/fund_selector_llm.py --topic 金融
   ```
   ⛔ 不要自己手选基金，不要绕过脚本直接读 CSV/XLSX

4. **⛔ 门控检查点 — 写 04 之前必须先输出以下确认清单：**

   ```
   【04 写作前确认】
   重点板块 1：[板块名] → fund_selector_llm.py --topic [X] → ✅ 已执行
   重点板块 2：[板块名] → fund_selector_llm.py --topic [Y] → ✅ 已执行
   重点板块 3：[板块名] → fund_selector_llm.py --topic [Z] → ✅ 已执行
   券商观点：已搜集 [N] 家券商观点 → ✅
   ```

   全部 ✅ 后才能开始写 04。有任何一项未完成，先补做再继续。

5. **写 04 后市展望**（800-1200 字）：融入券商观点 + 基金推荐结果，按标准输出模板的结构展开（**短期技术面判断** → **外部环境** → **内部环境** → **重点关注方向** → **题材与风控提示** → 风险提示）
6. 写完后进入 Step 4

---

### Step 4：自检 + fact-check + 存档 + 呈审

> ⛔ **本步不可跳过。** 之前出过事故：bot 写完就直接提交，跳过了 fact-check。

**执行：**
1. **自检**：按写作指南末尾的自检清单逐条检查并**明文输出自检报告**（有 ❌ 的先改稿再继续）
   > ⛔ 自检完成 ≠ 写稿完成。不要在自检后就提交研究部——**继续做下面的 fact-check。**
2. **现在读：** `Read skills/fact-check/SKILL.md`
3. **fact-check**：按 fact-check skill 的流程，逐条核实文中所有数字和事实性表述，输出核查报告
4. **⛔ 核查发现偏差 → 必须改稿**：不能只输出核查报告就算完。流程是：输出核查报告 → 根据报告中的 ❌ 项逐条修改稿件 → 修改完毕后重新核查 → 直到全部 ✅。**只出报告不改稿 = 未完成核查**
5. **存档**：改稿后的终稿 + 来源说明 + 自检报告 + fact-check 核查报告，一起写入 `memory/daily-recap/YYYY-MM-DD.md`
6. **交稿**：提交研究部审阅

## 网页访问规范

### 新闻链接：用 fetch_article.py（⭐ 首选）

研究部给的新闻链接（东方财富、财联社、微信公众号等）**用 `fetch_article.py` 读取**，不用浏览器。

```bash
python skills/daily-market-recap/fetch_article.py "链接1" "链接2" "链接3"
```

- 支持：东方财富、微信公众号、财联社、证券时报、新华社、央视新闻等
- 每条新闻约 1-3K 字符，比浏览器 snapshot（50-75K/页）**节省 98% 上下文**
- 如果某条链接返回"请改用浏览器访问"，再用浏览器打开那一条

### 行情/动态页面：用浏览器

以下场景用浏览器（fetch 无法获取动态渲染内容）：
- **东方财富首页行情数据**
- **申万指数页面**
- **fetch_article.py 失败的 fallback**

浏览器规则：
- **必须使用自己的 profile 浏览器**（bot9 专属 profile：`/home/rooot/.xhs-profiles/bot9/`）
- **禁止擅自启动有头浏览器**（headed browser / `-headless=false`），未经研究部同意不得使用
- 读完后**必须关闭浏览器**

```
browser_navigate(url: "链接")
browser_snapshot()
browser_close()   ← 读完必须关闭
```

## 研报引用规则

03 盘面解读和 04 后市展望中如需引用研报、券商观点等资讯：

- **当日研报优先**：有当天发布的研报/观点，优先采用，时效性最强
- **历史研报须严格甄别**：引用非当日研报时，必须判断其核心观点是否已被后续行情或政策变化推翻。如果市场环境、数据面已发生实质性变化，该观点视为过时，**禁止引用**
- **过时判断标准**：研报的关键假设（如利率走向、供需格局、政策预期）与当前实际情况不符，即为过时
- 引用时注明研报机构和日期，让读者自行判断时效

## 合规红线

- **不给操作建议** — 不说"建议买入/卖出""建议抄底""建议减仓""不建议追涨"等任何指导读者具体操作的话
- 不说"配置方向""建议配置"，用"重点关注""看好方向""值得留意"
- 不说"必涨/必跌/稳赚"，用"存在机会/需要警惕"
- **04 后市展望不提个股代码/名称**，只说板块/产业链（基金代码/名称除外）。01 原文摘录、03 盘面分析可提个股名称但不带代码
- 不做收益承诺，不炫耀具体盈亏数字
- **不提竞品** — 正文中禁止出现同花顺、Wind/万得、iFinD 等东方财富竞品的名称、产品或数据标注
- **东方财富不是新闻源** — 禁止出现"据东方财富报道""东方财富指出"等将东方财富当作报道方/观点来源方的表述。东方财富是数据聚合平台，没有新闻采编资质；正文引用时只许写实际的报道方或观点来源方（如"据中国证券报报道""中信证券认为"）
- 文末必加风险提示

## 基金池月度更新

基金池 xlsx 文件（`YYYYMM-权益基金池.xlsx`、`YYYYMM-指数基金池.xlsx`）每月初由天天基金研究部更新。

**检查是否需要更新**：看文件名前缀是否为当前月份。如果已过期（如当前 5 月但文件还是 `202604-`），需要更新。

**更新方法**：运行自动下载脚本，`--bot` 传入你自己的 bot ID：

```bash
python skills/daily-market-recap/download_fund_pools.py --bot <你的botID>
```

- 脚本自动从腾讯文档分享链接下载当月的两个 xlsx 文件，保存到 skill 目录，清理旧文件
- 如果腾讯文档登录已过期，脚本会截取微信登录二维码通过飞书发给研究部，等待扫码后继续
- 手动触发登录：加 `--login` 参数

**研究部说「更新基金池」时**，立即运行上述脚本。
