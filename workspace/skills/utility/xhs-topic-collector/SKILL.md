# 小红书素材积累

定时巡逻小红书与外部信息源，主动收集选题灵感写入素材库。**弹药库不是博物馆 —— 攒 30 发精弹，不是攒 300 发哑炮**。每次巡逻的核心动作是：**先清再补**，保持素材库随时处于"小而精"的状态。

---
## Step 1: 确定巡逻方向

每次巡逻前，先读身份文件确定近期关注方向：

1. **SOUL.md** — 人设定位、擅长领域、兴趣点
2. **MEMORY.md** — 最近在关注什么、在做什么

从中提取本次巡逻的**关键词列表**（3-5 个），根据人设方向动态调整。

---

## Step 2: 多渠道巡逻

根据关注方向，从以下渠道**并行**收集素材（不是每个都要，选相关的）：

### 2.1 小红书平台

```bash
# 刷首页，看平台在推什么
npx mcporter call xiaohongshu-mcp.list_feeds

# 按关键词搜索热门内容（每个关键词搜一次）
npx mcporter call "xiaohongshu-mcp.search_feeds(keyword: '<关键词>', filters: {sort_by: '最多点赞', publish_time: '一周内'})"

# 看自己笔记的评论区，粉丝在问什么
npx mcporter call xiaohongshu-mcp.get_notification_comments

# 看自己发过的笔记，哪些方向互动好
npx mcporter call xiaohongshu-mcp.get_self_notes
```

关注点：
- 爆款笔记的**选题角度**（不是抄内容，是学角度）
- 热搜话题中与自身人设能聊的**交集**
- 评论区高频出现的**问题和困惑**
- 自己互动数据好的笔记 → 这个方向值得继续挖

### 2.2 外部信息源

根据当前兴趣，浏览相关信息源：
- research-mcp.news_search搜索每个关键词查最近 1 天 top 5 条新闻
- agent 自身工具输出（持仓变动、跟踪任务、数据报告等）
- 日常浏览中偶然发现的有趣内容

关注点：
- 能从**自身人设视角**解读的热点事件
- 反直觉的数据或结论
- 争议性话题（适合引发讨论）
- 有趣的、能引发共鸣的生活观察

### 2.3 时间维度

- 季节/节气/节日 — 即将到来的时令话题
- 周期性事件 — 月度数据发布、财报季、政策窗口、产品发布会等
- 纪念日/周年 — 可以做回顾盘点的时间节点

## Step 3: 筛选与记录

### 筛选标准

不是看到什么都记，要过一遍筛选：

| 标准 | 说明 |
|------|------|
| 人设相关 + 有角度 | 符合定位，且能明确"我从什么视角聊" |
| 参考雪球热搜榜 | 关乎帖子发出后的热度 |
| 有信息量 | 能写出 200+ 字实质内容，不是水帖 |
| 不重复 | 素材库中没有相似未用素材 |
| 时效性判断 | 标注是时效性素材（尽快用）还是常青素材（随时可用）|

### 写入格式

向 `memory/topic-library.md` 的**灵感池**区域追加：

```markdown
- 💡 <选题简述> | 来源：<在哪看到的>（👍N 💬N ⭐N） | 日期：YYYY-MM-DD
  > 角度：从什么视角切入，为什么值得写
  > 时效：🔥 时效性（X天内要用）/ 🌲 常青（随时可用）
```

> **互动数据**：XHS 来源的素材，在来源括号内记录采集时的互动数据（👍点赞 💬评论 ⭐收藏），作为选题热度参考。非 XHS 来源可省略。

### 写入示例

```markdown
- 💡 "年轻人第一次理财踩过的坑"吐槽贴 | 来源：XHS 热帖评论区（👍1.2万 💬356 ⭐890） | 日期：2026-03-13
  > 角度：以过来人身份吐槽常见理财误区，轻松科普
  > 时效：🌲 常青（随时可用）

- 💡 某新出的效率工具体验分享 | 来源：知乎热议 | 日期：2026-03-13
  > 角度：亲测后写真实体验，不吹不黑
  > 时效：🔥 时效性（话题热度 1 周内）
```

## Step 4: 素材库维护（每次巡逻必做）

> ⚠️ **素材库是弹药库，不是博物馆**。新增素材之前先清理旧素材。维护不是可选项，不清理 = 这次巡逻没做完。

### 4.0 硬性上限检查（第一步，动手写新素材前做）

```bash
# 查当前总条数（💡 开头的活素材）
grep -c "^- 💡" /home/rooot/.openclaw/workspace-<self>/memory/topic-library.md
```

**铁律**：
- 活素材总数必须 **≤ 30 条**
- 文件总行数应 **≤ 300 行**，超过说明历史垃圾积压

**超标处置**：
1. > 30 条 → 先执行 4.1 清理过期，再执行 4.2 按价值裁剪
2. 仍 > 30 → 保留最有用的 30 条（按"剩余时效 × 人设契合度 × 独特性"排序），其余全删
3. 文件 > 300 行 → **一次性整档重写**：只保留 frontmatter + 活素材清单，其他所有历史 section、log、归档、发布记录全删

### 4.1 清理过期素材（真删除，不是 strikethrough）

用 **Edit 工具物理删除整行或整段**，让被清理的内容在文件里消失。

**什么算过期**：
- 🔥 时效性已过期（超出条目标注的"X天内"窗口）
- > 10 天未用且没有独特角度
- 已被某个帖子消化掉（bot 自己或其他 bot 都算）
- 同一角度出现了更新鲜/更具体的替代素材

**❌ 禁止的"假删除"**（这些都不算清理）：

| 错误做法 | 为什么不行 |
|---------|-----------|
| 改写成 `- ✅ ~~标题~~ \| 已消耗` 保留在原位 | 下次巡逻还会读进 context，放大 sys4 排期的 token 成本 |
| 把时效 section 标题改成 `## ~~🔥 时效性素材（已过期清理 MM-DD）~~` 留空壳 | 文件变成考古地层，每天新增一层空 section |
| 新建 `## 已用选题归档` / `## 最近更新` / `## 发布记录` section | topic-library.md 只放活素材，不是日志、不是归档、不是发布登记簿 |
| 保留"清理说明"的注释行（如 `<!-- 4/12 清理了5条 -->`） | 审计痕迹由 git 和 bulletin.jsonl 保管，文件本身不留证据 |

**为什么必须真删除**：
- 历史由 `git log` 保管，文件本身不需要自证清白
- 素材库是**下次巡逻和排期时的工作文档**，每一行都要进 sys4 的 context，冗余直接放大 token 成本
- strikethrough 条目下次巡逻你会再看一遍，浪费注意力
- 库存越精神，选出的好素材越突出 —— 堆满垃圾会让真金淹没在沙里

### 4.2 补充常备选题

清理完再看，如果发现某个内容分类的常备选题快用完了（≤ 2 条），主动补充一批（3-5 条）。**先清理再补充，不是反过来**。

### 4.3 更新分类

如果内容方向有变化（发现某个新方向效果特别好），直接修改现有分类标题，不要新建平行分类。同名分类只允许有一份。

### 4.4 清理后文件应该长什么样

**干净的 topic-library.md 参考结构**（理想状态 ≤30 条素材、≤200 行）：

```markdown
# 素材灵感池

> 最后更新：YYYY-MM-DD HH:MM
> 当前 N 条活素材（上限 30）

---

## 🔥 时效性素材

- 💡 <选题 A> | 来源：... | 日期：...
  > 角度：...
  > 时效：🔥 时效性（X天内）

- 💡 <选题 B> | ...

---

## 🌲 常青素材

- 💡 <选题 C> | ...

- 💡 <选题 D> | ...
```

**不该出现的 section**（看到立刻删）：
- `## 最近更新` / `## 巡逻日志`
- `## 已用选题归档` / `## 已消耗素材`
- `## 近期发布记录` / `## 发布状态表`
- `## ~~...~~`（任何 strikethrough 的 section 标题）
- 按日期分的 section（`## 2026-04-12 素材收集`），日期信息写在每条素材的 `日期：` 字段里就够了

## Step 5: 巡逻报告（写入汇报清单，不写进 topic-library）

每次巡逻完成后，如果有新增或清理动作，**只追加一行到 bulletin.jsonl**，魏忠贤会每2小时统一播报到飞书管理群。**不要在 topic-library.md 里留任何"巡逻完成"或"素材清理完成"的日志**。

```bash
echo '{"ts":"'$(date -Iseconds)'","from":"<bot名字>","title":"素材巡逻完成","content":"新增X条灵感，清理Y条过期。当前共N条活素材（上限30）。","priority":"normal"}' >> /home/rooot/.openclaw/cron/bulletin.jsonl
```

> - 如果本次巡逻没有发现值得记录的素材也没有可清理的，不写入 bulletin
> - 审计痕迹**只走 bulletin 和 git**，topic-library.md 本身不留任何工作日志、归档 section、发布记录
> - 管理员能在 bulletin 里看到"你巡逻了"，git log 里看到"你删了什么"，这两条审计链已经足够

## 积累原则

### 采集阶段（巡逻中）

1. **记角度不只记话题** — 不只记"写什么"，更记"怎么写"和"为什么写"
2. **标注时效性** — 区分必须尽快用的热点素材和随时可用的常青素材
3. **不限方向** — 只要符合人设、聊得有意思的都可以记
4. **质量 > 数量** — 每次巡逻记 2-5 条高质量素材，比记 10 条水素材强

### 存储阶段（写入前/清理时）

5. **硬性上限 30 条** — 巡逻一开始先查总条数，超过必须先清再加，不许用归档/strikethrough 规避
6. **真删除，不留痕** — 过期素材用 Edit 工具物理删除，不 strikethrough、不改"已消耗"、不建归档 section
7. **文件只放活素材** — topic-library.md 不是日志、不是归档、不是发布登记簿。想留记录走 bulletin.jsonl 和 git
8. **少即是多** — 库存越精神，排期时越容易挑出好选题。20 条好素材 > 50 条半死素材
9. **先清再补** — 任何一次巡逻的正确顺序是：检查上限 → 清理过期 → 评估缺口 → 采集新素材 → 补写。不准反过来
