# 基金推荐规则

## 触发条件（默认必须带）

**写热点板块分析时，基金推荐是必选项，不是可选项。** 在撰写模块 4 之前，必须先调用基金筛选脚本拿结果。

⛔ **禁止写完文章后再回头补基金。** 基金推荐必须在初稿中就包含。如果提交的稿件没有基金推荐，视为未完成。

---

## 基金来源

基金筛选现在统一通过本地 Python 脚本完成，不再手工读取基金池文件挑选。

优先使用：

```bash
python /home/rooot/.openclaw/workspace-bot9/skills/daily-market-recap/fund_selector_llm.py --topics 科技,新能源,医药 --limit 5
```

如果只有单个主题：

```bash

```

调试时可加：

```bash
python /home/rooot/.openclaw/workspace-bot9/skills/daily-market-recap/fund_selector_llm.py --topic 创新药 --limit 5 --debug
```

脚本内部会自己读取本地底库：

- `skills/daily-market-recap/东财基金.csv`
- `skills/daily-market-recap/202604-指数基金池.xlsx`
- `skills/daily-market-recap/202604-权益基金池.xlsx`

## 正确流程

主 bot / agent 在这里的正确动作是：

1. 先确定 2-3 个重点关注方向或主题
2. 每个方向或主题需要5只基金
3. 调用 `fund_selector_llm.py`（注意加 `--limit 5`）
4. 等脚本返回 JSON 结果（脚本内部有 LLM 调用，可能需要 60 秒以上，**耐心等待，不要因为慢就放弃**）
5. 根据返回结果写基金推荐文案

**不要手工翻基金池，不要用任何其他方式筛基金。** 这里的基金筛选逻辑和 [daily-market-recap/基金筛选接口.md](../daily-market-recap/基金筛选接口.md) 保持一致。

---

## 内部规则

脚本内部不是让 agent 手工挑基金，而是分两步：

1. **代码召回候选**：
   - 从 3 个本地基金文件中召回候选
   - 自动分层、去重、控制候选规模
2. **模型最终筛选**：
   - 调用大模型理解主题/板块
   - 根据优先级和补位规则选出最终基金

所以 agent 在这里的正确动作是：
- **运行脚本**
- **读取输出**
- **根据输出继续写稿**

而不是自己在表格里翻基金。

## 展示格式

**基金推荐集中在模块 4，独立成段展示：**

```
下面也整理了部分XX相关的基金供参考：

xxx基金A（012345，基金经理：张三）
xxx基金B（012346，基金经理：李四）
xxx基金C（012347，基金经理：王五）
xxx基金D（012348，基金经理：赵六）
xxx基金E（012349，基金经理：钱七）

（数据来源：东方财富Choice数据、银河证券，具体基金的相关风险、基金经理变更等
情况详见各基金法律文件及公告，历史数据不代表未来，不作投资推荐）
```

**格式要求：**
- 每只基金独占一行
- 展示：基金名称（代码，基金经理：姓名）
- 基金代码用半角括号（纯数字场景）
- 语气用"供参考""可以关注"，不用"建议买入""推荐"
- **数据来源声明必加**（放在基金列表下方）

## ⛔ 异常处理（脚本超时/报错时的唯一流程）

**铁律：基金筛选只有一条路——跑 `fund_selector_llm.py` 脚本。没有备选方案，没有手工筛选，没有"根据经验推荐"。**

### 脚本超时或报错时：

1. **第一次失败 → 立即重试**：原样再跑一次脚本（加 `--debug` 看报错原因）
2. **第二次仍然失败 → 向研究部报告**，说明错误信息，等待指示
3. **等待期间不要继续写稿**——模块 4 基金推荐是必选项，没有基金推荐的稿件视为未完成

### 脚本返回空结果时：

1. 检查 `--topic` 参数是否准确匹配板块名称，尝试调整关键词后重试
2. 仍然为空 → 向研究部报告，等待指示

### ⛔ 以下做法全部禁止：

- ❌ 手工读取 CSV/XLSX 文件挑基金
- ❌ 凭记忆或经验推荐基金
- ❌ 用搜索工具搜基金然后自己选
- ❌ 从之前的文章中复制基金列表
- ❌ 因为脚本慢/超时就跳过基金推荐先交稿
- ❌ 任何不经过 `fund_selector_llm.py` 的基金筛选方式

**一句话：脚本跑不出来就重试，重试不行就报告研究部，绝不自己想办法。**
