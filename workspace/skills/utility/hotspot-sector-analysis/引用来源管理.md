# 引用来源管理

> ⛔ **无来源不落笔**：正文中的每一个数字、每一条事实、每一个观点都必须有明确的、可追溯的来源。没有来源的内容一律不许写进正文——找不到来源就不写，没有例外。

## 观点归因规则

- **有对应机构和人的观点，正文中应注明机构/人员来源**：如果引用的观点能追溯到具体机构和具体人，用”XX证券XX认为””XX基金XX表示”等方式归因。这增加了观点的可信度和专业感
- **归因前必须确认**：只有当你**确定**该观点确实出自该机构的该人时，才可归因。如果只是”大致记得”或”可能是”，宁可不归因，用自己的话表达；如果只是在文中出现了xxx机构，但是实际观点与该机构无关，**不许乱写引用！**
- **归因格式**：简洁自然，嵌入分析段落中。例如：”中信建投陈果认为，当前市场处于……[2]””国泰君安策略团队指出……[3]”
- **禁止使用”明确表示””明确指出”等措辞**：不要写”XX证券明确表示””XX机构明确指出”，用”认为””表示””指出””分析”即可，不加”明确”二字
- **正文只保留必要归因，不放完整出处**：正文里可以写机构名、团队名、研究员名，必要时可带日期；但不要把长链接、完整标题、完整出处说明塞进正文
- **不确定来源的观点一律不用** — 如果一条观点无法追溯到具体机构/具体人，不许写进正文，也不许用自己的话”转述”。正确做法是**换一家有明确来源的机构观点**，宁可少引一家也不引来源不明的

### ⛔ 竞品与东方财富归因铁律

- **正文禁止出现竞品名称**：同花顺、Wind/万得、iFinD 等东方财富竞品的名称、产品或数据标注一律不许出现在正文中
- **东方财富不是新闻源/观点来源方**：东方财富是数据聚合平台，没有新闻采编资质。正文中禁止出现”据东方财富报道””东方财富指出””东方财富分析”等将东方财富当作报道方或观点来源方的表述。正文引用时只许写实际的报道方（如”据中国证券报报道”）或观点来源方（如”中信证券认为”）
- **文末来源说明中可以出现东方财富作为链接宿主**：如 `东方财富《转载标题》` + 链接，这是注明数据获取渠道，不是将其当作报道方
- **东方财富转载的文章归因给原始来源**：如果东方财富转载了中国证券报的文章，正文写”据中国证券报报道……[n]”，[n] 的来源说明可以写”东方财富转载 中国证券报《标题》”；但如果文章是东方财富自产的，没有原始报道方，则不得在正文中作为新闻源引用

### ⛔ 最常见的错误：凭空编造归因（反模式警告）

**这是出错率最高的问题，必须杜绝。**

**核心铁律：`[n]` 指向的文章/研报里，必须实际包含你归因的那个观点。如果 [n] 的原文里根本没有提到该机构或该观点，就是凭空编造，属于虚假归因。**

**错误做法（绝对禁止）**：
- 正文写”中信证券认为半导体景气度向上……[1]” → 但 [1] 指向的东方财富文章里根本没提到中信证券 → **这是凭空编造归因**
- 正文写”据中国证券报报道……[6]” → 但 [6] 指向的文章跟中国证券报毫无关系 → **这是张冠李戴**

**正确做法**：
1. **有原始研报**：用 `research_search` 找到机构的原始研报 → 正文写”XX证券认为……[n]”，[n] 指向该研报 PDF 链接
2. **转载/引述文章中提到了机构观点**：如果东方财富等聚合网站的文章中确实转述了”XX证券认为……”，可以归因给 XX证券，但 [n] 指向该聚合文章，来源说明中如实标注来源（如”东方财富《标题》”），正文写”XX证券认为……[n]”
3. **[n] 原文中根本没有该机构/该观点** → **绝对不许归因**，这不是转载问题，是编造问题

**自检方法：写完后，对每个 `[n]`，回溯检查——[n] 指向的原文中是否真的包含你所归因的机构名称和观点？如果原文里压根没提到该机构，立即删除该归因。**

## 新闻类引用铁律

- **新闻报道类内容必须写出新闻源**：正文引用新闻时，必须写成“据新华社报道……[1]”“中国证券报称……[2]”“央视新闻提到……[3]”这类格式，不能只写“有媒体称”“消息面显示”
- **没有新闻源，不得援引新闻**：如果搜索结果只有摘要、没有明确 `source_name/publisher`，或无法确认出处，允许作为内部检索线索继续找原文，但**禁止**直接进入成稿正文和来源清单；如果只是在文中出现了xxx新闻源，但是实际新闻与该机构无关，**不许乱写引用！**
- **新闻源归因不得张冠李戴**：正文写"据中国证券报报道……[n]"，则 [n] 指向的文章中必须确实是中国证券报发布或转载的内容。如果 [n] 的原文跟中国证券报毫无关系，就是凭空编造新闻源。东方财富转载中国证券报的文章，可以归因给中国证券报；但东方财富自己写的文章，不能说是中国证券报的
- **优先保留结构化字段**：采集素材时尽量保留 `raw_id`、标题、摘要、来源名、发布日期、完整 URL；如果是东方财富文章 ID，在确认规则成立时补成完整链接，方便文末核查
- **正文允许 `[n]` 角标，不允许裸链接**：`[1]`、`[2]` 属于引用标号，不算 URL；正文中仍然不能直接贴 `https://...`

## research-mcp 两个接口的返回字段差异

| 接口 | 返回结构 | 说明 |
|------|---------|------|
| `research_search` | `[研报ID, 摘要, 券商名, score]` | **已含券商名**，可直接用于正文归因；但不含研报标题和研究员，需 pdftotext 确认 |
| `news_search` | `[raw_id, 摘要, score]` | **不含来源名/报道方**，必须 web_fetch 打开链接确认实际报道方，禁止从摘要猜测 |

## API 返回 ID → 可访问 URL 的还原规则

research-mcp 和 news 接口返回的数据中包含 ID 字段，需要还原为可访问的 URL 才能放进来源说明。

### 研报类（research_search 等接口）

返回的 ID 形如 `AP202604071821041599`，还原规则：

```
https://pdf.dfcfw.com/pdf/H3_{ID}_1.pdf
```

示例：`AP202604061821031052` → `https://pdf.dfcfw.com/pdf/H3_AP202604061821031052_1.pdf`

### 新闻类（news_search 等接口）

返回的 ID 形如 `202604083696931327`，还原规则：

```
https://fund.eastmoney.com/a/{ID}.html
```

示例：`202604083696931327` → `https://fund.eastmoney.com/a/202604083696931327.html`

### 使用要求

- 从 research-mcp 获取到的研报/新闻数据，**必须按上述规则还原出完整 URL**
- 还原后的 URL 放入文末【来源说明】中，作为引用追溯依据
- 正文中不放链接，链接只出现在来源说明里

### ⛔ URL 必须实际下载并验证（铁律，违反 = 全文作废）

> 背景：2026-04-13 bot9 热点文章事故——bot9 把一条 AP ID 按公式拼成 dfcfw URL 写进来源说明，标注为"中银证券《国产算力实现突破》"。事后下载该 PDF 发现**首页署名是"头豹研究院"**，与中银证券毫无关系；另外两条 URL 下载回来是 33 字节的 `{"msg":"鉴权失败","code":403}` JSON，浏览器根本打不开。fact-check 还给这三条全打了 ✅，因为它只看"观点是否存在于某处"，没验证"URL 背后到底是什么"。

**硬约束**（每条还原出来的 URL 都必须走完这套流程，缺一项直接删掉该引用）：

1. **必须用 curl 下载** —— 不用 HEAD、不用 browser。HEAD 对 JSON 鉴权报错也返回 200，browser 的 headless Chrome 无法渲染 PDF，都骗得过你：
   ```bash
   mkdir -p /tmp/src-check
   curl -sSL "{还原后的URL}" -o /tmp/src-check/{ID}.pdf --max-time 30
   ```

2. **必须检查文件大小和 mime 类型** —— 小于 10KB 的"PDF"基本都是鉴权报错 JSON（典型 33 字节 `{"msg":"鉴权失败","code":403}`）：
   ```bash
   size=$(stat -c%s /tmp/src-check/{ID}.pdf)
   mime=$(file -b /tmp/src-check/{ID}.pdf)
   # size < 10240 或 mime 不含 "PDF document" → 验证失败，删除该引用
   ```

3. **研报必须用 pdftotext 读首页验证机构署名** —— 光 URL 能下还不够，PDF 首页署名的机构必须和你在正文里归因的机构**完全一致**：
   ```bash
   pdftotext -f 1 -l 2 -layout /tmp/src-check/{ID}.pdf - | head -30
   ```
   比如正文写"中银证券认为……"，首页抬头必须能看到"中银证券"或"中银国际证券"；如果首页抬头是"头豹研究院"、"华泰证券"或任何其他机构，**不许归因给中银证券**，要么改正文归因方、要么整段删除。

4. **新闻必须用 curl + grep 验证报道方** —— `fund.eastmoney.com/a/*.html` 页面 HTML 里通常有 `来源：xxx` 字段：
   ```bash
   curl -sSL "{还原后的URL}" --max-time 20 | grep -oE '来源[：:]\s*[^<]{2,20}' | head -3
   ```
   文中写"据中国证券报报道"，这里必须能抓到"来源：中国证券报"；抓到的是别的报道方就按实际报道方改，抓不到就删。

5. **验证完必须清理** `/tmp/src-check`，不得遗留 PDF。

**失败处理**：任何一步验证不通过（URL 打不开、是 JSON 报错、首页机构对不上、新闻源对不上），**立即从正文和来源说明中删除该引用**，不许保留、不许"先写着之后再补"、不许用摘要敷衍过去。如果删完之后正文某段论据不足，宁可整段删掉也不留带病上线的归因——这是 skill 第 9 行 "⛔ 无来源不落笔" 铁律的延伸执行。

## 来源说明（文末附上）

如果写作过程中参考了研报、券商观点、分析师观点、新闻资讯、行情数据、基金筛选结果等，**每一条都必须在正文后的“来源说明”中列出，不许省略，并尽量附上对应链接**。

格式如下（**每条来源之间必须空一行**，否则飞书渲染时会挤在一起）：

```
【来源说明】

[1] 新华社《报道标题》，2026-04-07
链接：https://...

[2] XX证券《报告标题》，2026-04-07
链接：https://...

[3] 东方财富 Choice 数据，2026-04-07
链接：https://...

[4] fund_selector_llm.py 基金筛选结果（本地脚本输出）
```

⛔ **格式铁律**：每个 `[n]` 条目之间必须有一个空行。链接紧跟来源描述下方，不缩进。这样在飞书消息中才能正确换行显示。

- 这段“来源说明”放在**发给研究部审核的正文后面**，用于统一追溯全文引用依据
- **正文里每个 `[n]` 都必须在文末找到唯一对应来源**，编号不得错位、跳号或一对多
- **同一来源可复用同一个编号**，不要为同一链接重复创建多个编号
- **如果正文提到了某家机构的观点，文末来源说明里必须能找到对应那条来源和链接**
- **如果正文引用了新闻事实，正文必须写出新闻源，文末必须附上该新闻源对应链接**
- **链接统一放在文末来源说明，不要放正文里**
- 如果全文没有引用任何外部观点/数据，则可以不附这一段
