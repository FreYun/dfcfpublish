Step 1 素材搜集

  • 前置：执行 date +%Y%m%d 确认日期
  • 线路 A：Tushare API 取板块指数 + 个股行情
  • 线路 B：research-mcp news_search 搜 24h 资讯
  • 线路 C：research-mcp research_search 搜机构观点
  • 校验：items 非空 + trade_date = 今日
  • 红线：news_search 不返回来源名，必须 web_fetch 确认

Step 2 四模块写作

  • 模块 1：行情播报（≤150 字，纯事实）
  • 模块 2：深度分析（500-600 字，2-4 个原因）
  • 模块 3：机构观点（400-500 字，3-5 家机构）
  • 模块 4：相关基金（fund_selector_llm.py 脚本）
  • 红线：机构归因必须 [n] 角标 + 正文点名

Step 3 质量自检

  • 输入：完整稿件（四模块 + 来源说明）
  • 流程：逐条检查 20+ 项，明文输出报告
  • 覆盖：数据准确 / 归因真实 / 格式合规 / 来源完整
  • 红线：有 ❌ 先改稿再交，不许带病提交
  • 注意：自检完成 ≠ 写稿完成，必须继续 Step 4

Step 4 Fact-Check + 呈审

  • 核查：逐条核实文中所有数字和事实性表述
  • 工具：research-mcp / web_fetch / pdftotext
  • 改稿：核查有 ❌ → 逐条改 → 重新核查 → 全 ✅
  • 存档：memory/hotspot-sector/YYYY-MM-DD-板块名.md
  • 红线：每条 ✅ 必须有对应工具调用，禁止伪造

最终输出产物

产物 来源步骤 内容说明 质量红线
标题 Step2 ≤30 字，核心信息前置，突出板块异动 有冲击力但不标题党，禁止"必涨/稳赚"
模块 1 行情播报 Step2 ≤150 字，板块指数涨跌幅 + 龙头个股 + 港股联动 + 配图占位 纯事实 不加"利好""利空"定性词
模块 2 深度分析 Step2 500-600 字，回答"为什么涨/跌"，2-4 个核心原因 + 数据支撑 有据可查 新闻引用必须点明新闻源 + [n]
模块 3 机构观点 Step2 400-500 字，3-5 家权威机构研判，正文写机构名 + [n] 角标 归因真实 [n] 原文必须包含归因观点
模块 4 相关基金 Step2 5 只板块相关权益类基金，调用 fund_selector_llm.py 筛选 禁止手工选基金，脚本失败→重试→报告
风险提示 Step2 "市场有风险，投资需谨慎" 固定文案 必加，不可省略
来源说明 Step2 文末附所有 [n] 引用来源 + 链接，每条间空一行 每个 [n] 都有唯一对应来源，URL 必须下载验证
自检报告 Step3 逐条 ✅/❌ 检查，覆盖行情/分析/观点/基金/合规 5 大类 有 ❌ 必须先改稿再交
Fact-Check 报告 Step4 逐条核实文中数字/事实/归因，标注验证方式 每条 ✅ 必须有对应工具调用记录
存档文件 Step4 终稿 + 来源说明 + 自检报告 + 核查报告，存入 memory/ 覆盖同日同板块文件

关键设计点

四模块固定结构 行情播报→深度分析→机构观点→相关基金，事实区（模块1）与分析区（模块2-4）分离，约 1200 字
三路素材并行搜集 Tushare 行情数据 + news_search 24h 资讯 + research_search 机构观点，全部搜集完再动笔
双重质控闭环 Step 3 自检清单 + Step 4 fact-check 两道关卡，任一不通过都要改稿循环直到全 ✅
来源全链路验证 research-mcp 搜素材 → curl 下载 URL → pdftotext/grep 验证署名和来源 → 失败即删引用
基金推荐自动化 fund_selector_llm.py 脚本筛选，禁止人工翻表、凭记忆推荐，脚本失败→重试→报告研究部
合规硬编码 不给操作建议、不提个股代码、不提竞品、东财不当新闻源 — 规则内嵌每个步骤
