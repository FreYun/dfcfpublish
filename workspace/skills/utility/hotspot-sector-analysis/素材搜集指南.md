# 素材搜集指南

> 铁律见 SKILL.md（已读过）：无来源不落笔、research-mcp 优先、数据获取失败→停下报告。

## Step 1：板块行情数据

### 确认今日日期

```bash
TODAY=$(date +%Y%m%d)
echo "今日交易日期: $TODAY"
```

所有 trade_date 参数用这个值，禁止手工输入。

### 1.1 板块指数涨跌幅

**16:00 之前（盘中/刚收盘）：**

用 Tushare API 取概念板块/申万行业指数数据：

```bash
# 先确认日期
TODAY=$(date +%Y%m%d)

# 申万一级行业（如板块属于某个申万行业）
curl -s -X POST https://api.tushare.pro \
  -H 'Content-Type: application/json' \
  -d "{
    \"api_name\": \"sw_daily\",
    \"token\": \"从 TOOLS.md 获取 Tushare Token\",
    \"params\": {\"trade_date\": \"$TODAY\"},
    \"fields\": \"ts_code,name,trade_date,close,pct_change,vol,amount\"
  }"
```

> 从返回的 31 个申万一级行业中，筛选与热点板块相关的行业。

### 数据返回校验（必做）

拿到 Tushare 结果后检查：
1. `items` 非空？空 = 当日数据不可用
2. `trade_date` 等于 `$TODAY`？不等 = 历史数据

**空数据 → 尝试使用research-mcp获取数据，若无，使用备选方案。**
**日期不符 → 不允许使用错误的日期。**

**16:00 之后（research-mcp 实时行情）：**
npx mcporter call "research-mcp.market_overview()"
npx mcporter call 'research-mcp.get_ashares_index_quote(symbol: "申万行业代码")'

**备用方案（仅在研究部同意后使用）：**

如果研究部确认可以继续，可尝试以下备用数据源：

```

# 备用方案 ：浏览器截取东方财富实时数据
browser_navigate(url: "https://quote.eastmoney.com/center/boardlist.html#boards-BK0XXX", profile: "bot9")
browser_snapshot(profile: "bot9")
browser_close(profile: "bot9")
```

备用数据源获取到的数据同样需要校验日期，确认是当日数据后方可使用。

### 1.2 板块内个股表现

用 research-mcp 搜索板块内龙头个股的当日表现：

```
npx mcporter call 'research-mcp.news_search(query: "板块名称 涨停 龙头", top_k: 10)'
```

或用浏览器访问东方财富板块页面查看个股涨跌排行：

```
browser_navigate(url: "https://quote.eastmoney.com/center/boardlist.html#boards-BK0XXX", profile: "bot9")
browser_snapshot(profile: "bot9")
browser_close(profile: "bot9")
```

### 1.3 港股/海外联动（如有）

如果板块有对应的港股/美股标的（如创新药→恒生医疗保健、AI→纳斯达克科技股），搜索相关市场联动：

```
npx mcporter call 'research-mcp.news_search(query: "板块名称 港股 联动", top_k: 5)'
```

### 1.4 数据校验

- 浏览器取的数据与 API 数据交叉验证
- 差异 ≤ 0.03%：正常，以收盘数据为准
- 差异 > 0.03%：**向研究部报告**，等待指示

## Step 2：24h 资讯搜集

> ⛔ **本步骤优先使用 `research-mcp.news_search`。** research-mcp 返回 `[raw_id, 摘要, score]`，**不返回来源名/报道方**。拿到 raw_id 后必须用 web_fetch 打开 `https://fund.eastmoney.com/a/{raw_id}.html` 确认实际报道方（页面 HTML 中有"来源：xxx"字段），不可从摘要内容猜测来源。research-mcp 搜了 2 轮仍找不到时，可以把时间放宽到48h，用research-mcp再检索一次；仍找不到时，可降级到 web_search 补充，但需在草稿中标注来源方式。

搜索 24 小时内与该板块相关的所有重要资讯，提取事件性驱动因素。

### 2.1 搜索方式（优先 research-mcp，备择 web_search）

```
# 搜索板块相关资讯（替换为实际板块名称）
npx mcporter call 'research-mcp.news_search(query: "板块名称 利好", top_k: 10)'
npx mcporter call 'research-mcp.news_search(query: "板块名称 政策", top_k: 10)'
npx mcporter call 'research-mcp.news_search(query: "板块名称 数据 发布", top_k: 10)'
npx mcporter call 'research-mcp.news_search(query: "板块名称 公告 突破", top_k: 10)'
```

### 2.2 搜索关键词参考

根据板块类型灵活组合关键词：

| 板块类型 | 推荐搜索词 |
|---------|-----------|
| 科技类（AI、半导体、算力） | "板块名 技术突破""板块名 订单""板块名 产能" |
| 医药类（创新药、中药） | "板块名 临床数据""板块名 获批""板块名 集采""板块名 出海" |
| 新能源类（光伏、锂电、风电） | "板块名 装机量""板块名 产能""板块名 出口""板块名 补贴" |
| 消费类（白酒、旅游、零售） | "板块名 消费数据""板块名 旺季""板块名 提价" |
| 周期类（煤炭、有色、化工） | "板块名 价格""板块名 库存""板块名 供需""板块名 地缘" |
| 金融类（银行、券商、保险） | "板块名 政策""板块名 业绩""板块名 资金面""板块名 改革" |
| 基建类（建材、建筑、交运） | "板块名 投资""板块名 项目""板块名 订单""板块名 专项债" |

### 2.3 整理为素材清单

从搜索结果中提取有价值的驱动因素，记在草稿中。保留 `raw_id` 和摘要。

**⛔ 确认来源（必做，不可跳过）**：
- 对每条准备引用的新闻，用 `web_fetch` 打开 `https://fund.eastmoney.com/a/{raw_id}.html`，从页面中提取实际的"来源：xxx"字段
- ⛔ **禁止从摘要猜测来源**——news_search 不返回来源名，之前 bot 凭摘要猜"新华社""中国证券报"等，实测 4 条里 3 条归属错误
- 只有**经 web_fetch 确认了实际报道方**的新闻，才可进入正文引用和文末来源说明
- web_fetch 无法获取来源时，降级用 browser 打开确认

```
【素材清单（草稿，不发布）】

板块：XXX

1. 政策驱动
   - 具体政策/文件名称 + 核心内容 + 来源

2. 产业/数据驱动
   - 具体数据/事件 + 核心数字 + 来源

3. 公司/技术驱动
```

## Step 3：机构观点搜集

> ⛔ **本步骤优先使用 `research-mcp.research_search`。** research-mcp 返回 `[研报ID, 摘要, 券商名, score]`，**已包含券商名**（如"招商证券""国信证券"），但**不返回研报标题和研究员**。券商名可直接使用，无需从摘要猜测；但拿到研报 ID（形如 `AP2026...`）后，仍须用 pdftotext 下载并提取 PDF 前 5 页，确认研报标题、研究员，以及你准备引用的具体观点确实存在于该研报中。research-mcp 搜了 2 轮仍找不到时，可降级到 web_search 补充，但来源说明中需如实标注实际 URL。

搜索 3-5 家权威机构对该板块的最新研判，供模块 3 使用。

### 3.1 搜索方式（优先 research-mcp，备择 web_search）

```
# 搜索券商对该板块的研报/观点
npx mcporter call 'research-mcp.research_search(query: "板块名称 券商 研报 观点", top_k: 10)'
npx mcporter call 'research-mcp.research_search(query: "板块名称 机构 看好 研判", top_k: 10)'
npx mcporter call 'research-mcp.research_search(query: "板块名称 基金经理 观点", top_k: 5)'

# 针对性搜索头部券商
npx mcporter call 'research-mcp.research_search(query: "板块名称 中信 中金 国泰君安", top_k: 5)'
```

### 3.2 搜索关键词参考

| 场景 | 推荐搜索词 |
|------|-----------|
| 常规 | "板块名 券商 研报""板块名 机构 观点" |
| 大涨日 | "板块名 机构 看好 逻辑""板块名 券商 推荐" |
| 大跌日 | "板块名 风险 机构""板块名 券商 警示" |
| 政策日 | "板块名 政策 影响 机构 解读" |
| 数据日 | "板块名 数据 券商 点评" |

### 3.3 区分来源类型

搜索结果中要严格区分两类来源：

| 类型 | 特征 | 正文归因方式 |
|------|------|-------------|
| **机构原始研报** | research_search 返回，ID 形如 `AP2026...`，**已含券商名**，但不含标题和研究员。仍须用 pdftotext 确认标题及具体观点（见下方 3.3a） | "XX证券认为……[n]"，[n] 指向研报 PDF |
| **聚合新闻转述** | news_search 返回，来自东方财富/新浪等，文中提到"XX证券认为…" | 可以归因给 XX证券，但 [n] 指向该新闻，来源说明如实写新闻来源 |

**⛔ 绝对禁止**：[n] 原文中根本没有提到某机构，却在正文中写"该机构认为……[n]"——这是凭空编造，不是转述问题。

### 3.3a ⛔ 研报标题与观点确认（必做，写稿前完成）

research_search 已返回券商名（第 3 个字段），可直接用于正文归因；但**不返回研报标题和研究员**，且摘要可能经过截断或改写，因此仍须下载 PDF 确认标题和具体观点：

```bash
# 对每个准备引用的研报 ID，下载 PDF 并提取前 5 页
mkdir -p /tmp/factcheck-pdf
curl -s "https://pdf.dfcfw.com/pdf/H3_{研报ID}_1.pdf" -o /tmp/factcheck-pdf/{研报ID}.pdf
pdftotext -f 1 -l 5 -layout /tmp/factcheck-pdf/{研报ID}.pdf -
```

从提取的文本中确认：
- **首页**：研报标题、研究员、日期（券商名已由 API 返回，可与首页交叉验证）
- **前 2-5 页**（"投资要点"/"核心观点"）：你准备引用的具体观点是否确实存在于该研报中

**确认完毕后清理临时文件**：`rm -rf /tmp/factcheck-pdf`
