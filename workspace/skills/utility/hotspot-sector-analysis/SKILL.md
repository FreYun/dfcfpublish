# 热点板块追踪（hotspot-sector-analysis）

> 交易日随机发文，有明显热点板块异动才发。编辑给出板块名称，AI 自主完成素材搜集和文章撰写。

---

## 触发场景

**触发**：研究部/编辑说"写一篇XX板块的热点分析"、"XX板块今天大涨，写一篇"、"追一下XX热点"、给出板块名称并要求写公众号文章等类似指令。

**适用条件**：
- 交易日当天有板块出现明显异动（大涨/大跌）
- 编辑明确给出了热点板块名称
- 目标是输出公众号文章初稿

**不适用**：
- 全市场每日复盘 → 用 `daily-market-recap` skill
- 只是聊聊某个板块、不需要出文章 → 正常对话即可
- 非交易日 → 不发

---

## ⛔ 铁律（全程有效）

1. **无来源不落笔** — 正文中的每一个数字、每一条事实、每一个观点都必须有明确的、可追溯的来源。找不到来源的内容一律不写，没有例外
2. **绝对禁止编造** — 没有读到的研报，不许写”XX证券认为...”；没有拿到的行情数据，不许凭记忆填写
3. **数据获取失败 → 停下来报告** — 不许在缺少关键数据的情况下继续写作，向研究部说明失败情况，等待指示
4. **全角标点** — 正文中所有标点必须中文全角（，。：””（）！？），基金代码等纯数字场景除外
5. **research-mcp 优先** — 不可跳过 research-mcp 直接用 web_search；research-mcp 搜至少 2 轮后才可降级
6. **浏览器只用 bot9 profile** — 禁止爬虫（curl/requests/WebFetch），读完必须关闭浏览器

---

## 分步工作流（⛔ 不要一次读完全部文件，按步骤走）

收到写热点板块分析的指令后，**按以下 4 步依次执行**。每一步开头标注了”现在读”的文件，到该步骤时再读，不要提前全部加载。

---

### Step 1：确认日期 + 素材搜集

**现在读：** `Read skills/hotspot-sector-analysis/素材搜集指南.md`

> ⛔ **日期铁律**：先执行 `date +%Y%m%d` 获取当日日期，禁止手工输入年份（之前出过把 2026 写成 2025 的事故）
>
> ⛔ **Tushare 校验**：拿到数据后必须检查 items 非空 + trade_date 等于今日。空数据/日期不符 → 停下来报告研究部，不得继续
>
> ⛔ **research-mcp 接口字段**：`research_search` 返回 `[研报ID, 摘要, 券商名, score]`（已含券商名）；`news_search` 返回 `[raw_id, 摘要, score]`（不含来源名，需 web_fetch 确认）

**执行：**
1. 执行 `date +%Y%m%d` 确认当日日期
2. 行情数据：Tushare API 取板块指数涨跌幅 + 龙头个股表现
3. 24h 资讯：`research-mcp.news_search` 搜索板块相关新闻/政策/事件
4. 机构观点：`research-mcp.research_search` 搜索 3-5 家机构最新研判
5. 素材搜集完成后，进入 Step 2

---

### Step 2：撰写文章（四模块）

**现在读：** `Read skills/hotspot-sector-analysis/写作指南.md` + `Read skills/hotspot-sector-analysis/引用来源管理.md`

> ⛔ **模块 3 归因铁律（本步最易违反）**：机构观点必须在正文中**明确写出机构名 + [n] 角标**（”招商证券认为……[1]”），不能只在文末来源列表提。每个 [n] 指向的原文必须确实包含你归因的观点，否则就是凭空编造
>
> ⛔ **无来源不落笔**：没有来源的观点、数字、事实一律不写进正文，有来源必须指出来自xxx机构

**执行：**
1. 模块 1：行情播报（≤150 字）— 板块指数 + 个股表现 + 配图占位
2. 模块 2：深度分析（500-600 字）— 2-4 个核心驱动原因，有数据/事件支撑
3. 模块 3：机构观点（400-500 字）— 3-5 家机构观点，正文中明确归因
4. **现在读：** `Read skills/hotspot-sector-analysis/基金推荐规则.md`
5. 调用基金筛选脚本：
   ```bash
   python skills/daily-market-recap/fund_selector_llm.py --topic 板块名称
   ```
6. 模块 4：相关基金（5 只）— 根据脚本结果写展示文本
7. 写完后进入 Step 3

---

### Step 3：质量自检

> ⛔ **自检报告必须明文输出**，不能只在心里过一遍。

**执行：**
1. 按写作指南末尾的自检清单逐条检查，**明文输出自检报告**
2. 有 ❌ 的先改稿，全部 ✅ 后进入 Step 4

> ⛔ **自检完成 ≠ 写稿完成。** 自检只检查格式和合规，你还必须做 Step 4 的 fact-check（数据准确性核查）。不要在自检后就提交给研究部——**现在立刻继续 Step 4。**

---

### Step 4：fact-check + 存档 + 呈审

> ⛔ **本步不可跳过。** 之前出过事故：bot 写完就直接提交，跳过了 fact-check。自检 ≠ 数据核查，两者都必须做。

**现在读：** `Read skills/fact-check/SKILL.md`

**执行：**
1. **fact-check**：按 fact-check skill 的流程，逐条核实文中所有数字和事实性表述。每条”✅”必须有对应工具调用，禁止不调工具直接编写核查报告
2. **⛔ 核查发现偏差 → 必须改稿**：不能只输出核查报告就算完。流程是：输出核查报告 → 根据报告中的 ❌ 项逐条修改稿件 → 修改完毕后重新核查 → 直到全部 ✅。**只出报告不改稿 = 未完成核查**
3. **存档**：改稿后的终稿 + 来源说明 + 自检报告 + fact-check 核查报告，一起写入 `memory/hotspot-sector/YYYY-MM-DD-板块名.md`
4. **呈审**：提交研究部确认，获得同意后才可发布

---

## 四模块结构概览

| 模块 | 内容属性 | 字数 | 说明 |
|------|---------|------|------|
| 1 行情播报 | 事实（数据） | ≤ 150 字 | 板块指数 + 个股表现 + 【配图占位】 |
| 2 深度分析 | 分析（原创） | 500-600 字 | 2-4 个核心驱动原因，有数据/事件支撑 |
| 3 机构观点 | 引述（归因） | 400-500 字 | 3-5 家机构观点，正文中明确写出机构名 + [n] 角标 |
| 4 相关基金 | 推荐（基金池） | — | 5 只板块相关权益类基金 |

---

## 合规红线

- **不给操作建议** — 不说”建议买入/卖出””建议抄底””建议减仓”等
- 不说”配置方向””建议配置”，用”重点关注””看好方向””值得留意”
- 不说”必涨/必跌/稳赚”，用”存在机会/需要警惕”
- 不提个股代码/名称（基金代码/名称除外）
- **不提竞品**（同花顺、Wind/万得、iFinD）
- **东方财富不是新闻源** — 禁止”据东方财富报道”等表述，引用时写实际报道方
- 文末必加风险提示

---

## 存档格式

存档到 `memory/hotspot-sector/YYYY-MM-DD-板块名.md`，文件顶部加 `# 热点板块追踪 YYYY-MM-DD 板块名`。如当日同板块已有同名文件则覆盖。文件结构：终稿正文 → 来源说明 → 自检报告 → fact-check 核查报告，各部分之间用 `---` 分隔。
