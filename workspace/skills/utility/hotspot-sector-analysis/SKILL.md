# 热点板块追踪（hotspot-sector-analysis）

> 交易日随机发文，有明显热点板块异动才发。编辑给出板块名称，AI 自主完成素材搜集和文章撰写。

---

## 触发场景

**触发**：研究部/编辑说"写一篇XX板块的热点分析"、"XX板块今天大涨，写一篇"、"追一下XX热点"、给出板块名称并要求写公众号文章等类似指令。

**适用条件**：
- 交易日当天有板块出现明显异动（大涨/大跌）
- 编辑明确给出了热点板块名称
- 目标是输出公众号文章初稿

**不适用**：
- 全市场每日复盘 → 用 `daily-market-recap` skill
- 只是聊聊某个板块、不需要出文章 → 正常对话即可
- 非交易日 → 不发

---

## ⛔ 写稿前必须先读文档（不读不许动笔）

收到写热点板块分析的指令后，**第一件事是依次读完以下文件**，全部读完再开始任何写作：

```
1. Read skills/hotspot-sector-analysis/写作指南.md            ← 四模块结构、标题写法、自检清单
2. Read skills/hotspot-sector-analysis/素材搜集指南.md        ← 行情+资讯+机构观点搜集流程
3. Read skills/hotspot-sector-analysis/基金推荐规则.md        ← 基金选取和展示规则
4. Read skills/hotspot-sector-analysis/引用来源管理.md        ← 观点归因规则
```

**没读完这 4 个文件就动笔 = 违规。** 不许凭记忆写，每次都要重新读。

## 工作流程

```
编辑给出板块名称（如"创新药"）
        ↓
Step 0：确认当日日期（执行 date +%Y%m%d，禁止手工输入年份）
        ↓
Step 1：素材搜集（按 素材搜集指南.md 执行）
  ├── 行情数据：板块指数涨跌幅 + 龙头个股表现
  │   └── ⛔ 拿到 Tushare 结果后必须校验：items 非空 + trade_date 等于今日
  │       → 空数据/日期不符 → 停下来报告研究部，不得继续
  ├── 24h 资讯：板块相关的最新新闻/政策/事件
  └── 机构观点：3-5 家权威机构对该板块的最新研判
        ↓
Step 2：撰写文章（按 写作指南.md 的四模块结构）
  ├── 模块 1：行情播报（≤150 字）
  ├── 模块 2：深度分析（500-600 字）
  ├── 模块 3：机构观点（400-500 字）
  └── 模块 4：相关基金（5 只基金）
        ↓
Step 3：质量自检
        ↓
Step 4：⛔ 执行 fact-check skill（必做，不可跳过）
  ├── Read skills/fact-check/SKILL.md
  ├── 按其流程逐条核实文中每个数字和事实性表述
  └── 有偏差 → 改稿 → 重新核查，直到全部通过
        ↓
Step 5：存档（fact-check 全部通过后再存档）
  └── 终稿 + 核查报告一起写入 memory/hotspot-sector/YYYY-MM-DD-板块名.md
        ↓
Step 6：呈审（未通过 fact-check 不得呈审）
```

> ⛔ **自检 ≠ 数据核查。** 质量自检（写作指南中的清单）检查的是格式和合规；fact-check skill 检查的是每个具体数字的准确性。两者都必须做，缺一不可。

---

## 四模块结构概览

| 模块 | 内容属性 | 字数 | 说明 |
|------|---------|------|------|
| 1 行情播报 | 事实（数据） | ≤ 150 字 | 板块指数 + 个股表现 + 【配图占位】 |
| 2 深度分析 | 分析（原创） | 500-600 字 | 2-4 个核心驱动原因，有数据/事件支撑 |
| 3 机构观点 | 引述（归因） | 400-500 字 | 3-5 家机构最新观点，正文中要明确写出是哪家机构的观点；完整引用来源和链接统一放文末说明 |
| 4 相关基金 | 推荐（基金池） | — | 5 只板块相关权益类基金 |

## ⛔ 全角标点铁律

正文中所有标点符号必须使用中文全角，不允许出现英文半角标点：
- 双引号用 \u201c\u201d 不用 ""
- 单引号用 \u2018\u2019 不用 ''
- 冒号用 ： 不用 :（基金代码等纯数字场景除外）
- 逗号用 ， 不用 ,
- 句号用 。 不用 .
- 括号用 （） 不用 ()（基金代码等纯数字场景除外）
- 感叹号用 ！ 问号用 ？

---

## ⛔ 时效性铁律

1. **所有数据必须是当日/实时的** — AI 必须调用实时数据接口，严禁使用陈旧数据
2. **资讯窗口 24 小时** — 只搜集和引用 24 小时内与该板块相关的资讯
3. **机构观点当日优先** — 当日发布的研报/观点优先引用；非当日研报必须确认核心观点未被后续行情推翻
4. **过时判断标准** — 研报的关键假设（利率走向、供需格局、政策预期）与当前实际情况不符，即为过时，**禁止引用**

---

## ⛔ 数据源工具优先级（research-mcp 优先，web_search 备择）

**之前出过事故：bot 直接跳过 research-mcp，全程用 web_search 搜索研报和资讯，导致返回结果无结构化字段（无 raw_id、无 source_name），引用来源无法追溯，ID→URL 还原规则完全失效。**

各环节的工具优先级：

| 环节 | 优先使用 | 备择（research-mcp 搜不到时） |
|------|---------|--------------------------|
| 行情数据 | Tushare API（`curl -s -X POST https://api.tushare.pro`） | research-mcp → web_search |
| 24h 资讯 | `npx mcporter call 'research-mcp.news_search(...)'` | web_search |
| 机构观点/研报 | `npx mcporter call 'research-mcp.research_search(...)'` | web_search |
| 访问具体链接 | `browser_navigate` + `browser_snapshot`（profile: “bot9”） | web_fetch |
| 基金信息核查 | research-mcp `get_fund_comprehensive_analysis` | web_search（research-mcp 搜至少 2 轮后） |
| 基金筛选 | `fund_selector_llm.py` 脚本 | — |

**为什么优先 research-mcp：**
- research-mcp 返回结构化字段（`raw_id`、`source_name`、摘要），方便引用来源追溯
- 引用来源管理中的 ID→URL 还原规则（研报 `pdf.dfcfw.com`、新闻 `fund.eastmoney.com`）只对 research-mcp 返回的 ID 有效
- research-mcp 索引的是东方财富研报库和资讯库，来源可信度有保障

**降级到 web_search 的条件：**
- research-mcp 已搜索至少 2 轮（换不同关键词），仍找不到需要的信息
- 降级时在草稿中标注”来源：web_search”，以便后续区分
- web_search 找到的内容，来源说明中如实标注来源网站和实际 URL，不得套用 research-mcp 的 ID→URL 还原格式
- ⛔ **不可一上来就用 web_search，跳过 research-mcp** — 这是之前出事故的根因
- ⛔ **research-mcp 工具调用失败（Unknown tool / 参数错误）时，先 Read research-mcp 的 SKILL.md 查可用工具列表，找到正确工具名再重试** — 不可因为一次调用失败就直接降级到 web_search。之前 bot 因猜错工具名 `fund_info`（正确名为 `get_fund_comprehensive_analysis`）而直接跳到 web_search，违反了优先级铁律

---

## ⛔ 数据准确铁律

1. **所有数据必须来自权威渠道** — 行情数据用 Tushare API，资讯和机构观点用 research-mcp 搜索（具体工具要求见上方”数据源工具铁律”）
2. **标注来源** — 每个具体数字/观点都必须可追溯到数据来源
3. **正文里可以写机构名和 `[n]` 引用角标，但不要写链接/长出处** — 如果文中提到机构观点，必须明确写出是哪家机构的观点；如果引用新闻事实，必须写出新闻源（如”据新华社报道……[1]”）；完整引用来源统一放在文末”来源说明”中列示，并附链接
4. **没有新闻源的新闻不得引用** — 如果只有摘要、没有明确来源名/发布机构，允许继续检索原文，但不得直接写进成稿正文或来源说明
5. **禁止虚假数据** — 不许编造、猜测任何数字，查不到就不写
6. **逻辑一致** — 模块 2 的原因分析必须与模块 1 的行情表现相呼应

## 浏览器铁律

- **只许用自己的浏览器**（`profile: "bot9"`），不许用其他 profile
- **禁止爬虫**（curl/requests/WebFetch），必须用浏览器访问链接
- **禁止擅自启动有头浏览器**（headed browser / `-headless=false`），未经研究部同意不得使用
- 读完链接后**必须关闭浏览器**

```
browser_navigate(url: "链接", profile: "bot9")
browser_snapshot(profile: "bot9")
browser_close(profile: "bot9")   ← 读完必须关闭
```

## 合规红线

- **不给操作建议** — 不说"建议买入/卖出""建议抄底""建议减仓""不建议追涨"等任何指导读者具体操作的话
- 不说"配置方向""建议配置"，用"重点关注""看好方向""值得留意"
- 不说"必涨/必跌/稳赚"，用"存在机会/需要警惕"
- 不提个股代码/名称，用行业板块描述（基金代码/名称除外，允许出现）
- 不做收益承诺，不炫耀具体盈亏数字
- **不提竞品** — 正文中禁止出现同花顺、Wind/万得、iFinD 等东方财富竞品的名称、产品或数据标注
- **东方财富不是新闻源** — 禁止出现"据东方财富报道""东方财富指出"等将东方财富当作报道方/观点来源方的表述。东方财富是数据聚合平台，没有新闻采编资质；正文引用时只许写实际的报道方或观点来源方（如"据中国证券报报道""中信证券认为"）
- 文末必加风险提示

---

## 研报引用规则

模块 2 和模块 3 中如需引用研报、券商观点等资讯：

- **当日研报优先**：有当天发布的研报/观点，优先采用，时效性最强
- **历史研报须严格甄别**：引用非当日研报时，必须判断其核心观点是否已被后续行情或政策变化推翻。如果市场环境、数据面已发生实质性变化，该观点视为过时，**禁止引用**
- **过时判断标准**：研报的关键假设（如利率走向、供需格局、政策预期）与当前实际情况不符，即为过时
- 引用时正文必须点明机构名称，必要时可带日期；新闻报道类内容必须写出新闻源并带 `[n]` 角标，如“据新华社报道……[1]”；完整出处、标题和链接统一放到文末“来源说明”中列示，方便读者追溯
- 如果新闻结果缺少来源名/发布机构，即使摘要内容看起来可用，也**禁止引用**，必须继续寻找可核查原文

## 写稿完成后

1. ## ⛔ 数据核查（fact-check）— 必做，不可跳过

   质量自检通过后、存档前，**必须执行以下操作**：

   ```
   Read skills/fact-check/SKILL.md   ← 必须实际读取这个文件
   ```

   然后**严格按 fact-check skill 的流程**，对完整稿件逐条核实：
   - 每一个具体数字（涨跌幅、金额、百分比、日期）
   - 每一个事实性表述（事件、排名、引用观点的归属）
   - 每一个数据来源是否可追溯

   核查完成后，**必须输出完整的数据核查报告**（不是自检报告），格式参照 fact-check skill 的要求。

   **硬性门槛**：
   - 核查发现偏差 → **先按核查意见改稿**，改完后重新核查，直到全部通过
   - 有「待确认」项 → 标注并告知研究部，不得默认正确
   - **未输出 fact-check 核查报告 = 未完成核查 = 不得存档和呈审**

   > 注意：写作指南中的「质量自检」检查格式和合规，fact-check 检查数据准确性。**两者是不同的步骤，都必须做。** 不要把自检报告当作 fact-check 报告。

2. **存档**（fact-check 全部通过后再存档）：将**改稿后的终稿**保存到 `memory/hotspot-sector/YYYY-MM-DD-板块名.md`（以当日日期和板块名命名）。文件顶部加一行 `# 热点板块追踪 YYYY-MM-DD 板块名`。如当日同板块已有同名文件则覆盖（以最新版本为准）。

   **文件结构**：终稿正文 → 来源说明 → 自检报告 → fact-check 核查报告，各部分之间用 `---` 分隔。核查报告追加在同一文件末尾，不另存文件。

3. **呈审**：完整稿件在正文后必须附上”来源说明”区块，集中列示本文使用的**全部来源**（行情数据、新闻资讯、机构观点、基金筛选结果等），并**附上对应链接**。正文中的事实句、新闻句、机构观点句应使用 `[1]`、`[2]` 这类角标与文末来源逐一对应；正文里如果提到机构观点，必须明确写出机构名称；如果引用新闻，必须明确写出新闻源。正文中不得出现裸链接，但可以出现 `[n]` 角标。然后稿件**必须先呈给研究部确认**，获得明确同意后才可交由 **gzh-publish** skill 发布。未经确认不得自行发布。
