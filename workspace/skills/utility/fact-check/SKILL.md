# 数据核查 /fact-check

**触发**：`/fact-check`、"核实一下数据"、"检查数据准确性"

---

## 铁律

1. **每一个具体数字都必须核查**——价格、百分比、金额、产量、日期，无一例外
2. **无法验证 ≠ 正确**——查不到来源的数据标为「待确认」，不能默认正确
3. **核查结果必须诚实**——数据有误就标错，不要为了好看而放过
4. **来源归属必须可追溯**——文中说"xxx机构观点"、"xxx报道"，必须确认该机构/媒体确实说过，不能编造或张冠李戴
5. **⛔ 禁止伪造核查报告（最高优先级铁律）**——之前出过严重事故：bot 在没有调用任何验证工具的情况下，直接编写了一份全部标"✅ 正确"的核查报告。这是**比数据错误更严重的违规**，因为它让研究部丧失了对稿件质量的判断能力。具体要求：
   - **核查报告中每一条"✅ 已验证"都必须有对应的工具调用记录**——你标了"✅ 正确"就意味着你实际调用了 research-mcp / web_fetch / browser 等工具并看到了返回结果
   - **没有调用工具就不能标"✅"**——如果你还没验证某条数据，要么现在去验证，要么标「待核查」
   - **核查报告必须注明每条数据的验证方式**——如"research-mcp.get_fund_comprehensive_analysis 返回"、"web_fetch 页面显示来源：澎湃新闻"、"pdftotext 首页显示东吴证券研究所"等，方便研究部追溯
   - **凭记忆/凭摘要/凭推测写的核查报告 = 伪造**——即使结果碰巧正确，过程不合规也是违规

---

## 执行流程

### Step 1 — 提取数据清单

逐段扫描文章，提取所有事实性表述，包括：
- 具体数字（价格、涨跌幅、产量、库存、渗透率等）
- 时间性表述（"本周"、"2025年四季度"等）
- 事件性表述（"Tesla 宣布采购29亿美元光伏设备"等）
- 排名/对比性表述（"最低"、"第一"等）
- **来源归属表述**（"据xxx机构"、"xxx研报指出"、"xxx媒体报道"、"某某基金经理认为"等）

### Step 2 — 逐条核查

按以下优先级选择核查手段：

| 优先级 | 核查方式 | 适用场景 |
|--------|---------|---------|
| 1 | research-mcp 工具 | 行情、基金（`get_fund_comprehensive_analysis`）、宏观数据等 |
| 2 | web_fetch 打开新闻链接 | 验证新闻来源归属（`fund.eastmoney.com` 页面的"来源：xxx"字段） |
| 3 | pdftotext 提取研报 PDF | 验证研报券商名、标题、研究员、观点内容（`curl` 下载 + `pdftotext -f 1 -l 5`） |
| 4 | `research_search` / `news_search` | 补充搜索研报观点、行业数据、事件类表述 |
| 5 | `deepresearch` skill | 需要深度搜索的复杂表述 |
| 6 | web_search | 公开新闻、公告类事实（research-mcp 搜不到时的降级手段） |

核查时注意：
- 数据有时效性，确认文章引用的数据对应的时间窗口
- 百分比要核实计算过程（基数和结果是否匹配）
- 转引数据追溯到原始来源（如"硅业分会数据"要确认是否真实存在该口径）
- **⛔ 基金三要素验证（代码 / 名称 / 经理）**：文中每只基金都必须逐只验证，流程如下：
  1. 用 research-mcp 的 `get_fund_comprehensive_analysis` 传入基金代码，获取该基金的官方信息
  2. 核对返回结果中的 **基金代码、基金全称/简称、现任基金经理** 是否与文中一致
  3. 不一致 → 标为「有偏差」，给出 research-mcp 返回的正确值，建议改稿
  4. 工具调用失败（Unknown tool 等）→ 先 Read research-mcp 的 SKILL.md 查可用工具列表，找到正确的基金查询工具再重试，**不可直接降级到 web_search**
  5. 三项全部一致才算通过，不能只核对其中一项
- **⛔ 来源归属必须打开链接核实**——research-mcp 的 `news_search` / `research_search` **不返回来源名/报道方**，只返回 `[raw_id, summary, score]`。因此：
  1. **禁止根据摘要猜测来源**——之前 bot 凭摘要内容猜"新华社""中国证券报"等，实际验证发现 4 条里 3 条归属错误（如实际来源是澎湃新闻却写成新华社）
  2. **每条引用链接必须用 web_fetch 打开原文页面**，从 HTML 中提取实际的"来源：xxx"字段（东方财富资讯页通常在正文头部或末尾有"来源：xxx"标注）
  3. web_fetch 无法获取来源信息时（JS 渲染页面等），降级用 browser 打开确认
  4. **研报链接（`pdf.dfcfw.com`）同样必须验证**——`research_search` 也不返回券商名、研报标题、研究员，bot 同样是从摘要内容猜测。验证方式：
     - **用 curl 下载 PDF（不用 HEAD、不用 browser）**：
       ```bash
       mkdir -p /tmp/factcheck-pdf
       curl -sSL "https://pdf.dfcfw.com/pdf/H3_{研报ID}_1.pdf" -o /tmp/factcheck-pdf/{研报ID}.pdf --max-time 30
       ```
     - **⛔ 下载后必须先检查文件大小 + mime 类型，确认是真 PDF 而不是鉴权报错 JSON**（2026-04-13 bot9 事故：dfcfw 对部分研报会返回 33 字节 `{"msg":"鉴权失败","code":403}`，HEAD 也返回 200，骗过了单纯的 URL 可达性检查）：
       ```bash
       size=$(stat -c%s /tmp/factcheck-pdf/{研报ID}.pdf)
       mime=$(file -b /tmp/factcheck-pdf/{研报ID}.pdf)
       # size < 10240 字节 或 mime 不含 "PDF document" → URL 不可用
       # 该研报引用必须从正文和来源说明中删除，不许保留
       ```
     - **用 pdftotext 提取前 5 页文本**：
       ```bash
       pdftotext -f 1 -l 5 -layout /tmp/factcheck-pdf/{研报ID}.pdf -
       ```
     - **⛔ 首页机构署名必须与正文归因的机构完全一致**——这是 bot9 事故的核心教训：AP ID 拼出的 URL 能下载 PDF 不代表 PDF 属于正文里那家机构（曾经下载到的是"头豹研究院"却被写成"中银证券"）。首页抬头必须真的是你在正文里写的那家券商，不一致立即删除该引用，不许保留、不许"换个标题挡一下"
     - **首页**确认：**券商名**（如"东吴证券研究所"）、**研报标题**、**研究员姓名**、**日期**
     - **前 2-5 页**（"投资要点"/"核心观点"部分）核对：文中转述的每个具体观点是否在研报原文中存在。券商研报的核心结论和推荐观点基本集中在前 5 页
     - ⛔ **不要用 browser 打开 PDF 链接**——Headless Chrome 无法渲染 PDF 内容（PDF viewer 插件在 headless 模式下不工作），browser_snapshot 返回空白
     - ⛔ **也不要用东方财富研报详情页** `data.eastmoney.com/report/info/{ID}.html`——部分研报 ID 会 redirect 到研报中心首页，不可靠
     - **每条研报 ID 必须独立验证**——不能因为一条下载失败就跳过剩余所有研报
     - 核实三项：**发布券商、研报标题、引用观点是否确实出自该研报**
     - ⛔ **验证完毕后必须清理临时文件**：`rm -rf /tmp/factcheck-pdf`——不得遗留 PDF 文件
  5. **⛔ 核实内容一致性（观点必须出自该引用原文，禁止混搭拼接）**——之前出过严重事故：bot 从 research_search 拿到多条研报摘要后，把 A 研报的数据 + B 研报的观点 + 自己的推理拼接在一起，全部归到一个券商名下。具体要求：
     - 文中写"XX证券指出……[n]"，那么 [n] 对应的研报/文章里**必须有这个具体观点**，不能是从其他来源拼过来的
     - **新闻用 web_fetch 打开原文、研报用 pdftotext 提取前 5 页后，必须核对文中转述的每个要点是否在原文中存在**——不能只看标题匹配就标 ✅
     - 发现文中观点在对应引用原文里找不到 → 标为「内容不匹配」，说明该观点实际出自哪条搜索结果（如果能定位的话）
     - 常见问题：
       - **混搭拼接**：把多条搜索结果的内容揉到一段话里，归到一个 [n] 引用下（**最常见**）
       - **过度演绎**：原文说"锂价震荡"，文中写成"锂价有进一步上行动力"
       - **张冠李戴**：A 券商的观点写成 B 券商说的
  6. 找不到原始出处或报道方不匹配的，标为「来源归属错误」或「来源待确认」，不能默认归属正确
  
  **⛔ 浏览器使用铁律（仅用于 web_fetch 降级到 browser 验证新闻来源时）：**
  - 研报验证用 pdftotext，**不需要 browser**
  - 新闻来源验证优先用 web_fetch，只有 web_fetch 无法获取来源信息时才降级用 browser
  - 如需用 browser：**同时最多 3 个标签页**，**验证完立即关闭**，**全部完成后确认无遗留标签页**
  
  常见问题：
  - 张冠李戴：观点存在但不是文中说的那个机构说的（**最常见**，因为 bot 从摘要猜来源）
  - 凭空捏造：该机构从未发表过相关观点
  - 过度概括：原始表述有限定条件，文章引用时去掉了
  - 内容不匹配：链接指向的文章与文中引用内容无关

### Step 3 — 输出核查报告

用以下格式输出，按核查结果分组：

```markdown
## 数据核查报告

### 已验证
| 原文表述 | 核查结果 | 数据来源 |
|---------|---------|---------|
| 硅料均价48元/kg | 正确 | research-mcp commodity 数据 |

### 有偏差
| 原文表述 | 实际数据 | 偏差说明 | 数据来源 |
|---------|---------|---------|---------|
| 出口同比增长6.4% | 实际为5.8% | 原文可能引用了不同口径 | 海关总署 |

### 来源归属与内容一致性验证（逐条打开链接核实）
| 引用编号 | 文中声称来源 | 链接实际来源 | 来源结果 | 文中转述的观点 | 原文中是否存在 | 建议 |
|---------|------------|------------|---------|-------------|-------------|------|
| [1] | 新华社 | 澎湃新闻 | ⛔ 来源错 | "四部委整治内卷" | ✅ 原文有 | 改为"据澎湃新闻报道" |
| [4] | 东吴证券 | 东吴证券（pdftotext首页确认） | ✅ | "4月排产超25年底" | ✅ pdftotext前5页有 | — |
| [6] | 国金证券 | 东吴证券（pdftotext首页确认） | ⛔ 券商名错 | "锂电材料业绩修复，六氟磷酸锂涨价弹性" | ⚠️ 部分存在但属于东吴观点 | 改券商名 + 拆分观点来源 |
| [6] | 国金证券 | — | — | "建议关注锂电材料龙头" | ⛔ 混搭拼接，实际来自另一条搜索结果 | 定位原始来源后修正 |

### 基金三要素验证
| 基金代码 | 文中名称 | 文中经理 | 实际名称 | 实际经理 | 结果 |
|---------|---------|---------|---------|---------|------|
| 026266 | 广发储能电池ETF联接C | 夏浩洋 | 广发国证新能源电池ETF联接C | 夏浩洋 | ⚠️ 名称略有出入 |

### 待确认（无法验证）
| 原文表述 | 核查尝试 | 建议 |
|---------|---------|------|
| 专利费收入30-100亿 | 未找到原始测算 | 建议删除或标注为"市场预期" |
```

### Step 4 — 给出修改建议

针对「有偏差」和「待确认」的条目，给出具体修改建议：
- 数据有误 → 给出正确数据和来源
- 无法验证 → 建议删除、模糊化处理（如改为趋势描述）、或标注信息来源
- 计算错误 → 给出正确计算过程

## 质量标准

| 检查项 | 标准 |
|--------|------|
| 数据覆盖率 | 文章中所有具体数字均已核查 |
| 验证方式标注 | 每条已验证数据都注明了验证方式（web_fetch / pdftotext / research-mcp） |
| 新闻来源归属 | 所有新闻类引用均已用 web_fetch 打开链接确认实际报道方 |
| **URL 物理可达性** | **所有研报 URL 都已用 curl 实际下载，文件大小 ≥ 10KB 且 `file` 命令判定为 PDF document；任何返回鉴权报错 JSON（`{"msg":"鉴权失败"...}`）或 HTML 错误页的 URL 对应引用必须已从正文和来源说明中删除** |
| 研报来源归属 | 所有研报类引用均已用 pdftotext 提取 PDF 首页确认券商名、标题、研究员 |
| **研报首页机构匹配** | **pdftotext 首页抬头的机构名必须与正文里归因的那家券商完全一致，不一致的引用必须已删除（2026-04-13 bot9 事故：AP ID 拼出的 PDF 实际署名"头豹研究院"，被错误归因为"中银证券"）** |
| 研报内容一致性 | 研报类引用的具体观点均已在 pdftotext 前 5 页文本中找到原文依据 |
| 基金三要素 | 所有基金的代码、名称、经理均已用 research-mcp 逐只验证一致 |
| 偏差标注清晰 | 有偏差的数据说明了实际值和偏差原因 |
| 建议可操作 | 修改建议具体到可以直接改稿 |
| 临时文件已清理 | `/tmp/factcheck-pdf` 目录已删除，无遗留 PDF |
