# 商单知识卡片 Skill（shangdan-card）

> 装备即生效。产出一套完整的小红书多图帖：3-4 张知识卡片 + 配套正文。

## 触发场景

| 触发条件 | 动作 |
|---------|------|
| 研究部说「做商单」「接了个基金推广」「帮 XX 基金出一套图文」 | 从 Step 1 开始 |
| 研究部给了基金产品资料 + 业绩走势图，让你出内容 | 从 Step 1 开始 |
| 研究部说「出一套科普卡片」「做个知识图文帖」且涉及基金/理财产品 | 从 Step 1 开始 |
| 研究部给了选题方向 + 要求带业绩图和风险提示 | 从 Step 1 开始 |
| 收到「✅」/「已批准」 | 按 `skills/xhs-op/投稿发布.md` 投稿 |
| 收到修改意见/「打回」 | 按意见修改后重新提交 |

### 不触发的场景

| 场景 | 用什么 |
|------|-------|
| 日常发帖（非商单） | `skills/xhs-op/daily-post.md` |
| 只需要封面图 | `james-cover` |
| 纯文字帖（不需要多图卡片） | `xhs-writing` |
| 非基金/理财类商单 | 根据品类选其他 skill |

## 铁律

0. **每次必须先 Read 本文件 + 你的 `../../SOUL.md` 和 `../../IDENTITY.md`** — 用你自己的人设、语气、风格写内容，不同 bot 产出必须有区分度。再按需 Read CARDS.md / STYLE.md / COMPLIANCE.md。
1. **先出方案再生图** — 研究部确认后才调用生图。
2. **指定图片原样用** — 研究部提供的业绩走势图等素材不要 AI 重画。
3. **每张图文字 ≤ 200 字** — 卡片是「一眼扫完」的浓缩版。
4. **正文 250-450 字** — 不许太长。
5. **不露骨推销** — 干货为主、产品自然植入。
6. **效果好的 prompt 记录到 `memory/好用的图片prompt.md`**。

## 帖子整体结构

一个商单帖 = **3-4 张图片** + **1 篇正文**（250-450 字）。

```
图 1（封面）   讲什么 + 为什么要看
图 2（逻辑A）  核心逻辑讲解 + 示意图
图 3（逻辑B）  补充逻辑/对比 + 示意图（可选）
图 4（业绩）   研究部提供的业绩走势图 + 风险提示

正文           口语化展开 + 产品植入 + 互动钩子
```

| 内容复杂度 | 图片数 |
|-----------|-------|
| 简单（1 个核心逻辑） | 3 张：封面 + 逻辑 + 业绩 |
| 标准（2-3 个逻辑点） | 4 张：封面 + 逻辑×2 + 业绩 |

**业绩图永远是最后一张。**

---

## 布局系统（3 种模式）

每张卡片内部选用以下布局：

| 模式 | 代号 | 用途 |
|------|------|------|
| 全宽 | `FULL` | 标题、单段文字、示意图 |
| 左右分栏 | `LR` | 对比、图文并排 |
| 横向并列 | `ROW` | 3-4 个平级概念 |

```
一件事展开讲 → FULL ｜ 两件事对比 → LR ｜ 多个平级概念 → ROW ｜ 拿不准 → FULL
```

---

## 配套正文

```
图片 = 骨架（每张 ~200 字）
正文 = 血肉（250-450 字，带情绪/互动/植入）
```

**正文不是把图片文字复制一遍，是换一种方式讲同一件事。**

### 正文模板

```
{emoji} {标题，口语化}

{共情开场：1-2 句}

{要点展开：每个 2-3 句，之间空一行}

{产品植入：自然过渡，1-2 个优点}

{互动钩子：开放问题}

———
市场有风险，投资需谨慎。
{场景化风险提示，见 COMPLIANCE.md}

# 话题1 # 话题2 # 话题3
```

### 正文 vs 图片分工

| 内容 | 图片 | 正文 |
|------|------|------|
| 核心观点 | ✅ 浓缩版 | ✅ 展开版 |
| 数据/佐证 | 图 4 走势图 | ✅ 文字解读 |
| 个人感受 | ❌ | ✅ 灵魂 |
| 产品植入 | 图 4 间接 | ✅ 自然植入 |
| 互动引导 | ❌ | ✅ 必须有 |
| 风险提示 | 每张底部简短 | 末尾完整版 |

## 工作流程

### Step 1 · 收集素材

研究部提供：主题、业绩走势图（必须）、产品信息、关键指标、标签。

**⚠️ 如果研究部只给了基金代码/名称而没给完整资料，直接跳到 Step 1.5 补充。**

### Step 1.5 · 研报补充（材料不足时必做）

**判断材料是否充足：**

| 情况 | 判断 | 动作 |
|------|------|------|
| 研究部给了完整产品资料 + 卖点 + 数据 | 充足 | 跳过，直接 Step 2 |
| 只给了基金名称/代码 + 走势图 | 不足 | **必须做研报补充** |
| 给了主题方向但没给具体数据 | 不足 | **必须做研报补充** |

**研报补充流程（Read `skills/research-mcp/SKILL.md`）：**

```bash
# 1. 拉基金基本信息
npx mcporter call "research-mcp.get_fund_basic(ts_code: '{基金代码}')"

# 2. 拉基金经理信息
npx mcporter call "research-mcp.get_fund_manager(ts_code: '{基金代码}')"

# 3. 搜相关研报（核心！用研报补充逻辑和观点）
npx mcporter call "research-mcp.search_report(keyword: '{基金名称或行业关键词}', limit: 5)"

# 4. 按需补充行业/宏观数据
npx mcporter call "research-mcp.get_index_daily(ts_code: '{相关指数}', limit: 5)"
npx mcporter call "research-mcp.get_macro_data(indicator: '{相关宏观指标}')"
```

**从研报中提取什么：**
- 行业逻辑/配置观点 → 用于图 2/3 的逻辑阐述
- 基金经理策略/风格 → 用于正文的产品植入段
- 行业数据/趋势判断 → 用于增加内容说服力
- **不要照搬研报原文**，用自己的人设语气改写

### Step 1.8 · Fact-check（材料不足时必做）

**当 Step 1.5 补充了研报数据后，必须跑 fact-check 验证。**

Read `skills/fact-check/SKILL.md`，对以下内容逐项验证：

- [ ] 基金收益率/回撤等指标是否与最新数据一致
- [ ] 研报中引用的行业数据是否准确
- [ ] 基金经理任职信息是否最新
- [ ] 宏观数据（GDP/CPI 等）是否用了最新值

**验证优先级：** research-mcp 直接查 > 联网搜索 > 标注为"未验证"

**⚠️ 未经验证的数据不得出现在图片和正文中。** 查不到的宁可不写，不要编。

### Step 2 · 规划方案

Read `CARDS.md`，按以下判断标准确定每张图的内容和布局，提交研究部确认。

**卡片数量判断：**

| 内容复杂度 | 卡片数 | 结构 |
|-----------|-------|------|
| 1 个核心逻辑 | 3 张 | 封面 + 逻辑A + 业绩 |
| 2-3 个逻辑点 | 4 张 | 封面 + 逻辑A + 逻辑B + 业绩 |

**布局选择：**
- 单一逻辑展开 → `FULL`
- 原因/策略、概念/落地、宏观/微观 两两对比 → `LR`
- 3-4 个平级概念并列 → `ROW`
- 拿不准 → `FULL`

**方案提交格式：**
```
图 1（封面）：标题 + 副标题 + 概念速览关键词
图 2（逻辑A）：主题 + 要点列表 + 示意图类型
图 3（逻辑B，如有）：主题 + 要点列表 + 布局模式
图 4（业绩）：基金名称 + 左栏要点 + 右栏走势图来源确认
```

**卡片数量决策（Read CARDS.md 后对照此表）：**

| 内容情况 | 图片数 | 布局方案 |
|---------|-------|---------|
| 只有 1 个核心逻辑点 | 3 张 | 封面 + 逻辑 A + 业绩 |
| 有 2-3 个逻辑点/需要正反对比 | 4 张 | 封面 + 逻辑 A + 逻辑 B + 业绩 |
| 逻辑点 > 3 且无法合并 | 4 张（合并至图 2/3）| 保持 4 张上限，合并同类 |

### Step 3 · 写正文

**⚠️ 写正文前必须 Read `COMPLIANCE.md`**，确认本次内容涉及哪些风险提示场景，再按正文模板的"场景化风险提示"栏填入对应条款。

250-450 字，按上方模板写。先 Read `skills/xhs-writing/SKILL.md`。

### Step 4 · 生图

Read `STYLE.md`，选对应模板逐张生成。

**图片保存规则：按商单建子文件夹，所有图片存在一起。**

```bash
# 文件夹命名：日期_基金/产品名称
# 例：memory/images/20260401_富国创新科技混合C/
mkdir -p ../../memory/images/{YYYYMMDD}_{产品名称简写}/
```

所有图片（图 1-3 AI 生成 + 图 4 业绩图）都存入同一个子文件夹：

```
memory/images/20260401_富国创新科技混合C/
├── 01_封面.png
├── 02_逻辑A.png
├── 03_逻辑B.png        （如有）
└── 04_业绩走势.png      （研究部提供的原图也复制进来）
```

生图调用时指定 workspace 路径：

```
npx mcporter call 'image-gen-mcp.generate_image(
  style: "{从 STYLE.md 选模板}",
  content: "{填入文字}",
  size: "1080x1440",
  workspace: "{bot workspace 路径}"
)'
```

生成后将图片移入商单子文件夹，研究部提供的业绩走势图也复制进来，保持一个商单一个文件夹。

### Step 4.5 · 图片质量自检（生图后必做）

每张生成图逐一检查：

| 检查项 | 通过标准 |
|-------|---------|
| 文字是否溢出边框 | 所有文字完整显示在卡片内 |
| 文字量是否超标 | 单图 ≤ 200 字（参照 CARDS.md 各图规格） |
| 风险提示是否存在 | 底部有「市场有风险，投资需谨慎。」+ 广告标 |
| 布局是否符合方案 | 与 Step 2 规划的布局模式一致 |
| 图 4 业绩区域 | 右栏为预留占位区，未用 AI 假图替代 |

**发现问题：** 重新调用生图，调整 prompt（文字减少/字号调小/布局调整），直至通过。
**效果好的 prompt 变体记录到 `memory/好用的图片prompt.md`。**

### Step 5 · 合规检查

**必须 Read `COMPLIANCE.md`**，逐项过 Checklist（图片 + 正文两个部分均需检查），通过后按 `skills/xhs-op/投稿发布.md` 投稿。

**快速记忆：每张图底部必须有风险提示 + 广告标；正文末尾必须有完整风险提示；图 4 需叠加场景化风险提示。**

### Step 4.5 · 图片质量自检（每张生成后立即做）

生成后逐张检查以下项目，不合格的重新生成：

| 检查项 | 标准 |
|-------|------|
| 文字可读性 | 所有文字清晰无模糊，字号不小于正文 14px 等效 |
| 文字溢出 | 标题/副标题/要点文字均在卡片边距内，无截断 |
| 排版完整 | 布局区域（TOP / MIDDLE / FOOTER）按比例呈现，无空白异常 |
| 风险提示 | 底部小字"市场有风险，投资需谨慎。"和"广告"标签可见 |
| 无 AI 生成假图 | 图 4 右栏显示占位框而非 AI 绘制的走势图 |

**不合格处理：** 调整 prompt（增加尺寸约束、减少文字量）后重新生成；生成成功的 prompt 记录到 `memory/好用的图片prompt.md`。
