# 日复盘

**篇幅**：1500-2500字
**适用**：交易日收盘后，当天市场回顾+观点
**范文**：`review-output/公众号_2026-03-19.md`（滞胀交易主题）

---

## 一、素材采集

统一使用 research-mcp 采集数据（`npx mcporter call "research-mcp.tool_name(...)"`）。

### 第一步：市场全景（必做）

```bash
# 1. A股大盘总览（指数涨跌、成交额、板块轮动）
npx mcporter call "research-mcp.market_overview()"

# 2. 主要指数详细行情（近期走势对比）
npx mcporter call "research-mcp.get_ashares_index_quote(symbol='000001.SH,000300.SH,399006.SZ,000905.SH', start_date='YYYYMMDD', end_date='YYYYMMDD')"

# 3. 全市场成交额（判断量能）
npx mcporter call "research-mcp.get_ashares_turnover(start_date='YYYYMMDD', end_date='YYYYMMDD')"

# 4. 恐慌指数（市场情绪）
npx mcporter call "research-mcp.get_ashares_gvix(start_date='YYYYMMDD', end_date='YYYYMMDD')"
```

采集到的数据模块：
| 模块 | research-mcp 工具 | 说明 |
|------|-------------------|------|
| 市场全景 | `market_overview` | 主要指数涨跌、成交额、板块概况 |
| 指数行情 | `get_ashares_index_quote` | 指数日/周/月涨跌幅 |
| 量能判断 | `get_ashares_turnover` | 全市场成交额走势 |
| 情绪指标 | `get_ashares_gvix` | 恐慌指数走势 |
| 指数估值 | `get_ashares_index_val` | PE_TTM + 历史百分位 |
| 债券收益 | `get_cn_bond_yield(maturity='10Y')` | 国债收益率（算 ERP 用） |

### 第二步：资金面（按需）

```bash
# 北向资金（个股级别）
npx mcporter call "research-mcp.get_stock_northbound_holding(stock_code='600519', start_date='YYYYMMDD', end_date='YYYYMMDD')"

# 港股通南向资金
npx mcporter call "research-mcp.get_southbound_hkd_turnover(start_date='YYYYMMDD', end_date='YYYYMMDD')"

# 个股资金流向（主力、超大单等）
npx mcporter call "research-mcp.get_stock_fund_flow(stock_code='600519', start_date='YYYYMMDD', end_date='YYYYMMDD')"
```

### 第三步：个股/板块数据（按需）

```bash
# 个股日K线
npx mcporter call "research-mcp.get_stock_daily_quote(stock_code='600519', start_date='YYYYMMDD', end_date='YYYYMMDD')"

# 个股估值
npx mcporter call "research-mcp.get_stock_valuation(stock_code='600519')"

# 个股行业分类
npx mcporter call "research-mcp.get_stock_industry(stock_code='600519')"
```

> **板块历史行情**：当主题涉及某板块逻辑时，可通过查该板块代表性个股（龙头股）的 `get_stock_daily_quote` 近 30 日走势来建立上下文，也可用 `get_ashares_index_quote` 查对应行业指数。分析角度：近期累计涨跌幅、区间高低点位置、成交额变化趋势、对比大盘表现。

> **原则：聊板块逻辑时，读者需要知道"这个板块从哪来、现在在哪、可能往哪去"，而不仅仅是"今天涨了/跌了"。**

### 板块逻辑解读（不许瞎猜，必须有据）

讨论板块逻辑时，必须先搜索外部解读：

1. **搜研报（research-mcp）：** `npx mcporter call "research-mcp.research_search(query='{板块名} 行业', top_k=3, search_day_ago=30)"`
2. **搜产业新闻（research-mcp）：** `npx mcporter call "research-mcp.news_search(query='{板块名} 产业 动态', top_k=5, search_day_ago=7)"`
3. **查知识星球：** 用 `zsxq-reader` skill 搜索该板块关键词
4. **补充搜索：** WebSearch `"{板块名}" 产业 最新动态`，交叉验证

拿到外部解读后，结合盘面数据拆解板块节奏：
- 研报逻辑和盘面走势是否一致？
- 板块处于什么阶段？（预期驱动 vs 业绩验证 vs 资金博弈）
- 当前节奏是主升、震荡还是退潮？

> **铁律：板块逻辑阐述必须有来源（研报、知识星球、产业新闻），不允许凭空编造产业逻辑或猜测政策方向。引用时标注来源。**

## 二、从数据到观点

**拿到数据后不要急着写，先做三件事：**

### 2.1 找今天的"主故事"

从复盘数据中找出今天最有解读空间的 1-2 个信号：

| 信号来源 | 怎么找 | 什么算好信号 |
|---------|--------|------------|
| 指数走势分化 | 上证 vs 创业板涨跌方向相反 | 风格切换、资金迁移 |
| 日内画像反转 | 上午跌下午涨（V）或相反（倒V） | 盘中发生了什么事？ |
| 板块异动 | TOP5 有新面孔，或某板块连续多日出现 | 新主线还是一日游？ |
| 情绪极端值 | 情绪评分 > 75（过热）或 < 30（冰点） | 拐点信号 |
| 涨跌停结构 | 涨停数暴增/暴减、跌停扎堆某行业 | 情绪拐点或风险暴露 |
| 资金方向 | 北向大进大出、融资余额突变 | 增量/减量信号 |
| 成交额变化 | 对比前一日（放量/缩量显著时） | 量能是否支持当前走势 |

**选择标准：选"读者明天开盘前最想知道的事"，而非"今天数据最大的变化"。**

### 2.2 形成观点

围绕主故事回答三个问题：
1. **发生了什么？**（盘面现象，1-2 句话描述）
2. **为什么发生？**（背后逻辑，政策/资金/情绪/事件驱动？）
3. **意味着什么？**（对后市的影响，对操作的启示）

### 2.3 选新闻

从搜索结果中选 3-5 条满足以下条件的新闻：
- 与主故事相关，或与投资决策直接相关
- 能给出差异化点评（不能只转述）
- 排除纯情绪性标题党、非财经新闻

## 三、文章结构

```
├── 标题（6字以内，口语化，带情绪但不标题党）
├── 核心观点引语（1句话，blockquote格式，点出今天最重要的信号）
├── 一句话概览：指数涨跌 + 成交额 + 涨跌家数，3 行内搞定
├── 主题段：今天最值得聊的 1-2 个话题（占全文 60%，800-1500 字）
│   ├── 不是流水账，而是围绕一个主题深入展开
│   ├── 从盘面现象 → 背后逻辑 → 对操作的影响
│   ├── 数据嵌入叙述中（不要列表格）
│   └── 穿插自身操作动态（如有）
├── 反向信号/值得注意的点（可选，200-300 字）
│   └── 与主流叙事相反的数据，避免单一视角
├── 新闻简评：3-5 条精选新闻 + 每条 2-3 句点评
│   └── 点评角度：这条新闻对投资有什么影响？
├── 结尾（简短收束，明天关注什么）
└── 风险提示（固定模板）
```

---

## 四、各段落写法

### 4.1 一句话概览

**模板：**
```
今天大 A {涨跌描述}。{主要指数涨跌}，成交 {金额} 亿，{涨跌家数描述}。
```

**示例：**
> 今天大 A 继续分化。上证微跌 0.26%，创业板涨 1.41%，成交 2.3 万亿，涨多跌少。

不要写成数据报表，用自然语言一句话概括。

### 4.2 主题段

**按信号类型选择数据组合：**

| 信号类型 | 需要的数据 | research-mcp 工具 |
|---------|-----------|-------------------|
| 风格切换 | 指数对比 + 板块概况 | `market_overview` + `get_ashares_index_quote` |
| 板块异动 | 板块排名 + 龙头股走势 | `market_overview` + `get_stock_daily_quote`（龙头股近30日） |
| 情绪拐点 | 恐慌指数 + 成交额变化 | `get_ashares_gvix` + `get_ashares_turnover` |
| 量能变化 | 成交额走势 | `get_ashares_turnover`（拉近期对比） |
| 资金信号 | 北向 + 国债收益率 + 指数估值 | `get_stock_northbound_holding` + `get_cn_bond_yield` + `get_ashares_index_val`（算 ERP） |
| 政策驱动 | 新闻 + 研报 + 相关个股走势 | `news_search` + `research_search` + `get_stock_daily_quote` |

**写作要点：**
- 不是把数据翻译成文字，而是用数据支撑你的判断
- 数据嵌入叙述中，不要单独列表格
- 引用具体数字时，给出简短解读："创业板涨了 1.4%，这已经是连续第三天跑赢大盘了"
- 讨论板块逻辑时，必须查龙头股近期走势和相关指数，读者需要知道"从哪来、现在在哪、可能往哪去"
- 板块逻辑不许瞎猜，必须用研报+新闻+知识星球交叉验证（详见上方"板块逻辑解读"）

### 4.3 新闻简评

**每条新闻格式：**
```
{新闻标题/事件}。{2-3 句点评，角度：这事对投资有什么影响}
```

**示例：**
> 央行今日净投放 2000 亿逆回购。跨季资金面本来就紧，这个量不算小，说明央行在主动呵护。短期利率中枢大概率稳住，对债市偏利好。

### 4.4 结尾

- 总结今天的盘面调性（一句话）
- 提一个明天值得关注的变量
- 可以用问句收尾（制造期待感）

**示例：**
> 今天的分化说明市场还在选方向，资金从周期往科技搬家的趋势第三天了。明天重点看创业板能不能放量站稳 3350，如果缩量回踩，可能只是反抽。

## 五、标题生成

从主题段中提炼标题。

**通用规律：**
- 字数：6 字以内
- 风格：口语化 + 情绪化 + 不透露全部内容
- 好标题举例："苟住！""有点麻。。""靴子落了""衰退来了？"
- 坏标题举例："3月18日A股市场复盘""今日大盘分析"（太干，没人想点）

**按盘面特征选标题方向：**

| 今日盘面特征 | 标题方向 | 示例 |
|------------|---------|------|
| 分化明显 | 口语反问 | "选谁？" |
| 大涨 | 情绪+克制 | "久违大阳。" |
| 大跌 | 自嘲/调侃 | "又学到了。" |
| 窄幅震荡 | 悬念 | "等个信号" |
| 板块异动 | 聚焦 | "种子疯了" |
| 趋势延续 | 判断 | "还没搬完" |
| 政策驱动 | 反问 | "降准之后？" |

---

## 六、关键原则

1. 主题驱动，不做流水账 — 每天选最有价值的 1-2 个话题深入聊
2. 有自己的判断 — 不是转述新闻，而是给出观点
3. 操作透明 — 有买卖动作就说，做错了也说
4. 敢说不知道 — "这个问题很复杂，我也没想清楚"比装懂更有信任感

## 七、写作流程

```
1. 拉数据    → 第一步：市场全景（必做）
               ↓
2. 补资金面  → 第二步：资金面（按需）
               ↓
3. 搜新闻    → news_search + WebSearch
               ↓
4. 找故事    → 第二章：从数据到观点
               ↓
5. 形成观点  → 发生了什么 → 为什么 → 意味着什么
               ↓
6. 写文章    → 第三/四章：结构 + 各段落写法
               ↓
7. 优化标题  → 第五章：标题生成
               ↓
8. 自检+核查 → 回到 SKILL.md Step 4-5
               ↓
9. 交付      → 回到 SKILL.md Step 6
```
