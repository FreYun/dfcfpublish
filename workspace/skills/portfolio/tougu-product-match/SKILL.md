# 投顾产品匹配 /tougu-product-match

**触发词**: `/tougu-product-match`, "你的投顾组合是什么", "你买了哪些投顾产品", "重新选品"

**定位**: 通用共享 skill。为当前 bot 自己做买方决策，从候选产品池中选出最适合的产品组合。每次执行都基于最新产品池、当前投资画像和最新市场状态重新选品，结果覆盖写入 `memory/portfolio/个性化投顾产品选择.md`。

**运行态要求**:
- 若任务上下文给了 `run_id / trade_date / allocation_id`,必须沿用
- Phase B 的正式输出是 `portfolio_plans`,不是 markdown
- markdown 只是归档展示；供 Phase C 真正读取的是 `portfolio_plans.plan_json`

**适用范围**: 所有 bot 共享。不同 bot 之间只允许复用方法，不允许复用结论。每个 bot 都必须基于自己的 `SOUL.md`、`IDENTITY.md`、`USER.md` 以及 `memory/` 重新判断。

**边界说明**:

- 用户问"你自己的投顾组合/你买了哪些"时触发本 skill
- 用户问"帮我选产品/我该买哪个"时**不应**触发
- 回答口径始终是第一人称买方视角，不是面向用户做推荐

## 输入约定

### 1. 候选产品池

优先级：

1. **MCP 数据库**（推荐）：调用 `tougu-portfolio-mcp.get_product_pool()` 获取产品池，再用 `get_product_detail(product_id)` 补充详情和绩效
2. 用户在当前任务中明确给出的产品池文件
3. 测试阶段的 `memory/portfolio/` 下的候选池 markdown

如果以上都没有，**停止执行**，输出"缺少候选产品池，无法完成产品匹配"。

### 2. 候选产品最小字段

每个产品至少应尽量提供：

- 产品名称、产品代码
- 主要赛道 / 资产类型
- 波动 / 回撤 / 收益风险特征
- 管理人
- 可比较的综合评分或底层指标
- 最低买入金额（min_deposit）、最高买入金额（max_deposit）
- 申购状态（buy_status）

缺少关键字段不要编造，标注信息缺失并降低置信度。

### 3. 记忆来源

按以下优先级读取：

1. `memory/portfolio/投资策略摘要.md`（主入口，存在则必须先读）
2. `SOUL.md`、`IDENTITY.md`、`USER.md`
3. `memory/long_term/`
4. `memory/research/`
5. `memory/posts/`

摘要足够则不全量扫描 memory。只在摘要缺失、过期、或与近期记录明显冲突时回看原始文件。

### 4. 市场状态 / 择时背景

优先级：

1. **MCP 数据库 / allocation_runs**（推荐）：读取最近一次 `market-context` 的输出，至少拿到 `regime / regime_code / confidence / timing_stance / asset_target_json`
2. `memory/portfolio/市场环境判断.md`
3. 若以上都缺失：允许继续选品，但必须标注"缺少市场状态，按中性环境处理"，默认 `timing_stance = hold`

这里的市场状态只决定**新产品什么时候建仓**，不直接替 bot 下结论买什么。

## 记忆使用原则

### A. `memory/long_term/` = 稳定偏好层

提取长期稳定的投资偏好：风险承受方式、主动 vs 被动、均衡 vs 集中、ETF 偏好、分红/低波偏好、明确不买的方向。

内容运营规则、排版规范等只影响表达风格，不影响投资判断。

### B. `memory/research/` = 高置信度能力圈层

判断 bot 真正懂哪些赛道。条件：有成体系研究、能解释驱动逻辑和风险、有独立判断。

`research/` 为空不代表没有能力圈，只代表高置信度证据不足。

### C. `memory/posts/` = 近期行为校准层

观察 bot 最近关注什么、如何表达风险、偏进攻还是偏防守。只用于校准，写过不等于懂。

## 执行流程

### Step 1: 读取投资策略摘要

读取 `memory/portfolio/投资策略摘要.md`，提取：

- 稳定画像、核心/卫星风险带、能力圈分层、产品偏好、禁忌、简版结论

### Step 2: 构建投资画像中间层

基于摘要 + 必要的原始记忆，压缩成结构化画像：

- 风险承受力（结论 + 证据 + 置信度）
- 偏好产品类型
- 投资理念
- 能力圈赛道
- 回避赛道
- 当前持仓/已知暴露
- 投资目标
- 近期风格倾向

### Step 3: 区分硬信号和软信号

**硬信号**（直接影响核心配置）：长期偏好、深度研究赛道、明确投资原则、已知持仓

**软信号**（只微调权重）：近期发帖主题、风险措辞、市场情绪

冲突时以硬信号为准。

### Step 4: 判断真实风险带

输出两个层次：

- **核心风险带**：承载大部分仓位
- **卫星风险带**：少量配置增强弹性

### Step 5: 建立认知深度分层

- `A档：深度能力圈` → 可进入核心候选
- `B档：持续跟踪` → 原则上只做卫星
- `C档：仅话题接触` → 原则上不纳入，除非产品极度均衡

### Step 6: 五维匹配打分

对候选产品逐个评分（1-10 分）：

| 维度 | 权重 | 核心问题 |
|------|------|---------|
| 风险匹配度 | 30% | 波动和回撤，bot 扛不扛得住？ |
| 认知匹配度 | 25% | bot 对产品赛道真懂、半懂还是不懂？ |
| 理念匹配度 | 20% | 产品投资方式和 bot 偏好是否一致？ |
| 产品硬实力 | 15% | 产品本身在同类里是否够强？ |
| 组合互补性 | 10% | 加入后是丰富组合还是高度重复？ |

每个分项附短依据。没数据时标注"低置信度"。

### Step 7: 市场状态映射为建仓闸门

根据 `market-context` 输出的 `timing_stance`，对新产品建仓做闸门判断：

| timing_stance | 建仓口径 | 执行含义 |
|------|------|------|
| `aggressively_add` | 立即建仓 | 允许按目标比例正常买入 |
| `add_on_pullback` | 分批建仓 | 允许纳入目标组合，但高波动产品默认分两到三次买 |
| `hold` | 谨慎观察 | 产品可以入选，但若优势不显著则延后实际买入 |
| `defensive` | 防守准入 | 只允许低波、固收、避险或明显降低组合风险的产品立即买入 |
| `risk_off` | 原则上延后 | 原则上不新增高波动权益产品，必要时只保留现金/低波防守位 |

补充规则：

- `timing_stance` 约束的是**新买入节奏**，不是否定 bot 的长期偏好
- 强画像匹配但时机一般的产品，可以进入目标组合，但执行建议写成"分批买"或"延后买"
- 若 `confidence=low`，所有新增高波动产品最多只能给到"分批建仓"，不能给"立即建仓"

### Step 8: 构建组合

- 核心持仓 1-3 个，卫星 0-2 个，不强制凑数
- 允许保留空仓/现金位
- 不为分散买不懂的产品
- 不为追热点突破核心风险带
- 买入金额需满足产品 min_deposit / max_deposit 约束（仅约束买入时的单笔金额，持有后市值增长不受此限制）
- 暂停申购（buy_status = 暂停申购）的产品不纳入新组合
- `asset_target_json` 若存在，优先用它约束权益/固收/黄金/现金的大类方向，再落到具体产品
- 对每个新入选产品都要给出执行标签：`立即买` / `分批买` / `延后买`

### Step 9: 文风改写层

文风放在决策之后。从 SOUL.md / IDENTITY.md / posts 提取表达风格。

**文风只影响怎么说，不影响选什么。**

### Step 10: 覆盖写入报告

**覆盖写入** `memory/portfolio/个性化投顾产品选择.md`。每次选品都生成新版本，不保留旧版。

### Step 11: 正式写入 portfolio_plans

在生成目标组合 markdown 后，必须把结构化结果写入数据库。

最小 `plan_json` 结构建议：

```json
[
  {
    "product_id": "AILVXKB",
    "product_name": "嘉实百灵全天候",
    "target_weight": 30,
    "target_amount": 30000,
    "role": "核心底仓",
    "entry_action": "立即买",
    "thesis": "..."
  }
]
```

然后调用：

```bash
npx mcporter call "tougu-portfolio-mcp.save_portfolio_plan(run_id: 'manual-bot1-2026-04-23', bot_id: 'bot1', trade_date: '2026-04-23', allocation_id: 123, plan_json: '[{\"product_id\":\"AILVXKB\",\"product_name\":\"嘉实百灵全天候\",\"target_weight\":30,\"target_amount\":30000,\"role\":\"核心底仓\",\"entry_action\":\"立即买\",\"thesis\":\"...\"}]', plan_md: '...')"
```

如果拿不到 `allocation_id`,可以先写 `allocation_id = 0`,但优先要求 Phase B0 先落 `allocation_runs`。

## 硬约束

1. **你是在替 bot 自己选，不是替用户选**：第一人称买方视角
2. **每次都重新选品**：不复用旧结果，基于最新产品池和画像重新匹配
3. **先做结构化画像，再做选品**：不从原始记忆直接跳到结论
4. **行为优先于口号**：嘴上稳健但行为追高，按行为校准
5. **长期偏好优先于短期热点**：long_term 和 research 权重高于 posts
6. **研究深度优先于提及频率**：提过不等于懂，商单不等于能力圈
7. **证据不足时允许不知道**：不编造人格、持仓或产品数据
8. **只从候选池中选**：禁止引入池外产品
9. **风格层与决策层分离**：文风改写不能反推选择
10. **允许结果是"少买"或"不买"**：没有合适产品时不强行构建
11. **买入需校验金额约束**：min_deposit / max_deposit / buy_status（仅约束买入，持有后不适用）
12. **所有关键判断留证据**：至少写到文件或记忆来源层级
13. **市场状态只约束节奏，不直接决定偏好**：择时不能覆盖 bot 的人格画像
14. **优先读取 allocation_runs**：能从数据库拿最新市场状态时，不依赖旧 markdown
15. **Phase B 正式结果必须入库**：生成目标组合后必须调用 `save_portfolio_plan`

_通用投顾产品匹配能力，所有 bot 共享。修改此文件会影响全部 bot。_

- 最低买入金额（min_deposit）、最高买入金额（max_deposit）
- 申购状态（buy_status）

缺少关键字段不要编造，标注信息缺失并降低置信度。

## 硬约束

1. **你是在替 bot 自己选，不是替用户选**：第一人称买方视角
2. **每次都重新选品**：不复用旧结果，基于最新产品池和画像重新匹配
3. **先做结构化画像，再做选品**：不从原始记忆直接跳到结论
4. **行为优先于口号**：嘴上稳健但行为追高，按行为校准
5. **长期偏好优先于短期热点**：long_term 和 research 权重高于 posts
6. **研究深度优先于提及频率**：提过不等于懂，商单不等于能力圈
7. **证据不足时允许不知道**：不编造人格、持仓或产品数据
8. **只从候选池中选**：禁止引入池外产品
9. **风格层与决策层分离**：文风改写不能反推选择
10. **允许结果是"少买"或"不买"**：没有合适产品时不强行构建
11. **买入需校验金额约束**：min_deposit / max_deposit / buy_status（仅约束买入，持有后不适用）
