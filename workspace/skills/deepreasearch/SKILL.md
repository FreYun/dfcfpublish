# 深度研究技能

## 核心理念

**深度研究 ≠ 简单的信息收集**

深度研究是系统性工程，要求：
1. **工具规范** — 严格使用指定工具（browser + MCP）
2. **流程迭代** — 多轮"查询-研究-核查"循环
3. **事实核查** — 关键信息必须交叉验证
4. **逻辑验证** — 推理链条必须完整可追溯
5. **成果留存** — 研究结果必须记录到 memory 文件

---

## 工具使用规范 ⭐⭐⭐

### 🔴 浏览器工具（Browser）

**必须使用 `profile="openclaw"` — 隔离浏览器模式**

```
永远使用隔离模式，不依赖 Chrome 会话
```

**适用场景：**
- 访问雪球（需登录态的网站）
- 东方财富网、同花顺、新浪财经
- 任何需要交互的网站
- web_fetch 失败时的备选

**示例：**
```
browser(action="open", targetUrl="https://xueqiu.com", profile="openclaw")
browser(action="snapshot", profile="openclaw")
browser(action="act", request={...}, profile="openclaw")
```

---

### 🔵 MCP 服务

**根据研究类型选择合适的 MCP：**

| 研究类型 | MCP 服务 | 用途 |
|---------|---------|------|
| 基金数据 | 且慢 MCP | 基金净值、持仓、业绩 |
| 网络搜索 | WebSearch MCP | 通用信息检索 |
| 其他 | 根据配置 | 按需调用 |

**优先级：**
1. MCP 专业数据源（且慢、WebSearch）
2. Browser（雪球等需交互的网站）
3. web_fetch（简单网页抓取）

## 研究流程（迭代循环法）

### 核心循环

```
┌─────────────────────────────────────────┐
│  查询 → 思考研究 → 核查 → 查询 → ...   │
│        ↑                        │       │
│        └────────────────────────┘       │
│           持续迭代直到满意               │
└─────────────────────────────────────────┘
```

**每一轮循环包括：**
1. **查询** — 收集新信息
2. **思考研究** — 分析和整合
3. **核查** — 验证事实和逻辑
4. **决策** — 继续迭代 or 输出成果

---

### 📍 步骤1：明确研究目标

**必须回答的问题：**

```
1. 研究什么？（主题/问题/现象）
2. 为什么研究？（目的/背景/动机）
3. 需要什么信息？（数据/观点/事实）
4. 如何判断完成？（完成标准）
```

**输出：** 研究框架（明确范围和目标）

---

### 🔍 步骤2：第一轮查询

**查询策略：**

```
优先级：
1. MCP 专业数据（如果有）
2. Browser 隔离模式（雪球等）
3. web_fetch（简单抓取）

信息类型：
- 基础事实（数据、事件）
- 专业观点（分析、评论）
- 市场反应（情绪、讨论）
```

**记录：** 收集到的所有信息（标注来源和时间）

---

### 🧠 步骤3：思考研究

**分析框架：**

```
1. 事实整理
   - 已知什么？
   - 未知什么？
   - 矛盾什么？

2. 逻辑推演
   - 因果链条是什么？
   - 假设是什么？
   - 风险点在哪？

3. 知识整合
   - 与已有知识如何关联？
   - 有什么新的发现？
   - 哪些需要进一步验证？
```

**输出：** 初步结论 + 待核查清单

---

### ✅ 步骤4：核查（Fact Check + Logic Check）

**Fact Check 清单：**

```
□ 时间准确性
  - 事件发生时间已确认
  - 区分历史事件和当前事件
  - 时间戳已标注

□ 事实真实性
  - 多个独立信息源交叉验证（至少2个）
  - 官方/权威来源已确认
  - 没有未经证实的传言

□ 数据准确性
  - 数值已核对
  - 单位和口径正确
  - 数据来源可靠
```

**Logic Check 清单：**

```
□ 逻辑完整性
  - 因果链条完整
  - 没有逻辑跳跃
  - 推理步骤可追溯

□ 假设合理性
  - 假设明确
  - 假设可靠
  - 敏感性分析

□ 结论稳健性
  - 多种情景考虑
  - 风险识别
  - 反例验证
```

---

### 🔄 步骤5：判断是否继续迭代

**决策标准：**

```
继续迭代，如果：
- ❌ 有关键事实未核实
- ❌ 有逻辑链条不完整
- ❌ 有矛盾信息未解决
- ❌ 研究目标未达成

输出成果，如果：
- ✅ 所有关键事实已核实
- ✅ 逻辑链条完整
- ✅ 无明显矛盾
- ✅ 研究目标达成
```

**如果继续迭代 → 回到步骤2，针对性查询**

## 核查技巧

### 交叉验证法

**至少 2 个独立信息源：**

```
来源1：雪球用户讨论
来源2：官方公告/新闻报道
结论：一致 → 可信 / 不一致 → 需进一步核查
```

---

### 时间追溯法

**永远先问"什么时候"：**

```
❌ 错误："今天发生X事件"
✅ 正确："X事件发生在YYYY-MM-DD，今天是后续发展"

步骤：
1. 搜索事件起始时间
2. 确认是否为历史事件
3. 标注准确时间戳
```

---

### 逻辑链条法

**逐步验证因果链条：**

```
A → B → C → D（结论）

验证：
- A → B：是否有充分证据？
- B → C：逻辑是否合理？
- C → D：是否必然？

任何环节有问题 → 整个结论存疑
```

---

### 反例验证法

**寻找推翻假设的证据：**

```
假设：X → Y

主动寻找：
- 有没有 X 发生但 Y 没发生的情况？
- 有没有 Y 发生但 X 没发生的情况？
- 有没有其他因素导致 Y？

找到反例 → 假设需修正
```

## 常见错误和纠正

### 错误1：单次查询就下结论

```
❌ 查了一次雪球，看了几条评论，就下结论
✅ 多轮迭代，交叉验证，逻辑核查
```

**纠正：** 必须至少进行 2-3 轮"查询-研究-核查"循环

---

### 错误2：不区分信息时效

```
❌ "雪球上说..."（没看发布时间）
✅ "雪球用户@XXX在YYYY-MM-DD HH:MM发布..."
```

**纠正：** 所有信息必须标注时间戳

---

### 错误3：逻辑跳跃

```
❌ "A涨了，所以B也会涨"
✅ "A涨 → 需求增加 → B供给受限 → B价格可能上涨（需验证）"
```

**纠正：** 检查每一步推理是否有依据

---

### 错误4：只收集不核查

```
❌ 收集了大量信息，但没有核实真伪
✅ 收集 → 交叉验证 → 逻辑核查 → 再收集
```

**纠正：** 核查是研究的一部分，不是可选步骤

---

### 错误5：成果未保存

```
❌ 研究完就结束
✅ 立即记录到 memory 文件
```

**纠正：** Text > Brain，不记录等于没研究
