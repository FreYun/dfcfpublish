# SKILL - 投研热点小红书发帖（完整流程）

## 概述

基于每日热点解读的分析结果，完成从内容生成 → 封面生图 → 投稿发布的完整小红书发帖流程。

- 发帖风格参考「Bool 资本不眠」（详见 `memory/long_term/Bool 资本不眠 - 发帖风格分析.md`）
- 封面生图参考 `skills/laicaimeimei-fupan/SKILL-image.md`
- 投稿发布参考 `skills/submit-to-publisher/SKILL.md`

## 触发条件

- 用户要求把热点解读发到小红书

## 前置条件

- 已完成当日热点解读（`skills/laicaimeimei-fupan/SKILL.md`），产出两个报告文件
- `image-gen-mcp` 可用（封面生图）

## 核心工作流

### Phase 1：内容生成

#### Step 1：选题

1. 读取当日热点解读报告（`memory/posts/YYYY-MM-DD.md`）
2. 读取雪球讨论原文（`memory/posts/YYYY-MM-DD-帖子内容.md`）
3. 读取风格参考（`memory/long_term/Bool 资本不眠 - 发帖风格分析.md`）
4. 调用 `list_notes` 获取最近一篇已发帖子，读取其内容，对照风格
5. 从热点中选择最有传播力的 1-2 个话题

**话题选择优先级：**
1. 有反常识观点的 — 打破常规认知，容易引发讨论
2. 有对比的 — A vs B、过去 vs 现在
3. 蹭上热点事件的 — 大涨大跌、政策利好、熔断等
4. 有实操建议的 — 读者能直接用得上

#### Step 2：生成标题（< 20 字）

选择最合适的标题公式：

| 公式 | 格式 | 示例 |
|------|------|------|
| 疑问式 | [主题] + 吗/如何/为什么？ | 黄金股与黄金，如何选择？ |
| 反差式 | [过去状态] + 现在/这次 + [转折] | 煤化工，一直在蛰伏，现在迎来顺风 |
| 热点式 | [热点事件] + ，+ [解读角度] | 韩国股市，上涨熔断 |
| 结论式 | [核心观点] + ；+ [补充说明] | 布局有色，为什么要买有矿的 |

#### Step 3：生成配图文字

- 1 行，简短有力
- **必须包含当日热点行业或板块名称**（如「黄金」「半导体」「新能源」等）
- 点明核心观点即可，不要堆砌内容
- 这段文字将作为封面图上的大字

#### Step 4：撰写正文

```
【开篇点题】1-2 句，点明核心观点或今日市场现象
    ↓
【概念解释/背景介绍】科普基础知识，降低理解门槛（如有需要）
    ↓
【核心分析】数据 + 逻辑，分点论述（2-4 点）
  - 引用具体数据，标注来源
  - 使用分析框架（双轮驱动/供需分析/三周期等）
    ↓
【投资建议/策略】给出具体配置思路（用指数基金代替个股）
    ↓
【风险提示】简短提醒（合规需要）
```

#### Step 5：组装标签（8-10 个）

顺序：垂直标签 → 热点标签 → IP 标签

- **垂直标签**：根据内容选择行业标签（如 #黄金 #有色 #创新药）
- **热点标签**：当日热点相关（如 #A股 #今日复盘）
- **固定标签**：#我的炒股日记 #股票 #基金

#### Step 6：内容复核（个股名称脱敏）

逐一检查标题、配图文字、正文中是否出现个股名称。如出现，将第一个字替换为其大写拼音首字母。

**替换规则：** 个股名称的第一个汉字 → 该字的拼音首字母（大写），其余字保留不动。

**示例：**

| 原文 | 替换后 |
|------|--------|
| 宁德时代 | N德时代 |
| 比亚迪 | B亚迪 |
| 贵州茅台 | G州茅台 |
| 隆基绿能 | L基绿能 |
| 中芯国际 | Z芯国际 |

**检查范围：** 标题、配图文字、正文全文。标签中如有个股名称同样替换。

#### Step 7：用户确认内容

将标题、配图文字、正文、标签展示给用户，等待确认或修改。

---

### Phase 2：封面生图

> 完整流程见 `skills/laicaimeimei-fupan/SKILL-image.md`，以下为摘要。

#### Step 8：判断情绪 → 生成封面图

1. 根据帖子内容判断情绪类型（大涨 / 大跌 / 通用）
2. 读取 `memory/branding/cover-prompt.md`，取对应 base prompt
3. 将配图文字替换到 prompt 中的文字位置
4. 拆分为 `style` + `content`，调用 `image-gen-mcp.generate_image`：
   ```
   工具：image-gen-mcp.generate_image
   参数：
     style: "{风格描述}"
     content: "{内容描述，含配图文字}"
     size: "1024x1536"
     model: "banana"
   ```

#### Step 9：保存封面图

将生成的图片复制到 `memory/posts/`：

```bash
DATE=$(date +%Y-%m-%d)
DEST_DIR="memory/posts"
BASE="${DEST_DIR}/${DATE}-封面图"
if [ ! -f "${BASE}.png" ]; then
  DEST="${BASE}.png"
else
  N=1
  while [ -f "${BASE}-${N}.png" ]; do N=$((N+1)); done
  DEST="${BASE}-${N}.png"
fi
cp "{generate_image 返回的 files[0]}" "$DEST"
```

#### Step 10：用户确认封面图

1. 告知用户封面图路径
2. 中文渲染乱码是常见情况，可后期叠字
3. 不满意可调整后重新生成

---

### Phase 3：投稿发布

#### Step 11：提交到印务局

用户确认标题、正文、封面图均 OK 后，以 **image 模式**（图文帖）提交到发布队列：

```bash
cat > /tmp/post_body_$$.txt << 'BODYEOF'
{正文内容，完整不得省略}
BODYEOF

folder=$(bash skills/xhs-op/submit-to-publisher.sh \
  -a bot1 -t "{标题}" -b /tmp/post_body_$$.txt \
  -m image -r "direct:ou_xxx" \
  -i "{Step 9 的封面图路径}" \
  -T "{标签1,标签2,...}")
echo "FOLDER: $folder"
```

> **⚠️ body 文件写入和 submit 脚本必须在同一个 bash block 执行**，否则 `$$` PID 不一致会找不到文件。

> **⚠️ body 文件必须包含完整正文，不得压缩、省略、截断。**

#### Step 12：通知用户

回复：「《{标题}》已提交印务局，发布结果稍后通知。」

**任务到此结束，不需要等待印务局响应。**

## 内容红线

- 不直接荐股，用「指数基金」「板块方向」代替个股
- 不承诺收益
- 不暴露具体金额
- 数据标注来源（**不得出现「东方财富」「东方财富网」**，如需引用其数据用「Wind」「公开数据」等替代）
- **只描述指数涨跌幅，不说具体多少家上涨/下跌**，用模糊化语言代替（如「大部分上涨」「多数飘绿」「普涨」「跌多涨少」）
- 加风险提示
