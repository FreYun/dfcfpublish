# 研报对比 /report-compare

**触发词**：`/report-compare`、"这几份研报对比一下"、"不同券商怎么看"、"各家观点有什么不同"、用户上传 2+ 份同标的/同行业研报时自动建议

**定位**：在多份研报之间找分歧、找共识。一致预期往往已被 price in，真正有价值的是分歧点和边际变化。

---

## 执行流程

### Step 0 — 预处理

对每份研报先快速跑一遍 `report-digest` 的 Step 1-3（基本信息 + 核心观点 + 关键数据），但不需要完整输出，仅作为对比的数据基础。

确认研报的可比性：
- 同一标的/行业 → 直接对比
- 不同标的但同行业 → 以行业视角对比
- 完全不同领域 → 提醒用户，按需继续

---

### Step 1 — 基本信息对照表

```markdown
| 维度 | 研报A | 研报B | 研报C |
|------|-------|-------|-------|
| 券商 | | | |
| 分析师 | | | |
| 日期 | | | |
| 评级 | | | |
| 目标价 | | | |
| 当前价（报告时） | | | |
| 隐含涨幅 | | | |
```

---

### Step 2 — 观点共识区（一致预期）

找出各研报共同认可的观点，列出：
- 共识结论
- 支撑共识的核心逻辑

**点评**：一致预期通常已反映在股价中，标注"是否已充分 price in"。

---

### Step 3 — 观点分歧区（核心价值）

这是本 skill 最重要的部分。逐一列出分歧点：

```markdown
### 分歧 1：{主题}
- **券商A 观点**：xxx（理由：xxx）
- **券商B 观点**：xxx（理由：xxx）
- **我的判断**：谁的逻辑更扎实，为什么
```

常见分歧维度：
- 收入增速假设
- 毛利率/净利率趋势
- 行业空间测算
- 竞争格局判断
- 估值方法选择（PE/PB/DCF/EV-EBITDA）
- 催化剂时间点

---

### Step 4 — 盈利预测对比（个股报告适用）

```markdown
| 指标 | 券商A | 券商B | 券商C | 差异幅度 |
|------|-------|-------|-------|---------|
| 2025E 营收 | | | | |
| 2025E 归母净利 | | | | |
| 2026E 营收 | | | | |
| 2026E 归母净利 | | | | |
| 目标PE | | | | |
| 目标价 | | | | |
```

标注：预测最激进的和最保守的分别是谁，差异来自哪个假设。

---

### Step 5 — 边际变化追踪

如果同一券商有前后两份报告，或不同时间点的报告：
- 哪些观点在调整？调高还是调低？
- 评级是否有变化？
- 盈利预测修正方向？

边际变化 > 绝对水平，这是最值得关注的信号。

---

### Step 6 — 综合判断

```markdown
## 📊 综合判断

**市场共识**：一句话概括当前卖方主流看法
**最大分歧**：一句话说明核心争议
**值得深挖**：基于分歧，建议用户自己重点验证什么
**信号强度**：
- 🟢 强共识 + 估值合理 = 安全边际较高
- 🟡 有分歧但逻辑都成立 = 需要自己判断
- 🔴 分歧大 + 假设都偏弱 = 谨慎
```

---

## 注意事项

- 至少 2 份研报才能做对比，1 份请引导到 `/report-digest`
- 对比时优先关注**分歧**而非共识，共识大家都知道
- 日期差异大的研报（>3 个月）要提醒用户，市场环境可能已变
- 不同券商的报表口径可能不同（如是否包含少数股东），对比时注意统一
