# 行业深度文章 /industry-article

**触发**：`/industry-article`、"写一篇XX行业的公众号文章"、"行业深度"

---

## 铁律

1. **全文禁止提及任何个股**——正文、基金推荐、风险提示，任何位置都不允许出现个股名称或代码。只能提行业、环节、方向、基金。绝对红线。
2. **一篇文章只讲一个故事**——不求全面覆盖，只求一条主线讲透
3. **数据为叙事服务**——故事需要什么数据就用什么，不是有什么数据就写什么
4. **读者不欠你耐心**——开头3行抓不住人，后面写得再好也没用
5. **表格是最后手段**——能用一句话说清的不要塞表格，全文表格 ≤ 2个
6. **术语必须翻译**——第一次出现时用类比或大白话解释

---

## 执行流程

### Step 1 — 确定主题与数据源

1. 确认行业主题 → 找对应行研 tracker skill（如 solar-tracker、tmt-landscape）
2. Read 该 skill 的 SKILL.md + 子文件，理解产业链结构、跟踪变量、标的库
3. **无对应 tracker skill → 走 `industry-deep-dive` skill**：Read `skills/research/industry-deep-dive/SKILL.md`，按 5 层研究协议（产业链地图→供需价格→竞争格局→政策催化→盈利验证）生成 tracker 级行业知识，作为文章数据基础

### Step 2 — 选叙事风格

Read `narrative-styles.md`，按行业状态选风格：

| 行业状态 | 推荐风格 |
|---------|---------|
| 新旧动能切换 | 双轨叙事 |
| 价格传导明显 | 传导链 |
| 市场存在误解 | 先破后立 |
| 龙头公司有代表性 | 庖丁解牛 |
| 读者关心"能不能买" | 灵魂三问 |

不确定就用**传导链**（最通用）。

### Step 3 — 数据采集

**只采叙事需要的 20%，不求全面。**

| 必采 | 用途 |
|------|------|
| 1个反直觉/冲击力数据 | 开头 Hook |
| 3-5个核心价格/供需数据 | 支撑推导链 |
| 1组对比数据 | 制造认知落差 |

按 tracker skill 流程 + research-mcp（`research_search` / `news_search`）采集。各风格的额外数据需求见 `narrative-styles.md`。

### Step 4 — 提炼核心叙事

1. 从数据中找**一条故事线**（不是10个发现，是1个故事）
2. 写出一句话核心观点（能发朋友圈、有观点、有悬念）
3. 定义认知增量：读者看完会获得什么新认知？

### Step 5 — 写作

**文章骨架：**

| 部分 | 篇幅 | 要点 |
|------|------|------|
| Hook | 100-200字 | 好奇心/认知冲突/情绪共振 |
| 主体 | 1500-2000字 | 按选定风格结构展开（见 `narrative-styles.md`） |
| 收束 | 200-300字 | 回到投资层面，给思考框架 |
| 相关主题基金 | 300-500字 | 见下方基金筛选流程 |
| 风险提示 | 固定模板 | |

写作规则和 before/after 示范详见 `narrative-styles.md`，核心规则：
- **故事优先，数据克制**：每段正文最多出现 3-5 个关键数字，多余数字删掉或合并为趋势描述（如"跌了近九成"代替"从300跌到48，跌幅84%"）。全文具体数字总量控制在 15 个以内
- **数据为故事服务**：先写故事逻辑，再挑最有冲击力的数据嵌入，不是有数据就往里塞。如果去掉一个数字不影响读者理解，就去掉
- **禁止连续罗列数据**：不允许连续两句话各带数字，中间必须有叙事过渡
- 引用保留券商名称但不写日期，表述自然化
- 每500字设一个"小钩子"保持阅读节奏

#### 相关主题基金

研究部提供候选基金列表（未提供则主动询问）。列表中会包含**业绩相关性**字段，作为筛选的首要参考。拿到后：

1. 先按**业绩相关性**排序，优先选相关性高的
2. 用 `get_fund_comprehensive_analysis(fund_code)` 对高相关性基金逐只调研，补充：
   - 重仓板块分布（与文章行业方向的重合度）
   - 规模、持仓集中度
3. 综合业绩相关性 + 调研结果，选出 5 只以上
4. **展示格式：表格分列展示，仅包含以下 5 列，不加其他列**：

   | 基金代码 | 基金名称 | 基金经理 | 权益仓位 | 特点 |
   |---------|---------|---------|---------|------|
   | 000xxx | XX主题基金 | 张三 | 90% | 重仓XX环节，与文章主题高度相关 |

   - **特点**列只写持仓中与文章主题相关的信息（行业/环节/板块），必须基于 `get_fund_comprehensive_analysis` 返回的实际数据，严禁编造，严禁出现个股名称
5. 末尾附引导语："更多详情请到天天基金查看"
6. 附："基金过往业绩不代表未来表现，投资需谨慎"

### Step 6 — 质量自检

| 检查项 | 标准 |
|--------|------|
| 个股零容忍 | 全文无任何个股名称或代码 |
| 开头测试 | 前3行能留住不懂行业的人 |
| 表格 ≤ 2 | 多余表格改写为叙述 |
| 数字克制 | 全文具体数字 ≤ 15个，每段最多1-2个，无连续两句各带数字 |
| 朗读测试 | 像聊天不像念报告 |
| 引用自然 | 有券商名称，无论文式日期引用 |
| 有类比 | 至少1个生活化类比 |
| 标题有吸引力 | 不是"XX行业分析"这种 |
| 基金有关联 | 主题基金与文章方向挂钩 |
| 篇幅 | 2500-3500字 |

### Step 7 — 数据核查

写完初稿后，**必须执行 `/fact-check` skill** 对文章中所有数据进行核查。根据核查报告修正有偏差的数据、删除或模糊化无法验证的数据，确保交付版本的每个数字都有据可查。

### Step 8 — 交付

1. 展示完整文章
2. 附上：类型、字数、核心观点一句话
3. **附引用来源清单**（不在文章中，供研究部审阅）：
   ```
   【引用来源（不发布）】
   - 券商团队：《研报标题》（日期）
   ```
4. **自动保存文章到 `memory/articles/`**：
   - 文件名格式：`YYYY-MM-DD-行业关键词.md`（如 `2026-03-31-光伏.md`）
   - 文件内容包含：完整文章正文 + 末尾附引用来源清单
   - 目录不存在时自动创建
5. 走 `mp-cover-art` → `gzh-publish`（如需）
6. 记录到当日日记
