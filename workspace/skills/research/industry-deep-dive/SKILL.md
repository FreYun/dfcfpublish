# 行业深研 /industry-deep-dive

**触发**：`/industry-deep-dive`、"帮我深研XX行业"、"XX产业链分析"、被 `gzh-industry-article` 在无对应 tracker 时调用

## 铁律

1. **研报全文优先**：`research_search` 返回 ID、机构名、摘要、评分，但摘要只能**发现和筛选**，不能当数据源。必须下载 PDF → pdftotext → 通读后才能引用（fact-check 教训：凭摘要写内容出过张冠李戴事故）
2. **逐查询降级**：研报全文 → news_search → web_search → 标 `[数据缺失]`
3. **完备性先于深度**：每层有验收标准，未达标先扩搜再往下走
4. **标注来源+链接**：每条数据标 `[研报全文]` / `[资讯]` / `[网搜]` / `[缺失]`；>30 天标 `[时效待验证]`。**所有引用必须保留可追溯链接**：研报附 PDF URL（`pdf.dfcfw.com/pdf/H3_{ID}_1.pdf`）、网搜/资讯附原文 URL。报告末尾统一列出引用来源清单
5. **`[网搜]` 数据必须交叉验证**：web_search 返回的内容来源不可控（可能是自媒体、论坛），不能直接当事实用。关键数据（价格、市占率、政策日期等）必须≥2个独立来源一致才可引用，只有单一网搜来源的标 `[单一来源，待验证]`
6. **完成后必须 fact-check**：L1-L5 完成后，Read `fact-check/SKILL.md` 按完整协议执行数据核查。报告中不涉及的检查项（如无基金推荐则跳过基金三要素）直接跳过，但所有存在的数据必须逐条核查，**禁止不调用工具就标 ✅**

---

## 前置：工具准备

本协议依赖 research-mcp 的 `research_search`、`news_search`、`commodity_data` 等工具。**先 Read `research-mcp/SKILL.md`** 了解调用方式。这些不是 web_search——`research_search` 能返回研报 ID（用于下载 PDF 全文），web_search 做不到。

---

## Step 0 — 行业类型识别

`research_search("{行业} 行业 投资逻辑 深度", top_k=2, search_day_ago=30)`

| 类型 | 典型 | 估值锚 | 重点层 |
|------|------|--------|-------|
| 周期制造 | 钢铁/化工/光伏/锂电 | PB | L2 供需价格 |
| 科技成长 | 半导体/AI/消费电子 | PS/PEG | L1 链图 + L3 格局 |
| 消费 | 白酒/医药/食品 | PE | L3 竞争格局 |
| 金融 | 银行/保险/券商 | PB | L4 政策 |
| 主题概念 | 早期/概念性主题 | PS/市值 | L1 + L4 |

无法判断 → 默认周期制造。

## Step 1 — 研报全文阅读

**Read `report-reading.md`**，按协议发现、下载、验证、通读 2-3 篇头部券商深度研报。后续 L1-L5 优先从已读研报中提取，search API 仅补充时效数据。

---

## L1 — 产业链地图

**目的**：上中下游环节 + 每环节代表公司。

从已读研报的"产业链概览"/"受益标的"章节提取。补充：`web_search("{行业} 产业链 上市公司 {year}")`

**提取**：≥3 个环节 + 每环节 ≥2 家公司（`tushare_stock_basic` 确认代码）+ 产业链图 + 市场规模

**验收**：≥3 环节 AND ≥6 家公司。未达标 → 补充阅读更多研报或 web_search。

---

## L2 — 各环节供需与价格

**目的**：每环节的供/需/价格信号。周期制造型核心层。

从已读研报的"供给端/需求端"/"景气跟踪"章节提取。补充数据：

| 工具 | 用途 |
|------|------|
| `commodity_data(symbol)` / `futures_data(symbol)` | 大宗商品现货/期货价格（research-mcp） |
| `news_search("{行业} {环节} 价格 库存", search_day_ago=14)` | 最新价格动态 |
| `web_search("site:mysteel.com {行业} {环节} 价格")` | Mysteel 产业价格数据 |

工具选择：大宗品（原材料/化工/金属）→ commodity_data/futures_data；工业品/制成品 → news_search + mysteel。

**提取（每环节）**：需求驱动 + 供给变量 + ≥1 价格信号（品名+价格+趋势+源+频率）+ 库存周期 + 供需结论

**验收**：≥2 环节有完整供需画像。消费/金融型简化为需求趋势；主题概念型可标 `[数据有限]` 跳过。

---

## L3 — 竞争格局

**目的**：集中度、壁垒、格局稳定性。科技成长/消费型核心层。

从已读研报的"竞争格局"/"市场份额"章节提取。补充：
- `news_search("{行业} 价格战 新进入者", search_day_ago=30)`
- `web_search("{行业} CR5 市占率 {year}")`
- 科技型加：`web_search("{行业} 国产替代 自主可控 {year}")`

**提取**：CR3/CR5 + 壁垒类型 + 格局稳定性 + 价格战信号 + 国产替代（科技型）

**验收**：≥1 环节有集中度+壁垒。全缺标 `[格局数据不足]`，不阻塞。

---

## L4 — 政策与催化

**目的**：改变供需曲线的外部力量。金融/主题型核心层。

从已读研报的"政策梳理"章节提取。**政策时效性强，必须补充搜索：**
- `news_search("{行业} 政策 补贴 监管 {year}", search_day_ago=30)`
- `news_search("{行业} 重大事件 催化 {year_month}", search_day_ago=14)`
- 金融型加：`web_search("{行业} 监管改革 利率政策 {year}")`

**提取**：近 3 月政策（内容+日期+方向+影响曲线）+ 政策周期 + 催化事件 + 地缘风险

**验收**：≥1 政策/催化有日期。零条标 `[近期无重大政策]`，不阻塞。

---

## L5 — 盈利验证

**目的**：Tushare 数据验证 L1-L4 定性判断。对 6-8 家核心公司并行：

| 工具 | 用途 |
|------|------|
| `tushare_daily_basic(ts_code, trade_date)` | PE/PB/PS/市值 |
| `tushare_stock_daily(ts_code, 60日)` | 近 1M/3M 涨跌幅 |
| `tushare_fina_indicator(ts_code, 近2期)` | ROE/毛利率/净利率 |
| `tushare_income(ts_code, 近2期)` | 营收/净利润增速（龙头 3-4 家）|

**交叉验证**：毛利率 vs L2 供需、营收增速 vs L2 需求、PB vs 周期位置。矛盾必须标注。

**验收**：≥4 家有完整数据。缺的标 N/A。

---

## L6 — 数据核查

**Read `fact-check/SKILL.md`**，按完整协议执行。不涉及的检查项直接跳过：

| 检查项 | 深研报告是否适用 |
|--------|----------------|
| 具体数字逐条核查 | ✅ 必做 |
| 研报来源归属（pdftotext 首页） | Step 1 已验证 → 跳过；未验证 → 必做 |
| 新闻来源归属（web_fetch 打开链接） | 有 `[资讯]` 引用 → 必做；无 → 跳过 |
| `[网搜]` 数据交叉验证 | ✅ 必做（关键数据需≥2个独立来源） |
| 基金三要素 | 无基金推荐 → 跳过 |
| Tushare 财务数据 | API 直接返回 → 跳过（已是一手数据） |

**⛔ 禁止不调用工具就标 ✅**——这是 fact-check 最高优先级铁律。没有调用验证工具的数据只能标「待核查」。

## 异常降级

| 场景 | 处理 |
|------|------|
| 研报链接均无效 | 扩 top_k 重试 → web_search 找公开研报 → 降级纯搜索模式（标注） |
| 研报内容不匹配 | 丢弃换下一篇 |
| 某层验收未达标 | 回研报检查遗漏 → 补充搜索 → 3 轮后标注缺失 |
| Tushare 缺数据 | 标 N/A |
| 行业太新/小众 | 无深度研报，降级搜索模式，标 `[数据稀疏]` |
