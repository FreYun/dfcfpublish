## 供需分析框架

### 需求侧——三个需求引擎

**引擎1：游戏需求（用户时长 × 付费率 × ARPU）**
- 跟踪变量：iOS畅销榜排名、游戏行业月度流水（伽马数据）、出海流水(Sensor Tower)
- 数据源：`news_search("游戏 流水 畅销")` + browser(TapTap/七麦数据)
- 传导链：爆款上线 → 短期流水脉冲；行业整体ARPU↑ → 长期增长
- 推导要点：游戏是**产品驱动**行业 → 供需框架弱于产品周期框架
  - 有新爆款 → 该公司短期景气极高（但不可外推）
  - 无爆款 → 看存量产品流水衰减斜率 + 在研管线密度
- 出海：国内增速放缓 → 出海是结构性增量 → 海外流水增速比国内更重要

**引擎2：广告需求（GDP的函数）**
- 跟踪变量：社会消费品零售总额增速、广告主预算调研、品牌广告vs效果广告占比
- 数据源：`research_search("广告 投放 预算")` + `news_search("广告市场")`
- 传导链：GDP/消费↑ → 广告主预算↑ → 广告公司收入↑（强顺周期）
- 推导要点：分众=消费晴雨表（品牌广告占比高），蓝色光标=效果广告+AI
- 验证：分众的楼宇广告刊挂率 → 直接反映广告需求

**引擎3：AIGC商业化（0→1阶段）**
- 跟踪变量：AIGC工具MAU/付费用户增速、AI生成内容成本vs人工成本对比
- 数据源：`news_search("AIGC 用户 付费")` + `research_search("AI内容生成 商业化")`
- 传导链：AI降本 → 内容生产效率↑ → 传媒公司利润率改善 or 新商业模式
- 推导要点：当前是**成本端受益**（降低制作成本）而非**收入端驱动**（用户为AI付费）→ 关注利润率改善而非收入增长

### 供给侧——版号是核心供给约束

- 游戏：版号审批节奏 = 供给阀门
  - 版号收紧 → 新游减少 → 存量龙头受益（壁垒提高）
  - 版号放开 → 竞争加剧 → 用户注意力分散 → 中小公司更难
- 影视：内容供给波动大（靠爆款），长期看头部集中度提升
- 广告：供给端（广告位/流量）近乎无限 → 纯看需求 → 顺周期
- AIGC：AI降低内容生产门槛 → 供给曲线大幅右移 → 长期可能压低内容溢价
