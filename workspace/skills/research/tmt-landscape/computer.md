## 供需分析框架

### 需求侧——三个需求引擎

**引擎1：AI Infra投资（B端算力支出）**
- 跟踪变量：国内云厂商Capex（阿里/腾讯/华为/字节）、AI服务器招标
- 数据源：`research_search("AI服务器 Capex 算力")` + 财报/新闻
- 传导链：Capex↑ → 服务器/交换机/IDC/液冷需求↑ → 浪潮/曙光/润泽
- 推导要点：Capex同比增速 + 订单vs交付节奏 → 判断需求是否见顶
- 关键区分：**Capex增速放缓 ≠ 需求下滑**，只是斜率变化；Capex绝对值仍增=需求仍在扩张

**引擎2：信创替代（政策驱动需求曲线右移）**
- 跟踪变量：政府/央企信创招标金额、国产OS/DB部署进度
- 数据源：`news_search("信创 招标 国产")` + `research_search("信创 进度")`
- 传导链：招标落地 → OS/DB/中间件/办公软件需求↑ → 中国软件/金山办公/达梦
- 推导要点：信创是政策周期，看招标节奏（年末集中 vs 均匀释放）+ 2.0向行业端渗透进度

**引擎3：AI应用变现（C端+B端付费意愿）**
- 跟踪变量：大模型API调用量/价格、AI功能付费转化率、SaaS ARR增速
- 数据源：`news_search("AI应用 付费 API")` + `research_search("AI商业化")`
- 传导链：付费率↑ → AI应用公司收入↑ → 估值从PS切换到PE
- 推导要点：当前多数AI应用还在"用户增长但收入未兑现"阶段 → 跟踪付费转化拐点

### 供给侧——竞争与产能

- 跟踪变量：SaaS公司数量增速、安全厂商数量、开源大模型对商业模型的替代
- 推导逻辑：
  - 供给端门槛低（SaaS/安全）→ 价格战 → 毛利率压缩 → 即使需求增长也不一定利好利润
  - 供给端门槛高（EDA/工业软件/数据库）→ 寡头格局 → 需求增长直接传导为盈利
  - AI降低软件开发成本 → 长期供给曲线右移 → 对传统软件公司既是机遇也是威胁
