# TMT 全景研究 /tmt-landscape

**触发**：`/tmt-landscape`、"TMT怎么样"、"科技板块配置"、"半导体/AI/消费电子景气"

TMT = 电子 + 计算机 + 通信 + 传媒（申万2021版）。

## 铁律：推导 ≠ 总结

每个结论 = **数据 + 经济学逻辑 → 结论**。

❌ "半导体行业机遇与挑战并存" — 废话
❌ "研报普遍看好AI" — 鹦鹉
✅ "产能利用率72%（↓13pct QoQ）+ DRAM报价连跌3月 → 供过于求 → PE 45x隐含30%增速难兑现 → 估值下修压力"

## Step 1 — 定价环境

TMT定价 = f(利率, 风险偏好, 盈利周期)。先判断当前主导变量。

**并行拉取：**

| 数据 | 工具 |
|------|------|
| 中证TMT/沪深300/创业板 近60日 | `tushare_index_daily` |
| TMT估值分位 | `research-mcp: get_ashares_index_val` |
| 10Y国债 | `research-mcp: get_cn_bond_yield` |
| A股全景 | `research-mcp: market_overview` |

**推导：**
1. 10Y方向 → DCF折现率 → 成长股估值弹性方向
2. TMT超额(vs沪深300) + 成交额占比 → 资金偏好方向
3. PE分位 vs 盈利增速方向 → 估值领先还是滞后

→ 输出：当前TMT定价由 [利率/风偏/盈利] 主导

## Step 2 — 四方向景气推导

核心问题：**每方向供给增速 vs 需求增速，谁快？**

**数据（research-mcp 并行）：**
- `research_search` 各方向研报（top_k=3, 30天内）
- `news_search` 各方向动态（top_k=3, 7天内）

聚焦模式：Read 子文件，按子文件中的供需分析框架逐项推导，browser补充产业数据。

**每方向必须输出：**
1. 需求变量 + 数据 → 方向和斜率
2. 供给变量 + 数据 → 方向和斜率
3. 价格信号 → 验证供需判断
4. 景气结论 + 推导链

## Step 3 — 盈利验证

景气推导是"应然"，财务是"实然"。两者必须对齐。

**每方向2-3龙头，Tushare并行：**
- `tushare_daily_basic` 最近交易日 → PE/PB/PS/市值
- `tushare_stock_daily` 近60日 → 涨跌幅
- `tushare_income` 最近2期 → 营收/利润增速

**research-mcp补充：** `get_stock_valuation` → 估值历史分位

**推导逻辑：**
- 营收增速 → 验证需求端（景气上行但营收放缓 → 推导有误或份额转移）
- 毛利率方向 → 验证供需（扩张=偏紧, 收缩=过剩）
- PE vs 盈利增速匹配度 → PE高+增速放缓=危险

输出横向对比表。

## Step 4 — 资金行为解读

资金方向是现象，不是结论。必须解释**为什么**。

**数据：** `get_stock_northbound_holding` + `get_stock_fund_flow`（龙头，近20-60日）

**推导：**
- 北向增持方向 → 为什么？（估值洼地/盈利确定/全球产业链受益？）
- 主力方向 → 与基本面推导一致？
- 北向 vs 内资分歧 → 谁的逻辑更硬？

## Step 5 — 政策：供需曲线的位移器

政策不是催化剂清单，是改变供需平衡的外力。

**数据：** `news_search` + `research_search`（TMT政策，30天内）。browser按需补充。

**每条政策回答：**
- 改变了谁的供给/需求曲线？方向？
- 市场定价了多少？（发布日vs股价反应时点）

## Step 6 — 逻辑自洽检验

| 检查 | 一致条件 | 矛盾处理 |
|------|---------|---------|
| 景气 vs 毛利率 | 上行→毛利率稳定/扩张 | 不一致→重查供需 |
| 估值 vs 盈利 | 说便宜→增速>PE隐含 | 不一致→修正估值 |
| 资金 vs 基本面 | 方向应一致 | 不一致→分析分歧 |

矛盾 → 回补数据或修正推导，不模糊掩盖。

## 注意

- TMT高估值常态 → 用PE vs 盈利匹配度，不用PE绝对值
- 子行业轮动本质是景气差 → 全景模式核心价值 = 抓轮动方向
- 研报是数据源，不是结论源 → 提取事实，自己推导
