# 历史案例库

记录 market-regime-classifier 在真实复盘数据上的判断结果，用于校准阈值和发现边缘情况。

**格式**：每个案例包含日期、六维得分、总分、判定 regime、是否切换/逃生门、事后市场表现点评。

## 2026-03 一周前向回放（sanity check）

用 workspace-bot11 已有的 6 个复盘 MD 按时间顺序跑，起始 log 为空，观察 3 日确认 + 逃生门行为。

| 日期 | 总分 | 判定 | 切换/逃生门 | 说明 |
|---|---|---|---|---|
| 2026-03-12 | 0 | 中性震荡 | bootstrap | 首日，log 为空，默认 NEUTRAL |
| 2026-03-13 | -3 | 中性震荡 | 仅 warning | 单日进入弱势震荡区间，但历史不足 3 日 → 维持 |
| 2026-03-16 | -1 | 中性震荡 | - | 回到中性区间 |
| 2026-03-19 | -1 | 中性震荡 | - | 中性区间维持 |
| 2026-03-20 | **-6** | **弱势震荡** | 🚨 **EMERGENCY_DOWN** | 总分单日下降 5 分（-1 → -6），触发 `total_drop_5`，从 NEUTRAL 强制降 1 档 |
| 2026-03-26 | -6 | 弱势震荡 | - | 3 日窗口被 emergency 重置，当前只有 1 新日历史，无法进一步切换到 BEAR（需 3 日全部 ≤ -7） |

### 观察

- **2026-03-13 的警戒**：单日 -3 落在 WEAK_RANGE 区间，系统生成 `switch_warning` 但不切换 —— 这符合 "宁可踏空" 的设计，避免一日情绪拖拽。
- **2026-03-20 的逃生门**：这一天 HS300 -0.35% / CSI1000 -1.59% 并不极端，但 **涨跌家数比 11.4%**（662 up vs 4786 down）+ **情绪评分 25** + **最高连板 0 板** 的组合使总分从 -1 断崖跳到 -6。逃生门的 `total_drop_5` 条件精准捕捉了这种盘面结构性恶化。
- **2026-03-26 的"卡档"**：总分 -6 已在 BEAR 区间，但逃生门只允许每日下降一档，且 3 日窗口从 emergency 后重新开始。要进一步切到 BEAR，需要从 2026-03-20 之后再有 2 个交易日全部满足 BEAR 的确认条件（3 日 ≤ -7）。
