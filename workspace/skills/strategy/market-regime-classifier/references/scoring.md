# 六维打分详细规则

完整规则源：spec §4。本文件是实现层的细节澄清，供 skill 用户快速查阅打分阈值与边界值归属。

---

## 总览

| # | 维度 | 数据来源 | 打分函数 |
|---|---|---|---|
| 1 | 指数 vs 均线 | akshare HS300 + CSI1000 日线 | `scoring.score_ma_position` |
| 2 | 涨跌家数比 | 复盘 MD "市场全景 / 市场宽度" | `scoring.score_advance_decline` |
| 3 | 情绪(涨停-跌停) | 复盘 MD "情绪温度计 / 涨停:跌停" | `scoring.score_sentiment_delta` |
| 4 | 情绪评分 | 复盘 MD "情绪温度计 / 情绪评分" | `scoring.score_sentiment_index` |
| 5 | 最高连板高度 | 复盘 MD "连板 & 题材前瞻 / 最高连板" | `scoring.score_streak_height` |
| 6 | 成交量趋势 | akshare HS300 日线 (5 日均量 / 20 日均量) | `scoring.score_volume_trend` |

每维 **-2 ~ +2**，等权重，总分范围 **-12 ~ +12**。

## 1. 指数 vs 均线

### 单指数打分规则

| 得分 | 条件 |
|---|---|
| +2 | 站上 5/20/60/250, MA 多头排列 (MA5 > MA20 > MA60 > MA250) |
| +1 | 站上 5/20/60 |
| 0  | 站上 20, 60 位置不定 |
| -1 | 跌破 20 |
| -2 | MA 空头排列 (MA5 < MA20 < MA60 < MA250) |

### 实现注意

- **"站上" 采用 `>=`**（包含持平），与中文金融约定一致。close 刚好等于均线判为"站稳"而非"跌破"。
- **"多头/空头排列" 采用严格 `>` / `<`**，相等的均线不构成层级。
- 空头排列优先检测：如果 MA 是空头排列，即使 close 反弹到均线上方也给 -2（反弹是噪音，排列是结构）。

### 两指数合成

HS300 与 CSI1000 各自按上表打分，取算术平均后四舍五入到最近整数。

- 使用 **round half away from zero**（远离零方向），`1.5 → +2`，`-0.5 → -1`。不用 Python 内置 `round()`（banker's rounding）。
- 典型例：HS300 +2 + CSI1000 +1 → 平均 +1.5 → **+2**。

## 2. 涨跌家数比

公式：`上涨家数 / (上涨家数 + 下跌家数 + 平盘家数)`。`平盘家数` 在 MD 里缺失时按 0 处理。

| 得分 | 区间 |
|---|---|
| +2 | `ratio > 0.75` |
| +1 | `0.60 ≤ ratio ≤ 0.75` |
| 0  | `0.40 ≤ ratio < 0.60` |
| -1 | `0.25 ≤ ratio < 0.40` |
| -2 | `ratio < 0.25` |

边界归属：spec §4 表里的区间描述（如 "0.60-0.75"）视为 **闭区间**，所以 `ratio = 0.75` 归 **+1**，`ratio > 0.75` 才归 +2。

## 3. 情绪(涨停-跌停)

`delta = 涨停数 − 跌停数`（MD 里通常写作 `涨停:跌停 32:22`，解析后 delta = 10）。

| 得分 | 区间 |
|---|---|
| +2 | `delta > 80` |
| +1 | `40 ≤ delta ≤ 80` |
| 0  | `10 ≤ delta < 40` |
| -1 | `-40 ≤ delta < 10` |
| -2 | `delta < -40` |

---

## 4. 情绪评分

复盘 MD 中的 `情绪评分 N / 100` 字段，0-100 的复合情绪指标。

| 得分 | 区间 |
|---|---|
| +2 | `> 70` |
| +1 | `55 ≤ x ≤ 70` |
| 0  | `40 ≤ x < 55` |
| -1 | `25 ≤ x < 40` |
| -2 | `< 25` |

**历史背景**：这一维度原本是 "炸板率"，但在 spec 最终定稿前（commit `f681942`）改为 "情绪评分"。原因是炸板率在 daily_review 输出的复盘 MD 里不存在，而情绪评分直接写在"情绪温度计"章节。

## 5. 最高连板高度

复盘 MD 中的 `最高连板：N 板`。

| 得分 | 条件 |
|---|---|
| +2 | `≥ 5 板` |
| +1 | `= 4 板` |
| 0  | `= 3 板` |
| -1 | `= 2 板` |
| -2 | `≤ 1 板` |

## 6. 成交量趋势

`5 日均量 / 20 日均量`。使用 **HS300 成交量** 作为全市场成交量代理（与 MA 维度共用一份 akshare 日线数据）。

| 得分 | 区间 |
|---|---|
| +2 | `> 1.30` |
| +1 | `1.10 ≤ x ≤ 1.30` |
| 0  | `0.90 ≤ x < 1.10` |
| -1 | `0.70 ≤ x < 0.90` |
| -2 | `< 0.70` |

## 降级策略（spec §8）

- 某维度数据缺失 → 该维度得分按 **0 (中性)** 计入总分
- `missing_dims` 记录缺失的维度名
- `confidence` 按缺失数降级：0 维 → `high`，1-2 维 → `medium`，≥3 维 → `low`
- 不中断流程，始终输出结论

## 校准建议

- 阈值是初始经验值，上线后应每月看 `memory/regime-log.md` 校准一次
- 重点关注 **情绪评分** 与 **最高连板高度** 两个维度（A 股情绪波动大，阈值可能漂移）
- 如果发现某档得分长期偏离实际市场状态，**先停下来与用户讨论**，不要单方面改阈值
