# S5 龙回头战法 — 完整定义

## 一句话

前期龙头股（近 30 日内出过连板）冷却 2-7 天后，今天放量大阳反包 T-1 跌幅，T+1 集合竞价进。

## 三段确认

```
┌─ 龙头 ──────┐    ┌─ 冷却 ──────┐    ┌─ 反包(T 日)──┐
│ ≥2 连板     │ →  │ 2-7 个交易日 │ →  │ T 日涨 ≥ 7%   │
│ 近 30 日内  │    │ 阶段跌幅≥5%  │    │ 收盘 ≥ T-1 高 │
└─────────────┘    └──────────────┘    └───────────────┘
```

### 1. 龙头确认

**条件**：近 30 个交易日内出现过 **≥2 连板**。

**为什么是 2 连板**：1 连板太多噪音，3 连板要求过严会错过题材初期龙头。2 连板既排除了"一日游"，又能覆盖大部分中期龙头股。

**记录**：龙头日的日期、当日收盘价、最高连板数（`max_streak`）。如果 30 日内多次连板，取最高连板那天作为"龙头高点"。

**数据**：`ak.stock_zt_pool_em(date=YYYYMMDD)`，`连板数` 字段。

### 2. 冷却确认

**条件**：龙头高点之后到 T 日之间间隔 **2-7 个交易日**，且阶段跌幅（高点收盘 → T-1 收盘）**≥ 5%**。

**为什么 2-7 天**：
- 下限 2 天：避免"昨日连板今天又涨停"的接力情形（那是 S2 龙头板接力的活）
- 上限 7 天：超过 7 天后龙头身份开始衰减，不再算"龙回头"而算"老题材重启"
- 阶段跌幅 ≥5%：保证有真正的回调，不是横盘震荡

**计算**：
```
高点收盘 = stock_zh_a_hist[龙头日].收盘
T-1 收盘 = stock_zh_a_hist[T-1日].收盘
冷却天数 = T 日 - 龙头日 - 1（不含 T 日本身）
阶段跌幅 = (T-1 收盘 - 高点收盘) / 高点收盘
```

### 3. 反包确认

**条件**：T 日涨幅 **≥ 7%** **且** T 日收盘价 **≥ T-1 最高价**。

**两个条件都要满足**：
- 涨 7% 单独看：可能是低开高走收阴，仍然不强
- 收复 T-1 最高：保证今日实质上"吃掉"了昨天的下跌，是真反包不是反弹

**为什么不要求涨停**：S5 的精髓是反包不是接力，要求涨停会过滤掉很多优质的"快速 V 反"信号。

## 选股漏斗

```
T 日收盘后:
  1. 拉 T 日涨停池 → 提取所属行业 → 统计每个行业涨停家数 → 取前 3 = 热门行业
  2. 回查 T-30 ~ T-1 的涨停池 → 出过 ≥2 连板的代码 = 龙头池 (≈ 50-150 只)
  3. 龙头池 ∩ 行业属于热门行业 = 重点关注集 (≈ 10-50 只)
  4. 对每只拉近 35 日 K 线
  5. 跑三段确认 → 通过的是 candidate (通常 0-5 只)
```

**为什么用涨停池所属行业而不是 stock_board_concept_name_em？**
后者没有 date 参数，只能拿今日数据。涨停池有 date 参数，从中聚合行业既给出 T 日真实热门，又支持历史回测，一举两得。

## 入场/止损/止盈

| 字段 | 规则 | 说明 |
|---|---|---|
| `entry_zone_low` | T 日收盘 × 0.99 | 集合竞价低开 ≤1% 进 |
| `entry_zone_high` | T 日收盘 × 1.01 | 高开 >1% 不追，等回调 |
| `stop_loss` | T 日最低价 | 跌破 T 日最低硬止损 |
| `target_1` | T 日收盘 × 1.05 | 第一止盈位（出 1/2 仓） |
| `target_2` | T 日收盘 × 1.10 | 第二止盈位（清仓） |

**为什么止损放 T 日最低**：如果 T+1 跌破 T 日最低，说明反包失败，原本的"V 反"破位，逻辑结束应该走人。

## 仓位计算

base = `regime.playbook.position_limit.single`（单票上限，由 classifier 给出）

**乘数链**（同 [market-regime mapping.md](/home/rooot/.openclaw/scripts/market-regime/references/mapping.md)）：
```
final = base
if confidence == "low":         final *= 0.5
elif confidence == "medium":    final *= 0.8
if switched:                    final *= 0.8
if emergency_switch:            final *= 0.7
```

**示例**：
- WEAK_RANGE base = 0.15，confidence=high，switched=true：`0.15 × 0.8 = 0.12`
- NEUTRAL_RANGE base = 0.20，confidence=medium：`0.20 × 0.8 = 0.16`
- BEAR base = 0.10：S5 不在 BEAR 推荐里 → **直接拒绝出 candidate**

## 禁用规则

如果 `S5` 不在 `regime.playbook.recommended` 列表里，**整个 select 直接返回空**，输出 `skipped_reason`。具体哪些 regime 不适合 S5：
- **STRONG_BULL**：建议 S1/S2/S6 的接力突破，不是回头反包
- **BEAR**：建议 S8 空仓，最多 S7 超跌反弹一日游

S5 的合法 regime：**STRONG_RANGE / NEUTRAL_RANGE / WEAK_RANGE** 三档震荡。

## T+1 验证脚本

T+1 开盘后 5-10 分钟跑 `verify.py`，对每只候选执行：

| 状态 | 判定条件 | 操作建议 |
|---|---|---|
| `triggered` | 实际开盘价 ∈ entry_zone | ✅ 按计划买入 |
| `gap_up_skip` | 开盘价 > entry_zone_high | ⚠️ 高开过多放弃，不追 |
| `stop_hit` | 当前价 ≤ stop_loss | ❌ 止损位已破，弃单 |
| `wait` | 开盘后回调中，未触发 | ⏳ 等待回调到 entry_zone |

数据：`ak.stock_intraday_em(symbol=代码)` 取开盘后的实时分时。

## 已知边界

1. **涨停池历史窗口**：akshare `stock_zt_pool_em` 仅保留近 2-3 周数据。回测 T 日 < 当前日 - 20 天时，龙头池可能拉不到完整数据，candidate 数会偏少。
2. **行业分类粒度**：涨停池的 `所属行业` 是东财大行业分类（约 80 个），比"概念板块"粗。某些热点（如"AI 应用"）可能跨多个东财行业，会被切分。
3. **集合竞价数据延迟**：`stock_intraday_em` 不保证秒级实时，verify.py 的设计是"开盘后 5-10 分钟跑一次"，不是盘中持续监听。
4. **不处理停牌、ST**：涨停池本身就排除 ST，所以 dragon_pool 自动干净。停牌股 stock_zh_a_hist 会缺数据，signal_detector 会跳过。

## 调参建议（默认值在 [signal-tuning.md](signal-tuning.md)）

如果信号过少：
- 龙头门槛降到 1 连板 + 单日涨幅 ≥7%
- 反包涨幅降到 5%
- 行业 top3 → top5

如果信号过多：
- 龙头门槛升到 3 连板
- 冷却跌幅升到 8%
- 加资金流过滤（北向净买入或主力净流入）

校准时**先停下来与用户讨论**，不要单方面改阈值。
