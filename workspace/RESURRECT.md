# 复活手册 — 从基础资料到一个活人

> 把一个真实博主/KOL 的公开内容，提炼成一个有灵魂、能自主创作的 bot。

---

## 前置条件

你需要准备：

| 材料 | 数量 | 来源 |
|------|------|------|
| 目标博主的公开帖子 | 20-50 篇 | 小红书/公众号/微博 |
| 博主公开主页信息 | 1 份 | 粉丝数、签名、IP属地、定位 |
| （可选）第二个风格互补的博主 | 同上 | 用于风格融合，单源也行 |
| 新 bot 编号 | 1 个 | 当前最大编号 +1 |

---

## 第一步：搭骨架（5分钟）

创建 workspace 目录结构，复制通用模板：

```bash
N=11  # 新 bot 编号
BASE=/home/rooot/.openclaw

# 创建目录
mkdir -p $BASE/workspace-bot${N}/skills
mkdir -p $BASE/workspace-bot${N}/memory

# 复制通用 skill
cp -r $BASE/workspace/skills/xiaohongshu-mcp $BASE/workspace-bot${N}/skills/
cp -r $BASE/workspace/skills/chaojimali-style $BASE/workspace-bot${N}/skills/

# 复制模板文件
for f in AGENTS.md HEARTBEAT.md TOOLS.md; do
  cp $BASE/workspace-bot1/${f} $BASE/workspace-bot${N}/${f}
done

# 创建空文件
touch $BASE/workspace-bot${N}/{IDENTITY.md,SOUL.md,USER.md,MEMORY.md}
```

修改 `TOOLS.md` 中的 `account_id` 和端口号：
- account_id: `botN`
- 端口: `1806N`

这一步结束后你有了一个空壳 bot，能运行但没有灵魂。

---

## 第二步：采集素材

### 2.1 抓取博主主页

用小红书 MCP 或手动记录：

```
博主昵称：
小红书号：
粉丝数：
IP属地：
签名：
定位（一句话）：
```

### 2.2 采集帖子样本

从博主主页抓 20-50 篇帖子，覆盖其主要内容类型。重点记录：

- **标题**（原文，一字不改）
- **正文**（原文，保留 emoji 和换行）
- **点赞/收藏/评论数**（判断哪些是爆款）
- **发布时间**（看发布频率和节奏）

按点赞排序，重点分析 Top 10 爆款。

> 如果有两个源博主，两边各采 20-50 篇。

---

## 第三步：拆骨（最核心的一步）

逐维度拆解博主的写作模式，产出一份 `SKILL.md`。

### 3.1 拆标题

从 20+ 个标题中归纳出 5-8 种标题公式：

```
| 类型 | 公式 | 原文示例 |
|------|------|----------|
| 日记型 | 日期 + 关键词 + ！ | "3月9日 逻辑记录！" |
| 反问型 | 现象 + 要XXX了？ | "牛市要结束了？" |
| ... | ... | ... |
```

同时记录标题禁忌——博主**不会**用的标题风格。

### 3.2 拆正文结构

读 5-10 篇完整正文，总结出固定的结构模板：

- 开头怎么写？（事件切入？判断先行？故事引入？）
- 中间怎么展开？（用什么标记符？每段几句？）
- 结尾怎么收？（有免责声明？有互动引导？）
- 总字数范围？段落长度？

### 3.3 拆语言 DNA

这是最重要的一步——提炼出一行融合公式：

```
博主A的语言 = 维度1 × 维度2 × 维度3 × 维度4
```

每个维度用两个对立面中的一端来描述：

| 维度 | 一端 | 另一端 |
|------|------|--------|
| 专业度 | 金融专业术语 | 大白话科普 |
| 语气 | 口语化随意 | 书面正式 |
| 人称 | 强第一人称 | 无人称/客观 |
| 态度 | 自信判断 | 克制中立 |
| 文学性 | 偶尔诗意 | 纯干货 |
| 互动感 | 跟读者对话 | 自说自话 |

摘录 5-10 句最能代表其风格的**原文金句**，作为语感校准参考。

### 3.4 拆 Emoji 规则

统计博主的 emoji 使用模式：

- 哪些 emoji 高频出现？什么功能？
- 哪些 emoji **绝对不用**？
- 有没有反审查替代词？（如 🈶色 代替 有色金属）

### 3.5 拆内容类型分布

统计 20+ 篇帖子的类型占比：

```
行情日评 40% / 深度分析 25% / 热点解读 20% / 生活哲理 10% / 其他 5%
```

### 3.6 产出：风格 Skill

把以上拆解结果写成一个完整的 skill 文档：

```bash
mkdir -p $BASE/workspace-bot${N}/skills/风格名-style
```

产出三个文件：

| 文件 | 内容 |
|------|------|
| `SKILL.md` | 标题公式 + 正文结构 + 语言DNA + emoji规则 + 反审查词 + 发布前检查清单 |
| `EXAMPLES.md` | 3篇完整范文，覆盖不同内容类型 |
| `SYSTEM_PROMPT.md` | 给 AI 的角色指令（可选） |

> **如果有两个源博主**，在 SKILL.md 中先分别画像，再写融合公式。
> 融合不是各取一半，而是定义"取A的什么 + 取B的什么 = 新的什么"。

---

## 第四步：写范文

写 3 篇完整的帖子，放在 `EXAMPLES.md` 中：

| 范文 | 覆盖什么 |
|------|---------|
| 范文1 | 最高频的内容类型（如行情日评） |
| 范文2 | 第二高频的类型（如深度分析） |
| 范文3 | 最能体现人格的类型（如生活哲理） |

每篇范文必须完整——标题、正文、免责声明、标签，一个都不少。这是 AI 模仿的锚点，质量决定上限。

---

## 第五步：注入灵魂

### 5.1 IDENTITY.md（公开身份，4-6行）

```markdown
# IDENTITY.md - 我是谁
- **名字：** xxx
- **人设：** 一句话定位
- **性格：** 2-3个关键词
- **擅长：** 核心能力
- **Emoji：** 代表符号
```

### 5.2 SOUL.md（深层人格）

开头必须引用通用底座：

```markdown
> **首先 `Read ../workspace/SOUL_COMMON.md` 获取通用规范。**
```

然后写以下章节：

| 章节 | 写什么 |
|------|--------|
| 我的名字 | 名字 |
| 我是谁 | 2-3段人设故事（第一人称，不是简历） |
| 说话风格 | 从 SKILL.md 提炼的语言DNA + 必须遵循的规则 + 禁忌 |
| 内容风格 | 指向 skills/xxx-style/SKILL.md |
| 核心关注领域 | 这个人关心什么 |
| 边界 | 不做什么 |
| 连续性 | 记忆机制 |

**关键原则**：SOUL.md 要用第一人称写，像这个人在自述，不是在填表。

### 5.3 SOUL_COMMON.md 提供的底座（不需要再写）

通用底座已经替你处理了：
- **"你不是 AI"** — 禁止 AI 腔，要求像个有性格的人
- **身份锁定** — 绑定 account_id，防串号
- **雇佣关系** — 研究部是雇主
- **安全铁律** — 不泄密、不删文件、不发半成品
- **连续性** — "每次醒来你都是全新的，但文件就是你的记忆"

这些是所有 bot 共享的"人类操作系统"，不需要每个 bot 重复写。

---

## 第六步：配工作手册（AGENTS.md）

从模板复制后，修改以下部分：

- **启动读取顺序** — 确保读 SOUL_COMMON → SOUL → TOOLS_COMMON → TOOLS → 记忆
- **内容产出类型和频率** — 与 SOUL.md 的内容类型占比一致
- **发帖流程** — 指向正确的 style skill
- **技能清单** — 如果是专业型 bot（如金融），加上专业技能
- **记忆系统** — 日记、长期记忆、专题笔记的规则

---

## 第七步：试运行 & 校准

### 7.1 第一篇试稿

让 bot 读完所有文件后，写一篇最高频类型的帖子。

### 7.2 对照检查

拿试稿和源博主的真实帖子放一起看：

| 检查项 | 问自己 |
|--------|--------|
| 标题 | 混在源博主的标题里能不能以假乱真？ |
| 开头 | 第一句话的语气对不对？ |
| 数据密度 | 数字够不够具体？ |
| emoji | 用法和频率像不像？ |
| 结尾 | 收束方式对不对？ |
| 整体读感 | 像一个人在写还是像 AI 在生成？ |

### 7.3 校准

根据试稿暴露的问题，回去改 SKILL.md 和 SOUL.md。通常需要 2-3 轮校准。

常见问题和修法：

| 问题 | 改哪里 |
|------|--------|
| 语气太正式/太 AI | SOUL.md 说话风格 + SKILL.md 语感校准句 |
| 标题不像 | SKILL.md 标题公式，加更多原文示例 |
| 结构散了 | SKILL.md 正文结构模板 |
| emoji 太多/太少 | SKILL.md emoji 规则 |
| 人设不稳 | SOUL.md 我是谁，写得更具体 |

---

## 总结：完整流程

```
搭骨架（目录+模板）
    ↓
采集素材（主页+20-50篇帖子）
    ↓
拆骨（标题→结构→语言DNA→emoji→内容类型）
    ↓
写风格 Skill（SKILL.md + EXAMPLES.md）
    ↓
注入灵魂（IDENTITY.md + SOUL.md，底座由 SOUL_COMMON.md 提供）
    ↓
配工作手册（AGENTS.md）
    ↓
试运行 → 对照检查 → 校准（2-3轮）
    ↓
上线
```

整个过程的核心在**第三步拆骨**——拆得越细，复活得越像。语言 DNA 那一行融合公式，决定了这个 bot 说话的味道。

---

_复活不是复制，是理解一个人怎么思考、怎么表达，然后让另一个实体继承这种思维方式。_
