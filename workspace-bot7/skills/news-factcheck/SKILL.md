# 投研资讯核查 /news-factcheck

**触发词**：`/news-factcheck`、"帮我核实一下"、"这个消息靠谱吗"、"研报说XX行业XX，真的吗"、"这个数据准吗"、"帮我验证这个观点"

---

## 核查框架

**确定今日日期 `TODAY`**，所有时间判断的基准。

---

### Step 1 — 提取可核查命题

从输入中提取**具体的、可被数据验证的断言**：

```
例：研报称"AI服务器行业2024年出货量增速超80%，景气度持续高位"
提取：
  命题A：AI服务器 2024年出货量增速超80%（查 IDC/行业数据）
  命题B：景气度"持续高位"（查主要公司毛利率、营收增速趋势）
  命题C：[时间词] "持续" → 到底指哪个时间段？

例：研报称"某公司是行业毛利率最高的玩家"
提取：
  命题A：该公司毛利率数值（查 tushare_fina_indicator）
  命题B：同行业其他公司毛利率对比（需横向数据）
```

**时间敏感词扫描**（重点）——检查输入中是否含有：
- 相对时间词：今日、昨日、本周、近期、最近、刚刚、今年、上月、近来
- 模糊频次：持续、一直、不断、连续
- 如有 → 必须确认原文发布日期，将相对时间锚定到绝对日期

---

### Step 2 — 时间轴核验

```
1. 找到原文发布日期（页面时间戳 / URL日期 / 文内"XX月XX日"）
   - 找不到：用 browser 工具打开原链接确认
   - 仍找不到：标记"发布时间不明，时效性无法确认"

2. 计算信息时效（TODAY - 发布日期）：
   ≤7天   → 🟢 新鲜，可作为当前判断依据
   8-30天 → 🟡 较旧，市场可能已部分消化
   31-90天→ 🔴 旧闻，只作背景，不作当前催化剂
   >90天  → ⚫ 历史，需重新核实是否仍成立

3. 将相对时间词替换为绝对日期：
   "近期出货量大增" → "约[发布日-30天]~[发布日] 出货量大增"
   "今年以来景气度高" → "[发布年份]年以来"（不一定等于今年）
```

---

### Step 3 — 数据核实

**行业级命题**（首选）：
- 景气度/需求/价格方向 → 搜索（zhipu MCP）`"{行业} 出货量/价格/需求 {对应年月}"`
- 市场份额/集中度 → 搜索 IDC/Gartner/行业协会等权威来源，确认数据年份
- 行业毛利率 → `tushare_fina_indicator` 对多家公司拉取，横向验证
- **市场观点/分析师态度类命题** → 用 browser 直接读原文，不能只依赖搜索摘要。**雪球**是市场观点的首选来源，东方财富研报（`https://data.eastmoney.com/report/industry.jshtml`）可查分析师评级。同时用 zhipu MCP 搜索覆盖 browser 触达不到的信息源。
- **原文链接类命题**（用户粘贴了一条资讯）→ 若链接可访问，用 browser 打开原链接核查原文，不要只看摘要

**个股级命题**（支撑行业判断用）：

| 命题类型 | 核实工具 |
|---------|---------|
| 股价/涨跌幅 | `tushare_daily` |
| PE/PB/市值 | `tushare_daily_basic` |
| 营收/利润 | `tushare_income` |
| 毛利率/ROE | `tushare_fina_indicator` |
| 北向资金 | `tushare_moneyflow_hsgt` + `tushare_hsgt_top10` |
| 业绩预告 | `tushare_forecast` |

**核实结论**：
- 数据与命题一致 → ✅ 核实通过（注明数据来源+日期）
- 数据与命题不符 → ❌ 存在出入（给出具体数值对比）
- 无法获取数据 → ⚠️ 无法核实（说明原因，不猜测）

---

### Step 4 — 逻辑一致性检查

检查命题内部及各命题之间的逻辑是否自洽：

```
行业级常见逻辑陷阱：
- "行业景气度高" + 主要公司毛利率持续压缩 → 矛盾（价格战抵消了需求增长）
- "赛道进入高速增长" + 龙头营收增速放缓 → 需解释（份额被抢？市场饱和？）
- "政策大力支持" + 行业公司ROE连续下滑 → 政策支持未传导至盈利
- 用历史高峰期数据论证当前 → 时间混淆（如"行业曾达到80%增速"暗示现在也是）
- "最新调研显示" + 调研时间是6个月前 → 旧数据冒充新信息
- 选择性引用：只列正面公司，忽略同期业绩差的公司 → 幸存者偏差
- 因果倒置："股价涨了所以基本面好" → 股价不是基本面的证据
```

对每个问题：说明**哪里有问题、正确的判断逻辑是什么**。

---

### Step 5 — 输出核查报告

```
# 资讯核查报告
📅 核查日期：{TODAY}
📄 核查对象：[原文摘要，≤50字]

## 时间核验
- 原文发布：{发布日期} → 信息时效：🟢/🟡/🔴/⚫
- 相对时间词校正（如有）：
  · "近期" → {实际日期范围}
  · "今年" → {具体年份}，距今 {X} 个月

## 命题核实

### 命题1：[具体陈述]
- 核实方式：[zhipu搜索 / Tushare数据]
- 实际数据：[具体数值 + 来源 + 日期]
- 结论：✅ 通过 / ❌ 出入（[具体差异]）/ ⚠️ 无法核实

### 命题2：[具体陈述]
[同格式]

## 逻辑核查
[存在问题则逐条说明；无问题写"命题间逻辑自洽"]

## 综合结论
[属实 / 基本属实 / 部分失实 / 存疑 / 明显有误]
核心问题：[若非属实，点出最关键的问题点]
可信度影响因素：[信息时效 + 数据来源质量 + 逻辑完整性]

## 数据来源
Tushare Pro | zhipu MCP | browser（东方财富/雪球，市场观点类命题必用）
```

## 注意事项

- **行业数据的来源权威性有高低**：IDC/Gartner > 行业协会 > 研报 > 媒体报道
- **"持续"是最危险的词**：必须确认到底是几个季度/年的数据，不能接受模糊表述
- **旧数据不等于假数据**：🔴 的数据可能本身准确，但时效性意味着当前可能已变化
- **无法核实不等于假**：数据查不到就如实说，不要因为查不到就倾向于否定
- **量化 > 定性**：用具体数字对比，不用"大致符合"、"基本一致"这类表述
