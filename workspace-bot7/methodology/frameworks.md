## 框架1：技术采纳S曲线

研究任何TMT行业，第一个问题：**这个技术在S曲线哪个位置？**

| 阶段 | 特征 | 渗透率 | 估值方法 | 投资策略 |
|------|------|--------|---------|---------|
| 实验室 | 无收入，纯叙事/论文 | <1% | 不可估值 | 不参与 |
| 早期商用 | 首批订单，深度亏损 | 1-5% | PS, EV/Revenue | 极高风险高回报 |
| **拐点期** | 收入加速，毛利改善 | 5-15% | PEG, PS | **最佳介入窗口** |
| 规模化 | 出货量翻倍，盈利兑现 | 15-50% | PE, DCF | 稳定配置 |
| 成熟期 | 增速放缓，价格战 | >50% | PE, 股息率 | 价值/收割 |

**关键判断信号**：
- 实验室→早期商用：首个商业合同签约
- 早期→拐点：季度收入增速环比连续提升
- 拐点→规模化：行业年出货量首次翻倍
- 规模化→成熟：龙头毛利率开始下滑

## 框架2：TMT 产业链价值分析

```
芯片设计(IP/EDA) → 晶圆制造 → 封装测试 → 模组/板卡 → 系统集成 → 应用/服务
       ↑                                                      ↓
    设备/材料                                              终端用户
```

**每一层问三个问题**：
1. **谁赚最多？** — 毛利率最高的环节（通常是设计和应用两端）
2. **瓶颈在哪？** — 当前供不应求的环节（瓶颈=定价权=超额利润）
3. **谁有定价权？** — 能涨价且客户不跑的公司

**TMT投资核心逻辑**：投瓶颈环节 > 投高毛利环节 > 投量大环节。瓶颈会转移，跟着瓶颈走。

## 框架3：预期差模型

Alpha的来源是预期差。每次研究填这张表：

| 维度 | 市场共识 | 我的判断 | 差异方向 | 数据支撑 |
|------|---------|---------|---------|---------|
| 景气度 | | | 高估/低估/一致 | |
| 竞争格局 | | | | |
| 估值水平 | | | | |
| 催化剂 | | | | |

**三条铁律**：
1. 没有预期差 = 没有超额收益 = 不值得重仓
2. 有预期差但没催化剂 = 价值陷阱，耐心等
3. 有预期差 + 有催化剂 + 有时间线 = 机会

## 框架5：成长股估值工具箱

| 公司类型 | 首选估值 | 辅助指标 | 避坑 |
|---------|---------|---------|------|
| 亏损高增长 | PS, EV/Revenue | 用户数/订单增速 | PE此时无意义 |
| 盈利高增长 | PEG | PE + 未来2年增速 | PEG<1=相对便宜 |
| 稳定盈利 | PE, DCF | ROE, 自由现金流 | 看现金流质量 |
| 周期性TMT | PB, 利润中枢PE | 库存周期位置 | PE最低=景气顶部（反直觉） |
