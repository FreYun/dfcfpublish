Claude Code 源码泄露，51.2 万行代码全网裸奔。

先说我的判断：这不是八卦，是 AI 投资逻辑的拐点信号。

[一] 护城河在哪？不在模型，在工程

为什么 60k 人连夜 Fork？因为 Claude Code 真正的护城河，从来不是模型参数。

AI 大牛拆解出 6 大技术：实时上下文加载、Prompt 缓存复用、专用工具链、上下文压缩、结构化记忆、子 Agent 并行。

这意味着什么？

**只会调 API 的"套壳"公司，护城河比纸还薄。** 真正有工程化能力的，才能把 AI 落地到生产场景。

[二] 我的投资判断：谁在裸泳

AI 工程化拐点，三类公司要分化：

1. **利空**——纯 API 套壳、没有工程优化能力的"AI 应用"
2. **利好**——有真实场景、能把 AI 嵌入工作流的 ToB 软件
3. **观望**——国产模型厂商，ARPU 和商业化进度才是关键

说点实际的：
- 初级编码岗位被重塑→代码生成类工具先受益
- 代码 review 被重塑→DevOps、测试自动化需求增加
- 文档维护被重塑→知识库、协同办公有增量

[三] 国产模型的机会与风险

Kimi、GLM、Qwen 都在追，但差距不在模型本身。

**机会**：工程化能力可以补课，国产替代逻辑还在
**风险**：光有 DAU 不够，付费转化率才是生死线

参考 Character.ai 的前车之鉴——月活再高，商业化兑现不了，估值就是泡沫。

[打卡] 总结

源码泄露是偶然，AI 工程化是必然。

投资逻辑变了：从"谁有大模型"到"谁能把 AI 真正落地"。

护城河在工程优化里，在真实场景里，在付费意愿里。
