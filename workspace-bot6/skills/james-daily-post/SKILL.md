# 爱理财的James · 日常发帖 Skill

> 装备即生效。本 skill 只管「写什么」，写完后交给 `scheduled-post` 管「怎么发」。

---

## 入口：触发方式

> ⚠️ 本 skill 仅供 **cron 定时任务**自动触发。研究部直接说「去发帖」「投稿」等指令时，**不走本 skill**，直接按研究部要求投稿即可。

| 触发 | 进入步骤 |
|------|---------|
| **定时任务（cron）触发** | 从 **Step 1** 开始 |
| 收到 `[MSG:xxx]` 含「✅」/「已批准」 | 跳到 **scheduled-post Step C**（Read `skills/scheduled-post/SKILL.md`） |
| 收到 `[MSG:xxx]` 含修改意见/「打回」 | 跳到 **scheduled-post Step D**（Read `skills/scheduled-post/SKILL.md`） |

## Step 1 · 看大盘行情

调用 research-mcp 拉多资产行情快照：

```
npx mcporter call "research-mcp.market_overview()"
```

按需补充：

```
# 指数
npx mcporter call "research-mcp.get_index_daily(ts_code: '000001.SH', limit: 1)"
# 商品（黄金、原油等）
npx mcporter call "research-mcp.get_commodity_price(symbol: 'AU')"
# 债券/利率（如有宏观话题）
npx mcporter call "research-mcp.get_macro_data(indicator: '10Y_BOND_YIELD')"
```

**判断当日行情类型：**

| 条件 | 行情类型 | 发帖方向 |
|------|---------|---------|
| 主要指数或商品波动 ≥2% | 强信号日 | → 市场解读（优先） |
| 热搜有「基金」「大跌」「大涨」「降息」等 | 热点日 | → 市场解读 / 事件分析 |
| 自己有调仓计划或本周做了调仓 | 调仓日 | → 组合配置分享 |
| 勒布朗/NBA 有热门事件 | 篮球日 | → 程序员×理财×篮球 |
| 平淡横盘，无明显事件 | 普通日 | → 基金科普 / 程序员理财日常 |

> 行情数据拉不到时，默认走「基金科普」。

---

## Step 2 · 选题

### 内容类型配比（长期维持）

| 类型 | 目标占比 | 典型选题 |
|------|---------|---------|
| 组合配置 / 调仓分享 | 35% | 当前持仓逻辑、调仓原因、配置比例 |
| 市场解读 / 策略分析 | 25% | 行情分析、资产轮动、配置建议 |
| 基金科普 / 知识分享 | 20% | 用程序员思维讲基金概念 |
| 收益复盘 / 阶段总结 | 10% | 组合收益率、归因分析、经验教训 |
| 程序员×理财×篮球 | 10% | 码农理财日常、老詹聊球、退休梦想 |

### 选题流程

1. 读 `memory/发帖记录.md` — 确认今天没发过，统计近 7 天各类型发帖数
2. **行情驱动**：强信号日 / 热点日 / 调仓日 → 直接定选题方向
3. **配比驱动**：普通日 → 看近 7 天哪个类型占比最低，优先补缺口
4. 读 `memory/topic-library.md`（如有）— 从待写区取具体选题
5. 无选题库时，从以下常驻方向中选：

**基金科普常驻题库：**
- 指数基金 vs 主动基金怎么选
- 夏普比率 / 最大回撤 / 年化收益，一次讲清
- 商品基金的对冲逻辑（像代码里的 try-catch）
- 固收+ 到底「+」在哪里
- 定投 vs 一把梭，用回测数据说话
- 资产配置的「三条腿」：指数、商品、固收+

**篮球×理财常驻题库（先 Read `skills/james-topic-research/SKILL.md`）：**
- 勒布朗的持久力 × 定投的复利效应
- NBA 数据分析 × 基金归因分析
- 「老詹」外号撞名的趣味入口

## Step 3 · 写稿

**先 Read `skills/james-style/SKILL.md`** 回顾写稿规范、硬性检查点和自查清单，按其执行。

**本 skill 仅补充一条硬约束：** 涉及行情的数字必须来自 Step 1 数据，不凭记忆写。

---

## Step 4 · 生成封面文字（AI 生图可选）

**默认走 `text_to_image`，AI 生图不是必经步骤。** 这一阶段默认只需要产出封面文字，供 `scheduled-post` 在投稿时生成文字卡片。

封面文字要求：

- 4-6 字优先，最长不超过 8 字
- 要像标题的“第一锤”，但不要和标题一字不差重复
- 市场解读优先写判断或情绪，如「别急着抄底」「今天有点猛」
- 基金科普优先写认知钩子，如「回撤不是亏完」「别把分散配错」
- 篮球/个人表达类优先写态度，如「像老詹那样稳」「别只看一场球」
- 默认只交付文字，不生成图片路径

如用户或上游任务**明确要求使用图片笔记 / AI 生图**，再额外 Read `skills/james-cover/SKILL.md` 走封面图流程；否则不要把 AI 生图当成必经步骤，一律保持 `text_to_image`。

> 当前默认策略：先写内容与封面文字，直接按 `text_to_image` 发稿；只有在明确要求图片笔记时才补做封面图。

## Step 5 · 交给 scheduled-post 走审批投稿

内容生产完成，确认已产出以下变量：

| 变量 | 来源 |
|------|------|
| `{标题}` | Step 3 写稿产出 |
| `{正文}` | Step 3 写稿产出 |
| `{封面文字}` | Step 3 封面卡片文字（≤8字） |
| `{标签}` | Step 3 选定的标签（5-7个） |
| `{图片路径}` | 默认填 `无`；仅图片笔记场景才填写实际路径 |
| `{发帖模式}` | 默认填 `text_to_image`；仅明确要求图片笔记时才改 |
| `{类型描述}` | 如「市场解读 · 大盘大跌」或「基金科普 · 夏普比率」 |

**→ Read `skills/scheduled-post/SKILL.md`，从 Step A 开始执行。**

完成后更新：
- `memory/topic-library.md`（如有，将选题标记为「待发」）
- 当日日记 `memory/YYYY-MM-DD.md`
